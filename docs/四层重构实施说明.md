# 四层重构实施说明

## 0. 文档定位

本文是当前仓库真实实现的同步说明，重点回答三件事：

1. 四层主链现在如何落地
2. 哪些旧持久化机器产物已经被删除
3. 为什么现在只允许 `canonical graph` 持久化

本文不描述历史方案，只描述当前生效状态。

## 1. 当前主链与真相源

仓库主链保持为：

```text
Framework -> Config -> Code -> Evidence
```

当前硬约束如下：

- `framework/*.md` 是作者源
- `projects/*/project.toml` 是唯一项目配置入口
- `projects/*/generated/canonical.json` 是唯一机器持久化真相源
- 任何机器判定关系都必须来自 canonical graph

## 2. 持久化边界（本次收敛结果）

本次收敛后，机器持久化边界被明确限制为一项：

- 允许：`projects/<project_id>/generated/canonical.json`
- 不允许：任何派生 tree/report/snapshot 的持久化文件

因此，以下旧持久化路径已移除：

- `projects/*/generated/runtime_snapshot.py`
- `docs/hierarchy/shelf_framework_tree.json`
- `docs/hierarchy/shelf_framework_tree.html`
- `docs/hierarchy/shelf_evidence_tree.json`
- `docs/hierarchy/shelf_evidence_tree.html`

以下旧脚本已移除：

- `scripts/generate_framework_tree_hierarchy.py`
- `scripts/generate_evidence_tree_hierarchy.py`
- `scripts/generate_module_hierarchy_html.py`
- `src/hierarchy_models.py`

## 3. 四层实现现状

### 3.1 Framework

- 入口：`framework/<domain>/Lx-Mn-*.md`
- 解析：`src/framework_ir/parser.py`
- 输出：每个 markdown 生成独立 `FrameworkModule`，并显式产出 `Base` 与 `Rule` 结构

### 3.2 Config

- 入口：`projects/<project_id>/project.toml`
- 编译：`src/project_runtime/config_layer.py`
- 规则：`communication` 负责协作约束，`exact` 负责 Code 精确消费字段

### 3.3 Code

- 编译：`src/project_runtime/code_layer.py`
- 规则：每个 `CodeModule` 只消费对应 `ConfigModule.exact_export`

### 3.4 Evidence

- 编译：`src/project_runtime/evidence_layer.py`
- 规则：Evidence 只消费 Code export
- 输出：Evidence 数据进入 canonical 的 `evidence` 段，不额外落盘独立 report/snapshot

### 3.5 Canonical

- 编译与物化：`src/project_runtime/compiler.py`
- 输出目标：`generated/canonical.json`
- 结构：`project/framework/config/code/evidence/links`

## 4. 工具链行为（收敛后）

### 4.1 Materialize

- 脚本：`scripts/materialize_project.py`
- 行为：仅物化 canonical；并清理 `generated/` 下非声明产物

### 4.2 Validate

- 脚本：`scripts/validate_canonical.py`
- 行为：按 canonical 主链执行校验，支持 `--check-changes`
- 新增语义门禁：若 `communication / exact` 出现超出 framework projection 的路径，校验必须以 `FRAMEWORK_VIOLATION` 失败

### 4.3 VSCode 插件（Shelf AI）

插件入口：`tools/vscode/shelf-ai/extension.js`

当前行为：

- framework tree 与 evidence tree 都是运行时投影，不再读写 `docs/hierarchy/*`
- evidence tree 只在 canonical fresh 时开放正式视图
- `projects/*/generated/*` 受保护，不允许手改
- 导航与证据索引只基于 canonical 与作者源，不引入平行持久化真相

正式约束文档：

- `tools/vscode/shelf-ai/插件设计与实现契约.md`

## 7. 一句话结论

当前仓库已收敛为：

> 只有 canonical graph 一个机器持久化真相源；其它结构均为运行时投影或可丢弃缓存，不再作为持久化机器输入。
