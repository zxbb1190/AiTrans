# 全链实现框架与跳转逻辑详解

## 1. 当前主链

当前仓库的实现链路已经固定为：

```text
Framework Markdown
  -> Framework IR
    -> Package Registry
      -> selection.roots
        -> Module Tree
          -> Config Slicing
            -> Package Compile
              -> Package runtime_exports
                -> Runtime Bundle
                  -> Canonical Graph
                    -> Derived Views / Validation
```

## 3. 根模块选择

`selection` 已改为 `[[selection.roots]]`：

- `slot_id` 表达项目内稳定引用名
- `role` 表达 runtime scene 所需语义位
- `framework_file` 才是进入 registry 与 module tree 的正式入口

当前知识库项目通过三个 root role 绑定：

- `frontend`
- `knowledge_base`
- `backend`

但编译器不再把它们写死成固定结构槽位。

## 4. runtime 收敛

runtime 不再依赖专门 builder 手工汇总 `frontend_contract / workbench_contract / ui_spec / backend_spec`。

现在的主路径是：

1. root package 在 `compile(...)` 阶段直接产出 `runtime_exports`
2. `project_runtime.pipeline.assemble_runtime_exports(...)` 汇总导出
3. `KnowledgeBaseRuntimeBundle` 只作为汇总结果与运行时消费对象
