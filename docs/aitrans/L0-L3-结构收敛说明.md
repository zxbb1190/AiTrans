# framework L0-L3 结构收敛说明

@framework
## 1. 本次收敛原则

- 以仓库当前 `L0 -> L1 -> L2 -> L3` 单向收敛标准为准，重新对齐 `aitrans` 的层级语义。
- `B*` 继续优先表达结构，不把动作清单、状态清单或实现对象直接当作高层 `B*` 主句。
- 最低可复用、自足的叶子结构统一落到 `L0`；需要显式组合多个叶子结构的控制面落到 `L1`。
- 领域组合层统一落到 `L2`；总框架根落到 `L3`，只负责收敛五大能力面，不再反向充当叶子层。
- `L0` 停止继续下钻；继续拆分若只剩实现类、函数集合或 `Handler / Builder / Mapper`，则判为越出框架层。

## 2. 当前层级语义

- `L0`：叶子结构层，只保留最小、可复用、自足的稳定结构单元。
- `L1`：控制面与领域模块层，显式组合 `L0` 叶子，承接单一链路内的任务、适配、布局、治理等控制面。
- `L2`：领域组合层，显式组合 `L1` 模块，输出桌面宿主、截图交互、识别翻译、结果展示、运行治理五大结构面。
- `L3`：框架总根，只承接整体能力范围、顶层边界与跨领域组合规则。

## 3. 本次具体收敛结果

- 旧 `L3-M0 ~ L3-M59` 已整体重映射为新 `L0-M0 ~ L0-M59`，统一作为叶子结构层。
- 旧 `L2-M0 ~ L2-M24` 已整体重映射为新 `L1-M0 ~ L1-M24`，统一作为控制面与领域模块层。
- 旧 `L1-M0 ~ L1-M4` 已整体重映射为新 `L2-M0 ~ L2-M4`，统一作为五大领域组合层。
- 旧 `L0-M0-桌面截图翻译框架` 已重映射为新 `L3-M0-桌面截图翻译框架`，作为框架总根。
- 文档内的 `Lx.My[...]`、`Lx.My[...]` 与 `Lx-Mn-*.md` 引用已按上述映射同步平移，保持依赖方向单向向下。
- 在后续持续收敛后，当前 `aitrans` 已进一步压缩为连续编号的 `L0-M0 ~ L0-M35` 与 `L1-M0 ~ L1-M22`，不再保留空洞 `M` 序号。

## 4. 后续建议

- 下一步如果继续收敛，不应再机械增加层级，而应优先检查同层模块是否在重复同一拆分轴。
- 若发现多个 `L0` 叶子只是重复同一组织形式，应优先合并模块、减少 `B*` 数量，而不是继续增加叶子数量。
- 若发现多个 `L1` 模块共享大量会话、请求、错误或输出骨架，应优先抽共享 `L0` 叶子，再回收上层重复描述。
