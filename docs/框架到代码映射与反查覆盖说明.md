# 框架到代码映射与反查覆盖说明

当前正式主链已经收敛为：

```text
framework/*.md
  -> framework_ir
    -> framework_packages.registry
      -> project.toml selection.roots
        -> project_runtime.config_loader
          -> project_runtime.module_tree
            -> project_runtime.package_config
              -> package compile
                -> package runtime_exports
                  -> project_runtime.pipeline
                    -> generated/canonical_graph.json
```

## 正向映射

关键入口：

- [src/framework_ir/parser.py](../src/framework_ir/parser.py)
- [src/framework_packages/contract.py](../src/framework_packages/contract.py)
- [src/framework_packages/registry.py](../src/framework_packages/registry.py)
- [src/project_runtime/config_loader.py](../src/project_runtime/config_loader.py)
- [src/project_runtime/module_tree.py](../src/project_runtime/module_tree.py)
- [src/project_runtime/package_config.py](../src/project_runtime/package_config.py)
- [src/project_runtime/pipeline.py](../src/project_runtime/pipeline.py)

当前约束：

- 每个 framework 文件对应一个 package
- 每个 package 只有一个正式入口 class
- `project.toml` 通过 `[[selection.roots]]` 表达根模块树
- package 只能消费自己声明的字段级 config contract
- runtime bundle 由 package `runtime_exports` 汇总，不再走手工 builder 主路径

## 反向追溯

从运行时回查 framework，稳定路径是：

```text
runtime export
  -> canonical_graph.layers.code.package_results
    -> canonical_graph.layers.framework.registry_binding
      -> framework file
```

对应真相源：

- [projects/knowledge_base_basic/generated/canonical_graph.json](../projects/knowledge_base_basic/generated/canonical_graph.json)
