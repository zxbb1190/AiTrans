# GitHub Launch Kit For Shelf

This file is the shortest path from "the repo is technically solid" to "the repo looks ready for public discovery".

Use the content below directly in the GitHub repository settings UI and launch posts.

## Recommended repo identity

- Brand name: `Shelf`
- Target GitHub repo name: `shelf`
- Safe fallback only if `shelf` is unavailable: `shelf-framework`
- Avoid as the public repo name: `framework`

Reason:

- `Shelf` is the brand
- `framework` is too generic for search, memory, and sharing

## GitHub About field

Recommended:

`Structure-first AI coding framework that turns framework docs, product specs, and implementation configs into runnable systems with strict mapping validation.`

Shorter option:

Use this carefully and sparingly:

`Spec Kit is workflow-first. Shelf is structure-first.`

Longer version:

`Spec Kit helps agents execute a feature workflow. Shelf defines the structural language that the workflow should obey before implementation starts.`

## Immediate GitHub UI checklist

- set the About field
- add Topics
- upload a Social Preview image
- preferred file: `docs/github-social-preview.png`
- pin the strongest screenshot or short demo GIF near the top of the README
- enable Discussions if you want public architecture questions instead of scattered issues
- rename the repo to `shelf`
- if `shelf` is unavailable, use `shelf-framework`
- after renaming, follow [repo-rename-checklist.md](./repo-rename-checklist.md)

## What not to do

- do not buy fake stars or fake forks
- do not make the README sound bigger than the current proof
- do not hide the core chain behind vague AI marketing language

The differentiator is not "AI writes code".

The differentiator is that Shelf gives AI a real structural language to work inside.
