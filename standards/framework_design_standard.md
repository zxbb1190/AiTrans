# 框架设计标准（核心）

## 0. 定位
本文件只定义框架设计的核心方法，不包含工程执行约束。

核心范围仅包含五项：
1. 目标
2. 边界定义
3. 模块（最小可行基）
4. 组合原则
5. 验证

## 1. 目标（Goal）
定义框架要优化的对象与结果。

要求：
- 目标必须可度量
- 目标必须可验证
- 目标必须可与边界建立约束关系

## 2. 边界定义（Boundary）
定义框架适用范围与限制条件。

要求：
- 每个边界项都可测量
- 每个边界项都可校验
- 边界项需与目标直接相关

## 3. 模块（最小可行基，Module）
定义最小可运行能力单元（Minimal Viable Base）。

要求：
- 模块职责单一
- 模块可组合
- 模块可替换

## 4. 组合原则（Combination Principles）
定义模块如何形成有效系统。

要求：
- 组合规则必须可表达为显式条件
- 规则间不可互相冲突
- 组合结果必须可判定“有效/无效”

## 5. 验证（Verification）
定义如何判断框架结论是否成立。

要求：
- 必须有基线（baseline）
- 必须有目标值（target）
- 必须有通过条件（pass criteria）
- 必须有失败原因（failure reasons）
