# 逻辑自洽框架设计标准（通用，L0）

## 0. 治理地位与多级严格映射（强制）
本文件是仓库最高规范（L0）。

### 0.1 多级映射模型
- `L0`：通用抽象标准（本文件）
- `L1`：领域标准（如 `shelf_framework_standard.md`）
- `L2`：领域实现与验证机制（如 `shelf_framework.py`、`main.py`、`scripts/validate_strict_mapping.py`）
- `L3`：运行证据产物（如 `docs/logic_record.json`）

### 0.2 严格映射规则
- `L0 -> L1 -> L2` 是强制传导链。
- 当 `L0` 发生变更时，`L1` 与 `L2` 必须同步更新。
- 当 `L1` 发生变更时，`L2` 必须同步更新。
- 当 `L2/L3` 发生变更时，必须执行反向验证，确认未偏离 `L0/L1`。

### 0.3 机器可读映射与验证
- 映射注册文件：`mapping_registry.json`
- 验证命令：
```bash
uv run python scripts/validate_strict_mapping.py
uv run python scripts/validate_strict_mapping.py --check-changes
```

## 1. 文档定位
本标准用于定义“如何设计一个框架”，而不是定义具体业务。

目标：
- 可解释：每个设计决策有来源和作用
- 可验证：每个关键结论可被验证或推翻
- 可演化：规则可随证据迭代，而不是固定教条

## 2. 抽象元模型（统一语言）
任何框架都应抽象为：
- `G` Goal：目标
- `B` Boundary：边界
- `M` Module：模块
- `R` Rule：规则
- `H` Hypothesis：假设
- `V` Verification：验证
- `C` Conclusion：结论

推荐依赖链：
`G -> B -> M -> R -> H -> V -> C`

## 3. 设计原则（强制）

### P1. 来源一致性
每个 `B/M/R` 必须可追溯到 `G`。

### P2. 作用单一性
模块应保持主职责单一，多目标职责需显式拆分。

### P3. 规则可推导
`R` 必须由 `B + M` 推导，不允许先定规则再反推边界。

### P4. 假设可证伪
`H` 必须可被实验或数据推翻。

### P5. 约束最小化
只保留对目标有净贡献的约束。

### P6. 变更可逆
规则与模块变更必须含影响分析与回滚路径。

## 4. 设计接口标准

### 4.1 Boundary 接口
- 输入：约束维度列表
- 输出：`is_valid` 与错误原因
- 要求：每个维度可测量

### 4.2 Module 接口
- 输入：模块定义与能力声明
- 输出：可组合模块集合
- 要求：可枚举、可替换

### 4.3 Rule 接口
- 输入：模块组合与边界条件
- 输出：布尔判定 + 规则 ID
- 要求：每条规则可独立执行

### 4.4 Verification 接口
- 输入：边界、组合、基线、目标、上下文
- 输出：`passed` 与 `reasons[]`
- 要求：失败可解释

### 4.5 LogicRecord 接口
- 输入：`step_id`, `depends_on`, `evidence`
- 输出：自洽性检查结果
- 要求：支持审计导出（JSON）

## 5. 自洽性判定标准

### 5.1 完整性
- 每个 `G` 至少关联一个 `B`
- 每个 `C` 必须有 `V` 证据

### 5.2 无冲突性
- 同层规则不可互相矛盾
- 冲突必须定义优先级或权重

### 5.3 可追溯性
- `step_id` 唯一
- `depends_on` 仅能依赖前序步骤
- 不允许孤儿结论

### 5.4 可删减性
删除规则后目标不受损，则该规则不应为强制规则。

## 6. 反向验证标准（L2/L3 -> L1/L0）
当实现或运行证据变更时，至少完成：
1. 运行严格映射验证脚本（结构 + 变更链路）
2. 核对 `mapping_registry.json` 的映射项是否仍成立
3. 若验证失败：回滚实现或补齐标准更新

## 7. 工程约束（仓库规范）
- Python 与依赖管理必须使用 `uv`
- 开发与运行命令必须通过 `uv run ...`
- 新增依赖必须通过 `uv add <package>`，并提交 `pyproject.toml` 与 `uv.lock`
- 不允许把 `pip install` 写入仓库标准流程

## 8. 最小交付产物
- `L0/L1` 标准文档
- `L2` 可执行实现与验证脚本
- `L3` 逻辑记录证据文件
- 映射注册表（`mapping_registry.json`）
