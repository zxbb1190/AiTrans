# Shelf AI 0.0.33

## 中文说明

- 这一版把 Shelf AI 的边界跳转目标从旧的 `instance.toml` 约定切到了新的 `product_spec.toml` 约定。现在从框架边界跳到项目配置时，会直接落到当前仓库真正使用的产品规格文件，而不是历史命名。
- 边界 hover 卡片和相关测试也同步更新到了新的仓库分层：`Framework -> Product Spec -> Implementation Config -> Code -> Evidence`。这样插件里的导航语言、仓库里的生成链和运行时输出保持一致，不再混用旧术语。
- 这次发布主要是仓库结构迁移和导航语义校正，不是新的交互功能发布。若你之前已经打开过 Shelf AI 的 framework tree，升级后建议执行一次 `Shelf: Refresh Framework Tree`。

## English Notes

- This release switches Shelf AI boundary navigation from the legacy `instance.toml` convention to the new `product_spec.toml` convention. Boundary jumps now land on the actual product-spec file used by the repository instead of the historical config name.
- Boundary hover cards and repository-facing tests are also aligned with the new repository layering model: `Framework -> Product Spec -> Implementation Config -> Code -> Evidence`. This keeps extension terminology consistent with the compiler, runtime, and generated outputs.
- This is primarily a repository-structure and navigation terminology update rather than a new interaction feature release. If you already had the framework tree open, run `Shelf: Refresh Framework Tree` after upgrading.

## 安装 / Install

- 从本页面下载对应版本的 VSIX 安装包
- 在 VS Code 中执行 `Extensions: Install from VSIX...`
- 如果你已经打开过框架树页面，更新后请执行一次 `Shelf: Refresh Framework Tree`，或关闭并重新打开 `Shelf AI : Framework Tree`
