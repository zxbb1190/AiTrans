# 0.1.5

## 中文说明

- 本次发布恢复了 Shelf AI 的图形化框架树和证据树，重新提供缩放、拖拽、hover 与侧栏详情，而不是只输出纯文本列表。
- 框架树与证据树继续从当前 canonical 主链派生，没有恢复旧治理链或旧平行真相源。
- 发布相关的 CI、仓库模板与本地 `pre-push` 钩子已同步切到 `validate_canonical.py`，避免新版本继续引用已删除的 strict-mapping 校验入口。

对使用者可见的影响：

- `Shelf: Open Framework Tree` 与 `Shelf: Open Evidence Tree` 会重新打开交互式图形视图
- `Shelf: Refresh Framework Tree` 与 `Shelf: Refresh Evidence Tree` 生成的 HTML 不再退化成简单列表
- 新推送与新 tag 对应的仓库校验链会继续围绕 canonical 主链运行

安装或获取方式：

1. 下载发布资产 `shelf-ai-0.1.5.vsix`
2. 在 VSCode 中执行 `Extensions: Install from VSIX...`
3. 选择该文件完成安装

注意事项：

- `projects/knowledge_base_basic` 仍只用于本地验证、演示和回归测试，不属于公开发布物
- 若你已经打开过旧的 tree 标签页，安装或更新后建议重新执行一次刷新命令以获取新的图形化产物

## English Notes

- This release restores the interactive framework and evidence graph views in Shelf AI, bringing back zoom, drag, hover, and side-panel inspection instead of plain HTML lists.
- The framework tree and evidence tree still derive from the current canonical mainline; no legacy governance pipeline or parallel truth source was reintroduced.
- Release-facing CI, repository templates, and the local `pre-push` hook now validate against `validate_canonical.py`, so the new version no longer points at the removed strict-mapping entrypoints.

User-visible impact:

- `Shelf: Open Framework Tree` and `Shelf: Open Evidence Tree` open interactive graph views again
- `Shelf: Refresh Framework Tree` and `Shelf: Refresh Evidence Tree` no longer regenerate tree HTML as plain lists
- New pushes and version tags now stay aligned with the canonical-centered validation pipeline

Install / access:

1. Download the release asset `shelf-ai-0.1.5.vsix`
2. Run `Extensions: Install from VSIX...` in VSCode
3. Select the file to complete installation

Notes:

- `projects/knowledge_base_basic` remains a local validation, demo, and regression sample only; it is not a public release deliverable
- If you still have an old tree tab open, refresh the tree once after updating so VSCode reloads the regenerated graph artifact
