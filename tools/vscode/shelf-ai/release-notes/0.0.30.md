# Shelf AI 0.0.30

## 中文说明

- 这一版把框架树主视图从“全局 `L0/L1/L2` 横带”改成了“按 framework 文件夹分组的列布局”。现在 `frontend`、`knowledge_base`、`backend` 等框架会各自保留自己的本地 `Lx` 层级，组间顺序则按照真实跨框架引用排序，不再把不同框架的同名 `L0` 混在同一条全局横带里。
- 严格映射与框架校验也改成了“引用优先”模型：本地模块引用只要求指向当前框架内更低的本地层，外部引用只要求能解析到真实框架模块，并新增了整张框架引用图的无环校验 `FW029`。
- 仓库中的 frontend 框架文档也同步做了一轮压缩重构，减少了过度拆分的基础模块数量，并重新生成了框架树与映射产物。更新后建议执行一次 `Shelf: Refresh Framework Tree`，以确保本地树图页面与新结构保持一致。

## English Notes

- This release changes the framework-tree primary view from one global `L0/L1/L2` swimlane into framework-grouped columns. `frontend`, `knowledge_base`, `backend`, and the other framework directories now keep their own local `Lx` bands, while group ordering follows the actual cross-framework reference graph.
- Strict mapping and framework validation now use a references-first model: local inline refs only need to point downward inside the same framework, external refs only need to resolve to real framework modules, and the full framework inline-ref graph is now checked for cycles with `FW029`.
- The repository's frontend framework docs were also compressed and rebuilt to remove over-splitting at the base layer, and the framework-tree / mapping artifacts were regenerated from that new structure. After updating, you should run `Shelf: Refresh Framework Tree` once so the local graph page picks up the regenerated view.

## 安装 / Install

- 从本页面下载对应版本的 VSIX 安装包
- 在 VS Code 中执行 `Extensions: Install from VSIX...`
- 如果你在仓库里使用 Shelf AI 树图，请更新后执行一次 `Shelf: Refresh Framework Tree`
