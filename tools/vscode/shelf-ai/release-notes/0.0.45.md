## 中文说明

- 这次收紧了框架标准里 `基 / 组合原则 / 正向能力` 的关系表达。每个正向能力在最小可行基层现在必须归属到且只归属到一个 `B*` 来源表达，而 `R*` 仍然允许从多个基出发并导出多个能力；相关 lint、严格校验和现有框架文档已经一起收口。
- 修复了 Shelf AI 对已编辑文档残留旧报错的问题。编辑受监控文档时，扩展现在会先清掉该文件的旧 Shelf diagnostics，避免文档已经改对但 Problems 面板还挂着过期错误。
- 失败提示也改得更直接了：诊断文案前会带 `✖`，状态栏在自动触发或手动触发校验失败时都会切到明确的失败态，不再出现底部仍显示 `Shelf OK` 但 Problems 面板已经报错的误导状态。
- 安装方式：下载本版本附带的 `shelf-ai-0.0.45.vsix`，在 VS Code 中执行 `Extensions: Install from VSIX...` 安装。若升级前已经打开相关框架文档或树图页面，安装后建议执行一次 `Developer: Reload Window`，确保新的诊断与状态栏逻辑生效。

## English Notes

- This release tightens the framework relation model between bases, combination principles, and positive capabilities. Each positive capability must now map to exactly one `B*` source expression at the minimum-viable-base layer, while `R*` entries still legitimately combine multiple bases and produce multiple capabilities; the lint rules, strict validation, and existing framework docs have been synced to match.
- Shelf AI now clears stale diagnostics for edited watched documents. If you fix a framework document, the extension removes outdated Shelf diagnostics for that file before the next validation result lands, so the Problems panel no longer lingers on obsolete errors.
- Failure signaling is also much clearer: diagnostic messages now carry a visible `✖` prefix, and the status bar switches to an explicit failure state for both auto-triggered and manual validation failures instead of leaving a misleading `Shelf OK`.
- Install by downloading the bundled `shelf-ai-0.0.45.vsix` and running `Extensions: Install from VSIX...` in VS Code. If you already had related framework documents or tree pages open before upgrading, run `Developer: Reload Window` after installation so the refreshed diagnostics and status-bar logic take effect.
