## 中文说明

- 修复了 Shelf AI 在长时间打开后可能出现的严格校验失活问题。校验命令现在带超时保护；如果上一次校验已经卡住，手动执行 `Shelf: Validate Mapping Now` 会主动重启陈旧任务，而不再只能依赖 `Reload Window` 恢复。
- 修复了边界跳转对知识库模板的硬编码偏置。边界导航现在优先读取已物化项目的 `generated/governance_manifest.json`，从其中解析 framework 与 `product_spec.toml` section 的真实对应关系，所以像 `document_chunking` 这类自定义 framework 只要项目已物化，就能跳到正确的 `projects/*/product_spec.toml` 位置。
- 这一版还补上了对应回归测试：一组覆盖校验命令超时与陈旧任务重启，另一组覆盖自定义 framework 通过治理清单完成边界跳转，避免后续再次退回到知识库专用写死逻辑。
- 安装方式：下载本版本附带的 `shelf-ai-0.0.46.vsix`，在 VS Code 中执行 `Extensions: Install from VSIX...` 安装。若你已经打开了仓库窗口，升级后建议执行一次 `Developer: Reload Window`，确保新的校验调度与导航逻辑完全生效。

## English Notes

- This release fixes the long-lived strict-validation stall path in Shelf AI. Validation commands now run with timeout protection, and if a previous validation task has gone stale, a manual `Shelf: Validate Mapping Now` request will restart it instead of requiring `Reload Window` to recover.
- Boundary navigation no longer stays biased toward the knowledge-base demo. When a project has already been materialized, the extension now prefers `generated/governance_manifest.json` to resolve the real framework-to-`product_spec.toml` section mapping, so custom frameworks such as `document_chunking` can jump to the correct `projects/*/product_spec.toml` location.
- This release also adds regression coverage for both fixes: one test path covers command timeout / stale-task restart behavior, and another covers boundary navigation for a custom framework resolved through generated governance metadata.
- Install by downloading the bundled `shelf-ai-0.0.46.vsix` and running `Extensions: Install from VSIX...` in VS Code. If you already have the workspace open, run `Developer: Reload Window` after upgrading so the refreshed validation scheduling and navigation logic take effect.
