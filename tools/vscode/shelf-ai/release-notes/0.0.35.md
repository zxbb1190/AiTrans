# Shelf AI 0.0.35

## 中文说明

- 这一版把框架 Markdown 的固定格式补全补齐了，不再只靠用户记住 snippet 前缀。现在在框架文档里可以直接补全 `@framework`、五个标准主章节标题、`C/P/B/R/V` 条目，以及 `R*.1 ~ R*.4` 子项。
- 同时保留了显式命令 `Shelf: Insert Framework Module Template` 作为保底入口。也就是说，就算编辑器当前没有正确弹出 snippet 建议，仍然可以直接插入完整标准模板。
- 仓库规范也同步强化了这件事：plain `@framework` 对应的标准模板入口现在被视为框架作者基本能力，不能再随意删除或降级成“自己记格式”。这次发布包含了相应的回归测试，后续若有人删掉这类入口，发布前会直接失败。

## English Notes

- This release adds fixed-shape framework-markdown completions so authoring no longer depends on remembering snippet prefixes alone. In framework files, Shelf AI now completes `@framework`, the five standard top-level section headings, `C/P/B/R/V` entries, and `R*.1 ~ R*.4` rule child lines.
- The explicit fallback command `Shelf: Insert Framework Module Template` remains available. Even if snippet suggestions are not showing correctly in the editor, you can still insert the full standard template directly.
- Repository standards are now aligned with this requirement: the plain `@framework` standard-template entry is treated as a hard framework-authoring capability rather than an optional editor convenience. This release also adds regression coverage, so future removals of that entry will fail before release.

## 安装 / Install

- 从本页面下载对应版本的 VSIX 安装包
- 在 VS Code 中执行 `Extensions: Install from VSIX...`
- 如果当前窗口里已经打开了旧版 Shelf AI，升级后执行一次 `Developer: Reload Window`
