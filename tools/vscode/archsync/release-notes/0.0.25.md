# ArchSync 0.0.25

## 中文说明

- 这一版把框架边界和项目实例配置真正连了起来：当边界已经在实例层开放配置时，`SURFACE`、`VISUAL`、`ROUTE`、`A11Y`、`LIBRARY`、`PREVIEW`、`CHAT`、`CONTEXT`、`RETURN` 这类边界标识现在可以直接跳到对应 `instance.toml` section。
- 边界悬浮卡片也补上了配置追溯信息：会显示当前实例配置文件路径，以及该边界的主配置 section，方便从框架文档反查实例化落点。
- 这次发布延续了前一版的模块跳转与悬浮能力，因此现在 ArchSync 同时支持“模块/规则跳转”和“边界到实例配置跳转”两条导航链路。

## English Notes

- This release adds boundary-to-instance navigation for instantiated framework boundaries. When a boundary is exposed as project configuration, symbols such as `SURFACE`, `VISUAL`, `ROUTE`, `A11Y`, `LIBRARY`, `PREVIEW`, `CHAT`, `CONTEXT`, and `RETURN` now jump directly to the matching `instance.toml` section.
- Boundary hover cards now include configuration traceability: the active project config file path and the owning section for that boundary are shown in the hover.
- This release keeps the existing module and rule navigation improvements, so ArchSync now supports both module/rule jumps and boundary-to-instance configuration jumps in one flow.

## 安装 / Install

- 从本页面下载 `archsync-0.0.25.vsix`
- 在 VS Code 中执行 `Extensions: Install from VSIX...`
- 安装后如未立即生效，执行一次 `Developer: Reload Window`
