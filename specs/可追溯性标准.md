# 可追溯性标准

## 1. 目标
确保任何结论都能追溯到其来源规则、来源模块、来源边界与来源能力声明；若记录了设计意图，也应可回溯到对应意图。

## 2. 强制要求
- 每个步骤必须有唯一 `step_id`
- 每个步骤必须声明 `depends_on`
- 每个结论步骤必须有 `evidence`
- 禁止无依赖来源的孤儿结论

## 3. 文档追溯
- 标准层：`L0/L1/L2/L3` 关系必须在 `specs/规范总纲与树形结构.md` 显式声明
- 映射层：`mapping/mapping_registry.json` 必须包含从标准锚点到实现符号的对应关系

## 4. 代码追溯
- 关键概念（Capability/Boundary/Base/Rule/Verification）必须能在实现中定位到符号
- 若领域实现以 `Module`、`Component` 等术语承接 `Base`，必须在 `mapping/mapping_registry.json` 中显式声明语义对应关系
- 变更后必须运行映射验证脚本，确认追溯链未断裂
