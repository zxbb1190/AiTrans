# 规范总纲与树形结构（L0）

## 1. 目的
本文件是仓库规范总纲，定义标准树、模块分层目录约束和变更传导规则。

## 2. 标准树（强制）

标准文件目录采用“职责分离”：
- `specs/`：承载 L0/L1 抽象规范
- `framework/<module>/Lx/`：承载领域模块分层标准
- `mapping/`：承载机器可读映射注册（L3）

```text
specs/
├── 规范总纲与树形结构.md        # L0
├── 框架设计核心标准.md          # L1
├── 可追溯性标准.md              # L1
└── 可删减性标准.md              # L1

framework/
├── shelf/
│   └── L2/置物架框架标准.md
├── curtain/
│   └── L2/窗帘框架标准.md
└── frontend/
    ├── L0/工程与运行底座层.md
    ├── L1/设计令牌与主题层.md
    ├── L2/前端框架标准.md
    ├── L3/组件原子层.md
    ├── L4/状态与数据编排层.md
    ├── L5/页面与路由编排层.md
    ├── L6/业务场景与流程层.md
    └── L7/体验目标与验证层.md

mapping/
└── mapping_registry.json        # L3
```

仓库级框架标准树（L0-L3）如下：

```text
L0 规范总纲
└── specs/规范总纲与树形结构.md
    └── L1 核心标准层
        ├── specs/框架设计核心标准.md
        ├── specs/可追溯性标准.md
        ├── specs/可删减性标准.md
        └── L2 领域标准层
            ├── framework/shelf/L2/置物架框架标准.md
            ├── framework/curtain/L2/窗帘框架标准.md
            ├── framework/frontend/L2/前端框架标准.md
            └── L3 映射注册层
                └── mapping/mapping_registry.json
```

## 3. 模块分层规则（强制）

- 层级编号 `X` 必须由目录 `Lx/` 决定，文件名前缀不参与层级判定。
- 模块名 `M` 必须由 `framework/<module>/` 目录名决定。
- 模块基础节点必须可规范化表示为：`L{X}.M{module}.B{n}`。
- 组合边仅允许：`Lx.M{module}.B* -> L(x+1).M{module}.B*`（下游到上游）。
- 禁止同层组合：`Lx -> Lx`。
- 禁止跨层跳级组合：`Lx -> L(x+2)`、`L(x+1) -> Lx` 等反向/跳级关系。
- 禁止跨模块组合（默认）：`Malpha -> Mbeta`。

## 4. 树化映射规则（强制）

- `mapping/mapping_registry.json` 的 `tree` 字段仅用于表达 L0-L3 框架标准树。
- 所有“标准 -> 实现”的一一对应关系，必须挂载在 `mapping/mapping_registry.json` 的 `mappings` 字段。
- `tree` 必须具备稳定 `id`，并使用 `children` 维护父子关系。
- `tree` 中每个 `kind=file` 节点必须指向唯一文件，禁止重复挂载同一文件。
- `tree` 禁止挂载工程执行文件或实现证据文件（`AGENTS.md`、`src/*`、`docs/*` 等）。
- `L0/L1` 文件必须位于 `specs/`。
- `L2` 文件必须位于 `framework/<module>/L2/`。
- `L3` 注册文件必须位于 `mapping/`。

## 5. 变更传导规则（强制）

- 当 L0 变更时，L1/L2/L3 必须同步评估并按需更新。
- 当 L1 变更时，L2/L3 必须同步评估并按需更新。
- 当 L2 变更时，L3 必须同步更新。
- 当 L3 变更时，必须执行反向验证，确认不违反 L0/L1/L2。
- 模块层级文档变更时，必须通过模块分层校验。

## 6. 验证机制

机器可读映射：`mapping/mapping_registry.json`

验证命令：
```bash
uv run python scripts/validate_strict_mapping.py
uv run python scripts/validate_strict_mapping.py --check-changes
```

## 7. 评审准入门槛

合并前至少满足：
- 标准树关系未被破坏
- 映射验证通过
- 模块分层与组合关系验证通过
- 变更方向满足传导规则
