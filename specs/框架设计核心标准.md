# 框架设计标准（核心）

## 0. 定位
本文件定义框架设计核心方法，不包含工程执行细节。

核心范围仅包含五项：
1. 目标
2. 边界定义
3. 模块（最小可行基）
4. 组合原则
5. 验证

补充定位：
- 框架设计必须从 `L0` 抽象结构起步，再逐层展开到高层语义。
- `L0` 只定义结构能力与组合空间，不引入具体实例值。

## 1. 目标（Goal）
定义框架优化对象与结果。

要求：
- 目标必须可度量
- 目标必须可验证
- 目标必须与边界建立约束关系

## 2. 边界定义（Boundary）
定义框架适用范围与限制条件。

要求：
- 每个边界项都可测量
- 每个边界项都可校验
- 边界项需与目标直接相关
- 边界用于约束结构展开空间，不用于定义具体实例值

举例说明：
- 可以抽象 `Button` 结构能力，不应在基定义中固化“红色按钮”这类实例。
- 颜色、尺寸、主题等实例差异应通过边界参数表达。

## 3. 最小可行基（Bases）
定义最小可行结构单元（Minimal Viable Base）。

要求：
- 模块是文件级单元：一个 `framework/<domain>/Lx-Mn-*.md` 文件对应一个模块。
- 当同层存在多个模块文件时，文件名必须使用 `Lx-Mn-*.md` 表示模块编号（例如 `L1-M0-...md`）。
- 模块别名（若需要）是文件级信息，不得绑定到单个 `B*`。
- 每个基必须具备 `B{n}` 标识
- 每个基必须能映射为 `L{X}.M{m}.B{n}` 的模块内结构单元
- 基是结构单元，不是模块单元
- `B*` 必须是抽象、可复用、跨实例稳定的结构单元
- 最小可行基应尽量收敛自由度，在满足当前层生成要求的前提下，选择自由度最少且最合适的结构
- 功能必须由结构导出，不得把功能清单直接当作基定义
- `B*` 格式应为 `B* 名称：<结构定义或 Lx.My[...]>。来源：\`...\``；若引用上游模块，必须直接将模块引用内联写在主句中，不得追加 `上游模块：...` 字段，基级细节由组合原则约束。
- `L0` 作为根层，其 `B*` 必须自足，不得再引用其他模块。

## 4. 组合原则（Combination Principles）
定义模块如何形成有效系统。

要求：
- 组合规则必须可表达为显式条件
- 规则间不可互相冲突
- 组合结果必须可判定“有效/无效”
- 组合只允许发生在相邻层：`Lx -> L(x+1)`（下游向上游）
- 禁止同层组合与跨层跳级组合
- 允许同一下层节点被多个相邻上层节点复用，但每个非叶子节点必须至少由一个相邻下层节点组成
- 每一层都可以定义本层新的基础结构，但对下层的引用必须保持相邻可见，禁止跳级引用
- 从树结构看，基础结构落在叶子节点，其它节点都由下层节点逐层组成

## 5. 验证（Verification）
定义如何判断框架结论是否成立。

要求：
- 必须有基线（baseline）
- 必须有目标值（target）
- 必须有通过条件（pass criteria）
- 必须有失败原因（failure reasons）
- 必须可自动验证层级与组合合法性
- 必须可区分“结构定义问题”与“实例配置问题”

## 6. 外部关联
本文件的外部关联由顶层规范树定义：
- `specs/规范总纲与树形结构.md`
- 层级传导方向：`L0 -> L1 -> L2 -> L3`
- 模块框架文件命名约束：`framework/<module>/Lx-Mn-*.md`（层级由 `Lx-` 前缀判定）
- L0-L3 树节点仅承载框架标准文件；标准到实现的对应关系在 `mapping/mapping_registry.json` 的 `mappings` 中维护。

## 7. Python 实现质量（静态类型）
定义 Python 代码的最低工程质量门槛。

要求：
- Python 代码必须通过静态类型检查。
- 静态类型检查命令统一为：`uv run mypy`。
- 未通过静态类型检查时，不得进入推送流程。
