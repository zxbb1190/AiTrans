# 发布与版本说明标准

## 1. 目标

本标准定义仓库内公开发布物的版本说明要求，确保发布记录可读、可安装、可追溯，而不是只保留自动生成的 compare 链接。

## 2. 适用范围

- GitHub Release
- 可安装产物发布，例如 VSCode `.vsix`
- 其它面向外部使用者的版本化交付物

## 3. 强制要求

- 每次公开发布必须附带人工编写的版本说明，不得只依赖自动生成说明。
- 版本说明必须同时包含中文和英文两部分。
- 版本说明必须与发布版本一一对应，按版本落库保存。
- 发布页必须附带可直接安装或直接使用的正式产物；只有源码压缩包不算完成发布。
- 若本次发布主要是流程修复、包装修复或文档修复，也必须明确写清楚，不得伪装成功能发布。

## 4. 版本说明最低结构

每份版本说明至少应包含以下内容：

- `## 中文说明`
- `## English Notes`
- 本次变更摘要
- 对使用者可见的影响
- 安装或获取方式（若发布产物需要手动安装）

若存在迁移、兼容性变化、已知限制或回滚提示，应额外写明。

## 5. Shelf AI 约束

对于 `tools/vscode/shelf-ai`：

- 每个公开版本必须提供 `tools/vscode/shelf-ai/release-notes/<version>.md`
- GitHub Release 正文必须直接使用该文件内容
- 发布资产中必须包含对应版本的 `.vsix`
- 若缺少 release notes 文件、缺少双语段落或缺少 `.vsix`，则发布流程必须失败

## 6. 执行原则

- 发布说明属于交付物本身的一部分，不是可选附属品。
- 能被用户下载到的版本，必须能从发布页直接看懂“这版改了什么、怎么安装、是否有注意事项”。
