# knowledge_base_basic 框架严格映射基线

## 1. 目的

本文件把以下两份框架标准落实到 `knowledge_base_basic` 的当前实现：

- [前端框架标准模块](../../framework/frontend/L2-M0-前端框架标准模块.md)
- [知识库工作台场景模块](../../framework/knowledge_base/L2-M0-知识库工作台场景模块.md)

目标不是口头对应，而是把实例化后的真实落点写清楚：

1. `framework -> code`
2. `framework -> config`
3. `framework -> validation`

当前实现已经形成完整编译链：

`framework markdown -> framework IR -> instance.toml -> frontend/workbench contract -> rule validation -> generated project bundle`

关键落点：
- IR 解析：[src/framework_ir/](../../src/framework_ir/)
- 实例编译：[knowledge_base.py](../../src/project_runtime/knowledge_base.py)
- 前端通用契约与规则：[src/frontend_kernel/](../../src/frontend_kernel/)
- 知识库工作台契约与规则：[src/knowledge_base_framework/](../../src/knowledge_base_framework/)
- 实例配置：[instance.toml](./instance.toml)
- 生成产物：`generated/framework_ir.json`、`generated/workbench_spec.json`、`generated/project_bundle.py`、`generated/generation_manifest.json`

状态约定：
- `已落地`：当前实现已有稳定代码和验证落点
- `部分落地`：当前已落代码，但仍有可继续抽象或扩展的空间
- `缺口`：当前没有稳定代码或验证落点

约定：
- 框架边界保持大类建模
- `instance.toml` 以同名大类 `section` 承载实例值
- section 内字段为该边界下的小类变化项
- 跨边界联动不通过字段归属解决，而通过 `R*` 校验解决

| 边界 | 当前配置落点 | 状态 | 说明 |

| `SURFACE` | `[surface]`、`[surface.copy]` | 已落地 | shell、layout、preview_mode、density 与 surface 文案都归 `surface` 大类 |
| `INTERACT` | `[library.copy].search_placeholder`、`[chat.copy].placeholder`、`[chat].composer_variant`、`[context].selection_mode` | 已落地 | 交互入口与输入方式由小类字段承载，但都挂到明确大类 section |
| `STATE` | `[context].sticky_document`、`[preview].show_toc`、`[return].anchor_restore` | 已落地 | 当前文档、当前章节、TOC 与 return anchor 状态都来自边界内实例值 |
| `EXTEND` | `[framework].frontend/domain/backend` | 已落地 | 扩展槽位绑定到显式 framework refs |

## 5. 编译产物映射

| 产物 | 来源 | 用途 |
|---|---|---|
| `generated/framework_ir.json` | `framework markdown -> framework IR` | 冻结被选中的 primary modules 与闭包模块 |
| `generated/workbench_spec.json` | `IR + instance.toml + contracts + validation` | 给页面、API、调试和追溯使用的统一 spec |
| `generated/project_bundle.py` | `workbench_spec.json` | 生成后的 Python 侧导入入口 |
| `generated/generation_manifest.json` | `framework + instance + generator output hashes` | 证明当前产物只能由框架和实例配置重新生成 |

## 6. 结论

`knowledge_base_basic` 已经不再是“文章工作台 demo”。

当前版本已经具备以下严格映射能力：
- 框架文档被解析为可追溯 IR
- `instance.toml` 只负责边界内允许变化的值
- `projects/<project_id>/generated/*` 只允许由生成器物化，不允许手改
- 前端通用层和知识库工作台层都产出显式 contract
- `R1-R4` 在启动期变成硬校验
- 编译产物会把 IR、实例值、contract 和验证结果一起写出

还可继续增强的点：
- `A11Y` 可以进一步下放更多实例化参数
- backend `L2-M0` 还可以继续向统一结果壳体和写入治理收束
- 后续可在此基础上继续扩展 `framework IR -> code generator` 模板族
