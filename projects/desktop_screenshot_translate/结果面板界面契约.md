# AiTrans 结果面板界面契约

## 1. 文档职责

本文件位于 `Framework -> Product Spec -> Implementation Config -> Code` 分层中的项目级 UI 契约层。

它只回答一件事：`AiTrans` 的结果面板在当前产品实例里应该如何交付结果、组织信息层级、约束状态切换与动作反馈。

它不替代：

- `framework/aitrans/L2-M3-结果展示层.md` 的领域共性语义
- `framework/frontend/*.md` 的通用前端结构语言
- `implementation_config.toml` 的技术选型与运行时细节

它的作用是把“紧凑双栏、低打扰悬浮交付”编译成当前项目可直接实现的前端契约。

## 2. 面板目标

- 面板必须是“一次截图会话 -> 一次结果交付”的悬浮结果面，不应退化成常驻主页或调试控制台。
- 面板首屏必须优先服务“看懂结果并立即操作”，而不是优先展示品牌、阶段装饰或内部诊断。
- 面板必须满足 `floating_compact_dual_column`：紧凑、悬浮、双栏正文优先、操作闭环明确。
- 面板必须满足“低打扰”：可见、可控、可拖动，但不通过大面积装饰抢夺注意力。

## 3. 首屏信息优先级

### P0: 必须首屏可见

- 当前状态
- 原文
- 译文
- 关闭
- 复制译文
- 粘贴原文
- 重新截图
- 重试翻译

### P1: 可首屏轻量出现

- 源语言
- 目标语言
- 失败原因摘要

### P2: 默认折叠或降级到二级详情

- 截图预览
- Provider 名称
- OCR / 翻译诊断信息
- `capture_session_id`
- `task_id`
- 内部执行路径和调试说明

## 4. 布局契约

### 4.1 首屏结构

- 顶部状态栏：状态、语言摘要、关闭动作。
- 中部正文区：原文列 + 译文列。
- 原文列允许直接编辑；编辑后的原文应作为新的翻译输入。
- 底部动作栏：复制、重试翻译、重新截图。

### 4.2 结构限制

- 不允许在首屏正文之前放置大面积品牌 Hero。
- 不允许把阶段条、诊断卡、执行信息卡放在原文和译文之前。
- 不允许把截图预览作为首屏主视觉。
- 不允许把错误详情铺满整个面板并挤压掉原文、译文与动作区。

### 4.3 详情区

- 详情区默认折叠。
- 详情区用于承接截图预览、Provider、诊断信息与会话标识。
- 详情区展开后不得破坏正文区和动作区的稳定位置。

## 5. 状态到视图映射

- `idle`：显示空态提示，不显示诊断细节。
- `capturing`：显示轻量状态，不提前展开大面积骨架屏。
- `ocr_processing`：保留正文区位置稳定，用处理中内容替换正文。
- `translation_processing`：保留原文，译文区显示处理中提示。
- `translation_ready`：原文、译文和动作区完整可用。
- `failed`：保留原文和动作区，失败信息以摘要形式出现；详细错误进入二级详情。
- `empty`：显示“未识别到可翻译文本”，动作区仍保留关闭与重新截图。
- `closed`：由显隐控制层处理，不在面板正文中模拟。

## 6. 动作契约

- `copy_translation`：仅在存在可复制译文且非忙碌态时可执行。
- `edit_source_text`：原文区应允许直接编辑；编辑不得破坏当前会话与详情区状态。
- `paste_source_text`：允许从剪贴板粘贴文本到原文区；粘贴后应能直接进入翻译闭环。
- `retry_translation`：仅在已有原文时可执行；不得在忙碌态重复触发。
- `recapture`：除明确禁用状态外应始终可见。
- `close_panel`：始终可见。

动作区要求：

- 动作反馈必须轻量、即时、就地。
- 动作不得把主界面切换成调试页面。
- 动作执行后不应把详情区自动展开到主视野。

## 7. 悬浮行为契约

- 面板初始位置应遵循“靠近截图结果、避免遮挡、保持最小完整显示”原则。
- 面板允许拖动，但拖动不应改变正文区结构。
- `auto_hide = manual_only`，因此面板不应做定时自动消失。
- 当存在外部焦点恢复目标时，面板应可见但不抢焦点。

## 8. 可访问契约

- 阅读顺序应优先：状态与动作 -> 原文 -> 译文 -> 详情区。
- 键盘主路径必须覆盖：复制、粘贴原文、重试、关闭、重新截图。
- 原文编辑后的重新翻译应具备轻量、就地的反馈，不得要求跳转到其它页面。
- `capture started / capture cancelled / translation ready / translation failed` 四类状态应具备可感知播报出口。

## 9. 明确禁止

- 结果面板被托盘直接当成无会话首页打开。
- 品牌区在视觉上大于正文区。
- 诊断信息默认常开。
- 首屏先看见截图预览，后看见译文。
- 交互动作被挤到正文与诊断信息之后。
