## 中文说明

本次发布把 `knowledge_base_basic` 明确收敛成 Shelf 当前默认主线样例：一个由框架文档、产品规格和实现配置共同编译出来的知识库工作台，而不是散落的 demo 文件集合。

本次变更摘要：

- 固化了知识库主链的单一编译入口：`project_runtime.knowledge_base.materialize_knowledge_base_project`
- 清理并禁止了旧时代残留生成物 `project_bundle.py` / `workbench_spec.json`
- 将知识库链路对齐说明单独写成文档，方便直接审查“框架 -> 配置 -> 代码 -> 证据”的闭环
- 保持默认运行入口为知识库工作台，同时把 legacy 置物架样本继续下沉为非默认路径

对使用者可见的影响：

- `projects/knowledge_base_basic/generated/` 现在只保留 canonical 生成物集合
- 默认运行命令继续是 `uv run python src/main.py`
- 知识库页面、API、引用回路和治理证据仍然由同一套上游输入共同导出

安装或获取方式：

1. 获取本目录下的 `knowledge-base-basic-0.3.0.tar.gz`
2. 解压后进入目录
3. 执行 `uv sync`
4. 执行 `uv run python src/main.py serve --product-spec-file projects/knowledge_base_basic/product_spec.toml`
5. 浏览器访问 `http://127.0.0.1:8000/knowledge-base`

注意事项：

- 该发布物当前是仓库内版本化交付物，不等于 GitHub Release 页面已同步创建。
- 若你修改了框架文档、`product_spec.toml` 或 `implementation_config.toml`，必须重新执行 `uv run python scripts/materialize_project.py`。

## English Notes

This release turns `knowledge_base_basic` into Shelf's explicit default mainline sample: a knowledge-base workbench compiled from framework markdown, product spec, and implementation config, rather than a loose collection of demo files.

Summary of this release:

- Keeps a single compile entrypoint for the knowledge-base sample: `project_runtime.knowledge_base.materialize_knowledge_base_project`
- Removes and forbids stale legacy artifacts: `project_bundle.py` and `workbench_spec.json`
- Adds a dedicated chain-alignment document so the `Framework -> Config -> Code -> Evidence` path can be audited directly
- Keeps the knowledge-base workbench as the default runtime path while preserving the legacy shelf sample as a non-default route

User-visible impact:

- `projects/knowledge_base_basic/generated/` now contains only the canonical generated artifact set
- The default run command remains `uv run python src/main.py`
- Knowledge-base pages, APIs, citation loop, and governance evidence still derive from one shared upstream input set

Install / access:

1. Download `knowledge-base-basic-0.3.0.tar.gz` from this directory
2. Extract the archive and enter the folder
3. Run `uv sync`
4. Run `uv run python src/main.py serve --product-spec-file projects/knowledge_base_basic/product_spec.toml`
5. Open `http://127.0.0.1:8000/knowledge-base`

Notes:

- This is currently a repository-side versioned delivery, not a hosted GitHub Release page.
- If you change framework markdown, `product_spec.toml`, or `implementation_config.toml`, you must rerun `uv run python scripts/materialize_project.py`.
