# 知识库目录预览对话编排模块:KnowledgeLibraryPreviewConversationOrchestration

@framework

## 1. 能力声明（Capability Statement）

- `C1` 文件库接口编排能力：把文件列表、上传、移除与引用域选择编排为稳定接口集合。
- `C2` 预览接口编排能力：把文档预览、目录读取与章节锚点读取编排为稳定接口集合。
- `C3` 对话引用接口编排能力：把对话请求、回答输出与引用返回编排为稳定接口集合。
- `C4` 非能力项：不负责前端布局、索引实现与模型推理过程本身。

## 2. 边界定义（Boundary / 参数）

- `LIBAPI` 文件库接口边界：文件列表、上传、移除与文件聚焦接口需统一。来源：`C1 + C3`。
- `PREVIEWAPI` 预览接口边界：正文、目录与章节锚点读取接口需统一。来源：`C1 + C2`。
- `CHATAPI` 对话接口边界：输入、输出、引用域与引用条目返回需统一。来源：`C2 + C3`。
- `CONSIST` 一致性边界：文件、预览与对话返回结构需共享统一外层约束。来源：`C1 + C2 + C3`。
- `AUTH` 权限边界：上传、移除与敏感接口需显式声明权限或替代策略。来源：`C1 + C3`。
- `TRACE` 追踪边界：请求参数、错误码与引用来源需可追踪。来源：`C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 文件库接口编排基：L0.M0[R1,R3]。来源：`C1 + LIBAPI + CONSIST + AUTH`。
- `B2` 预览接口编排基：L0.M0[R1,R2]。来源：`C2 + PREVIEWAPI + CONSIST + TRACE`。
- `B3` 对话引用接口编排基：L0.M0[R2,R3]。来源：`C3 + CHATAPI + TRACE + AUTH`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 文件预览主链
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先对齐文件库接口与文档预览接口的字段与外层结构，再接入对话接口。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`LIBAPI + PREVIEWAPI + CONSIST + TRACE`。
- `R2` 引用对话并轨
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：将文档预览、章节锚点与对话引用返回对齐，形成可回看来源的接口面。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`PREVIEWAPI + CHATAPI + TRACE + CONSIST`。
- `R3` 接口集合闭环
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：联合文件库、预览与对话接口编排结构，形成统一知识库接口集合。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`LIBAPI + PREVIEWAPI + CHATAPI + CONSIST + AUTH + TRACE`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：文件、预览与对话接口返回结构不一致，或引用来源无法追踪的方案均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`LIBAPI + PREVIEWAPI + CHATAPI + CONSIST + AUTH + TRACE`。

## 5. 验证（Verification）

- `V1` 文件库覆盖性：文件列表、上传与移除接口必须全部存在。
- `V2` 预览覆盖性：正文、目录与章节锚点接口必须全部存在且结构稳定。
- `V3` 对话覆盖性：对话输入、回答输出与引用返回必须满足 `CHATAPI + TRACE` 约束。
- `V4` 一致性：所有接口必须共享统一 `CONSIST` 外层结构。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 边界绑定` 的通过或失败结论。
