# 知识库文件资源与对话契约原子模块:KnowledgeFileResourceConversationContracts

@framework

## 1. 能力声明（Capability Statement）

- `C1` 文件资源抽象能力：把知识库文件、文档预览与元信息抽象为稳定资源结构。
- `C2` 对话引用契约能力：把对话输入、回答输出与引用条目抽象为稳定契约结构。
- `C3` 约束外化能力：把字段校验、数量限制与错误表达外化为稳定边界。
- `C4` 非能力项：不负责前端页面编排、向量索引策略与模型推理实现。

## 2. 边界定义（Boundary / 参数）

- `FILE` 文件边界：文件标识、文件名、类型、标签与更新时间需稳定。来源：`C1 + C3`。
- `PREVIEW` 预览边界：正文、目录、摘要与章节锚点字段需完整。来源：`C1 + C2`。
- `CHAT` 对话边界：输入消息、输出消息与回合标识需稳定。来源：`C2 + C3`。
- `CITATION` 引用边界：引用文件、章节锚点、片段与说明字段需稳定。来源：`C2 + C3`。
- `VALID` 校验边界：数量、长度、后缀与必填约束需明确。来源：`C1 + C3`。
- `ERROR` 错误边界：错误码、错误原因与字段级失败需结构化表达。来源：`C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 文件资源结构基：由文件标识、文件元信息与目录字段构成的资源结构。来源：`C1 + FILE + VALID`。
- `B2` 文档预览契约基：由正文、目录、摘要与章节锚点字段构成的预览结构。来源：`PREVIEW + FILE`。
- `B3` 对话引用契约基：由对话输入、回答输出与引用条目字段构成的契约结构。来源：`C2 + C3 + CHAT + CITATION + ERROR`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 文件资源先行
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先对齐文件资源与文档预览字段，再允许更上层接口编排消费。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`FILE + PREVIEW + VALID`。
- `R2` 对话引用并轨
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：把文档预览结构与对话引用契约对齐，形成可回看来源的输出结构。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`CHAT + CITATION + PREVIEW + ERROR`。
- `R3` 契约治理闭环
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：在文件资源、预览结构与对话引用契约上叠加校验与错误表达，形成稳定后端原子契约。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`FILE + PREVIEW + CHAT + CITATION + VALID + ERROR`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：字段定义不稳定、预览与引用无法对齐或错误不可结构化的方案均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`FILE + PREVIEW + CHAT + CITATION + VALID + ERROR`。

## 5. 验证（Verification）

- `V1` 资源完整性：文件资源字段必须满足 `FILE + VALID` 约束。
- `V2` 预览一致性：文档预览契约必须满足 `PREVIEW` 约束并能对齐文件资源。
- `V3` 引用有效性：对话与引用契约必须满足 `CHAT + CITATION + ERROR` 约束。
- `V4` 最小必要性：移除任一 `B*` 后，`V1~V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* + 边界绑定` 的通过或失败结论。
