# 置物架装配编排模块:ShelfAssemblyOrchestrationModule

@framework

## 1. 能力声明（Capability Statement）

- `C1` 装配流程编排能力：将抽象结构转化为可执行装配步骤与检查点。
- `C2` 空间单元编排能力：将分格单元组织为可落地层板与访问通道结构。
- `C3` 运维治理能力：支持复核、回退与场景重配。
- `C4` 非能力项：不负责具体摆放策略、业务规则和视觉装饰配置。

## 2. 边界定义（Boundary / 参数）

- `STEP` 工序边界：装配顺序、依赖关系与完成条件需明确。来源：`C1 + C3`。
- `ALIGN` 对齐边界：立向与横向偏差阈值需可量化。来源：`C1 + C2`。
- `TOL` 公差边界：连接与锁定公差范围需可执行。来源：`C1 + C3`。
- `ACCESS` 访问边界：单元访问路径和操作空间需满足约束。来源：`C2 + C3`。
- `RECOVER` 回退边界：装配失败或场景切换需具备回退策略。来源：`C3`。
- `SCENE` 场景边界：不同场景下装配策略差异需可追踪。来源：`C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 装配骨架编排基：L0.M0[R1]。来源：`C1 + STEP + ALIGN + TOL`。
- `B2` 空间单元编排基：L0.M0[R2]。来源：`C2 + ACCESS + SCENE + ALIGN`。
- `B3` 回退治理编排基：L0.M0[R3]。来源：`C3 + RECOVER + STEP + SCENE`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 工序主链组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立装配工序主链，再绑定对齐和公差复核点。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`STEP + ALIGN + TOL`。
- `R2` 单元并轨组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在装配主链上并入空间单元，形成可访问的结构面。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`ACCESS + ALIGN + SCENE + STEP`。
- `R3` 回退闭环组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将失败分支与回退链绑定到装配流程，形成运维闭环。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`RECOVER + SCENE + STEP + TOL + ACCESS`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：跳过装配主链、缺失访问约束或无回退链的组合无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`STEP + ALIGN + TOL + ACCESS + RECOVER + SCENE`。

## 5. 验证（Verification）

- `V1` 工序一致性：装配流程必须满足 `STEP + ALIGN + TOL` 约束。
- `V2` 访问有效性：单元编排必须满足 `ACCESS` 路径约束。
- `V3` 回退有效性：回退链必须覆盖关键失败分支。
- `V4` 场景适配性：场景切换后结构仍满足 `SCENE` 约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 边界绑定` 的通过或失败结论。
