# 前端标记与反馈原子模块:FrontendMarkerFeedbackAtoms

@framework

本模块定义前端最小标记与反馈原子。它承接标签、状态标记、消息气泡和空态提示这类“附着在主体结构上的反馈表达”，不再把装配元结构混入这一层。

## 1. 能力声明（Capability Statement）

- `C1` 状态标记能力：把标签、状态标记与摘要提示收敛为稳定标记结构。
- `C2` 反馈提示能力：把消息气泡、失败提示与空态说明收敛为稳定反馈结构。
- `C3` 可达反馈能力：把朗读关系、提示顺序与动作说明收敛为稳定可访问反馈结构。
- `C4` 非能力项：不负责文本编辑、集合导航与页面路由编排。

## 2. 边界定义（Boundary / 参数）

- `TAG` 标记边界：标签、状态芯片与摘要标记位需稳定。来源：`C1 + C3`。
- `BUBBLE` 气泡边界：消息气泡、提示文本与动作位需稳定。来源：`C2 + C3`。
- `EMPTY` 空态边界：空列表、空面板与无结果占位需稳定。来源：`C1 + C2`。
- `STATUS` 状态边界：成功、失败、处理中与禁用提示需稳定。来源：`C1 + C2`。
- `EVENT` 事件边界：展开、关闭、确认与撤销出口需稳定。来源：`C2 + C3`。
- `A11Y` 可访问边界：提示朗读、状态说明与动作说明需可达。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 状态标记结构基：L0.M0[R1,R2] + L0.M1[R2] + L0.M2[R1,R2]。来源：`C1 + TAG + STATUS + A11Y`。
- `B2` 反馈提示结构基：L0.M0[R1,R2] + L0.M1[R2] + L0.M2[R1,R2]。来源：`C2 + C3 + BUBBLE + EMPTY + EVENT + A11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 标记先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定标签、状态芯片与摘要提示的表达方式，再允许更高层结构挂接反馈动作。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`TAG + STATUS + A11Y`。
- `R2` 反馈闭合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：将气泡、空态说明与动作出口附着到稳定标记结构上，形成统一反馈原子。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`BUBBLE + EMPTY + EVENT + A11Y + TAG + STATUS`。
- `R3` 禁止组合
  - `R3.1` 参与基：`B1 + B2`。
  - `R3.2` 组合方式：反馈不附着稳定标记面、提示顺序不可解释或事件说明缺位的方案均无效。
  - `R3.3` 输出能力：`C4`。
  - `R3.4` 边界绑定：`TAG + BUBBLE + EMPTY + STATUS + EVENT + A11Y`。

## 5. 验证（Verification）

- `V1` 标记完整性：标签、状态芯片与摘要提示必须完整。
- `V2` 反馈完整性：气泡、空态说明与动作出口必须满足 `BUBBLE + EMPTY + EVENT` 约束。
- `V3` 可达完整性：朗读关系与提示说明必须满足 `A11Y` 约束。
- `V4` 最小必要性：移除任一 `B*` 后，`V2~V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* + 边界绑定` 的通过或失败结论。
