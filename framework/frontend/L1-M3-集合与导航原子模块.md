# 前端集合与导航原子模块:FrontendCollectionNavigationAtoms

@framework

本模块定义前端最小集合与导航原子。它把列表、树和节点导航放在同一模块中，因为这些对象共享“集合项 + 当前项 + 层级移动”的结构骨架。

## 1. 能力声明（Capability Statement）

- `C1` 集合层级承载能力：把列表、树与节点分组关系收敛为稳定集合结构。
- `C2` 节点导航能力：把当前选中节点、展开路径与跳转出口收敛为稳定导航结构。
- `C3` 焦点回路能力：把节点焦点、键盘移动与层级阅读关系收敛为稳定可达结构。
- `C4` 非能力项：不负责文本编辑、反馈提示与页面级场景流程。

## 2. 边界定义（Boundary / 参数）

- `LIST` 列表边界：列表顺序、分组关系与当前项需稳定。来源：`C1 + C3`。
- `TREE` 层级边界：层级深度、展开顺序与父子关系需稳定。来源：`C1 + C2`。
- `ITEM` 节点边界：节点外壳、节点摘要与动作位需稳定。来源：`C1 + C2`。
- `NAV` 导航边界：选中入口、展开出口、跳转出口与返回出口需稳定。来源：`C2 + C3`。
- `FOCUS` 焦点边界：节点焦点、列表焦点与层级阅读顺序需稳定。来源：`C2 + C3`。
- `A11Y` 可访问边界：键盘导航、层级播报与当前项提示需可达。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 集合层级结构基：L0.M0[R1,R2] + L0.M1[R2] + L0.M2[R1]。来源：`C1 + LIST + TREE + FOCUS`。
- `B2` 节点导航结构基：L0.M0[R2] + L0.M1[R1,R2] + L0.M2[R1]。来源：`C2 + C3 + ITEM + NAV + FOCUS + A11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 集合层级先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定列表分组、层级深度与节点顺序，再允许导航动作接入。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`LIST/TREE/FOCUS`。
- `R2` 节点导航闭合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：将当前选中、展开路径、跳转出口与层级提示并入集合结构，形成统一集合导航原子。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`ITEM/NAV/FOCUS/A11Y/LIST/TREE`。
- `R3` 禁止组合
  - `R3.1` 参与基：`B1 + B2`。
  - `R3.2` 组合方式：缺失父子关系、跳转出口不可解释或节点焦点无法回收的方案均无效。
  - `R3.3` 输出能力：`C4`。
  - `R3.4` 边界绑定：`LIST/TREE/ITEM/NAV/FOCUS/A11Y`。

## 5. 验证（Verification）

- `V1` 集合完整性：列表顺序、层级深度与当前项必须完整。
- `V2` 导航完整性：选中、展开与跳转出口必须满足 `NAV/ITEM` 约束。
- `V3` 可达完整性：键盘移动与层级播报必须满足 `FOCUS/A11Y` 约束。
- `V4` 最小必要性：移除任一 `B*` 后，`V2~V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
