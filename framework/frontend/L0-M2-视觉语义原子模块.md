# 前端视觉语义原子模块:FrontendVisualSemanticAtoms

@framework

本模块定义前端最底层的视觉语义结构。它只负责令牌、主题、密度和可读对比等视觉基础，不再把启动、交付和治理这类运行对象混进同一层。

## 1. 能力声明（Capability Statement）

- `C1` 语义令牌组织能力：把排版、色彩、间距与层级语义键收敛为稳定视觉结构。
- `C2` 主题配方映射能力：把主题集、状态色调与变体配方收敛为稳定主题结构。
- `C3` 可读约束承接能力：把密度、对比与可读性回退链收敛为稳定视觉约束结构。
- `C4` 非能力项：不负责宿主挂载、页面流程与实例品牌文案。

## 2. 边界定义（Boundary / 参数）

- `TOKEN` 令牌边界：排版、色彩、间距与层级语义键需稳定。来源：`C1 + C3`。
- `THEME` 主题边界：主题集、状态映射与局部覆盖回退链需稳定。来源：`C1 + C2`。
- `DENSITY` 密度边界：紧凑、常规与宽松密度档位需稳定。来源：`C1 + C3`。
- `TONE` 色调边界：主色调、辅助色调与状态色调关系需稳定。来源：`C2 + C3`。
- `A11Y` 可读边界：对比约束、可读字号与焦点可见提示需满足可访问要求。来源：`C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 语义令牌结构基：由排版、色彩、间距、层级语义键与密度档构成的视觉语义结构。来源：`C1 + TOKEN + DENSITY`。
- `B2` 主题配方结构基：由主题集、状态色调、局部覆盖回退链与可读约束构成的主题配方结构。来源：`C1 + C2 + C3 + THEME + TONE + A11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 令牌先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定令牌键和密度档，再允许更高层组件消费这些视觉语义。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`TOKEN/DENSITY`。
- `R2` 主题闭合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在令牌语义上叠加主题配方、状态色调与可读回退链，形成稳定视觉基础层。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`THEME/TONE/A11Y/TOKEN/DENSITY`。
- `R3` 禁止组合
  - `R3.1` 参与基：`B1 + B2`。
  - `R3.2` 组合方式：没有令牌语义就直接声明主题变体、色调关系不可解释或可读回退链缺位的方案均无效。
  - `R3.3` 输出能力：`C4`。
  - `R3.4` 边界绑定：`TOKEN/THEME/DENSITY/TONE/A11Y`。

## 5. 验证（Verification）

- `V1` 令牌完整性：排版、色彩、间距与层级语义键必须完整。
- `V2` 主题完整性：主题集、状态色调与变体配方必须满足 `THEME/TONE` 约束。
- `V3` 可读完整性：密度档与可读回退链必须满足 `DENSITY/A11Y` 约束。
- `V4` 最小必要性：移除任一 `B*` 后，`V2~V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
