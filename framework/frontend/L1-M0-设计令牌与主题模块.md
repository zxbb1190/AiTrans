# 设计令牌主题模块:DesignTokenTheme

@framework

## 1. 能力声明（Capability Statement）

- `C1` 语义抽象能力：将视觉语义抽象为稳定令牌，而非具体实例值。
- `C2` 主题映射能力：将语义令牌映射到多主题与多模式输出。
- `C3` 扩展治理能力：支持令牌演进、兼容和可访问性约束治理。
- `C4` 非能力项：不负责具体业务页面布局和场景流程编排。

## 2. 边界定义（Boundary / 参数）

- `TOKEN` 语义令牌集合：令牌必须按语义分层命名。来源：`C1 + C2`。
- `THEME` 主题集合：主题数量与命名规范需可治理。来源：`C2`。
- `MODE` 显示模式：亮色、暗色和高对比模式需可切换。来源：`C2 + C3`。
- `SCALE` 尺度体系：字号、间距、圆角和阴影尺度需统一。来源：`C1 + C2`。
- `SEM` 语义映射约束：状态语义到样式令牌映射必须稳定。来源：`C1 + C2`。
- `OVRD` 覆盖策略：局部覆盖优先级和回退路径必须明确。来源：`C2 + C3`。
- `A11Y` 可访问约束：对比度和可读性阈值必须达标。来源：`C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 语义令牌基：L0.M0[R1,R2,R3]。来源：`C1 + TOKEN + SCALE + SEM`。
- `B2` 主题映射基：L0.M0[R2]。来源：`C1 + C2 + THEME + MODE + SEM`。
- `B3` 覆盖约束基：L0.M0[R2,R3]。来源：`C2 + C3 + OVRD + A11Y + MODE`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 语义抽象组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立语义令牌层，再允许具体组件按语义消费。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`TOKEN/SCALE/SEM`。
- `R2` 主题映射组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在语义令牌基础上绑定主题与模式映射，保持语义不随实例变化。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`THEME/MODE/SEM/TOKEN`。
- `R3` 治理闭环组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：通过覆盖策略和可访问约束确保多主题输出稳定且可扩展。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`OVRD/A11Y/THEME/MODE/SEM`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：直接把实例样式写入基定义，或跳过语义层直接绑定主题值的方案均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`TOKEN/THEME/MODE/SCALE/SEM/OVRD/A11Y`。

## 5. 验证（Verification）

- `V1` 抽象完整性：令牌定义不得依赖具体实例页面取值。
- `V2` 映射一致性：同一语义在不同主题与模式下映射必须可解释。
- `V3` 治理有效性：覆盖策略必须存在明确优先级和回退路径。
- `V4` 约束符合性：输出结果必须满足 `A11Y` 与 `MODE` 约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
