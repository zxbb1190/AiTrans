# 前端框架模块标准（L5）页面与路由编排层
<!--@layer module=frontend; level=L5-->

## 1. 目标（Goal）

将状态编排能力提升为页面与路由流程能力，保证可导航、可观测、可回退。

## 2. 边界定义（Boundary）

- `PAGE_SCOPE`：页面容器职责边界。
- `ROUTE_SCOPE`：路由图与守卫边界。
- `NAV_SCOPE`：导航状态边界。

## 3. 最小可行基（Bases）

- `B1` 页面容器编排基  <!--@base id=B1; name=PAGE_ORCH_BASE; capability=page_container_orchestration-->
- `B2` 路由流程基  <!--@base id=B2; name=ROUTE_FLOW_BASE; capability=route_flow_control-->
- `B3` 导航策略基  <!--@base id=B3; name=NAV_POLICY_BASE; capability=navigation_policy-->

## 4. 组合原则（Combination Principles）

仅允许下游向上游组合：`L5 -> L6`。禁止同层组合与反向组合。

组合约束集合：
- `C1` 相邻层约束
- `C2` 下游上行约束
- `C3` 页面到业务流程契约

- 页面容器编排基组合到业务用例基。  <!--@compose from=L5.Mfrontend.B1; to=L6.Mfrontend.B1; rule=adjacent_only; principle=P_DOWNSTREAM_UPSTREAM; constraint=C1_C2_C3-->
- 页面容器编排基组合到领域契约基。  <!--@compose from=L5.Mfrontend.B1; to=L6.Mfrontend.B2; rule=adjacent_only; principle=P_DOWNSTREAM_UPSTREAM; constraint=C1_C2_C3-->
- 路由流程基组合到业务用例基。  <!--@compose from=L5.Mfrontend.B2; to=L6.Mfrontend.B1; rule=adjacent_only; principle=P_DOWNSTREAM_UPSTREAM; constraint=C1_C2_C3-->
- 导航策略基组合到领域契约基。  <!--@compose from=L5.Mfrontend.B3; to=L6.Mfrontend.B2; rule=adjacent_only; principle=P_DOWNSTREAM_UPSTREAM; constraint=C1_C2_C3-->

## 5. 验证（Verification）

- 每个 L5 基必须至少指向一个 L6 父节点。
- 每个 L6 基必须至少由一个 L5 子节点支撑。
- 不允许 `L5 -> L5` 或 `L5 -> L7`。
