# 前端触发与选择原子模块:FrontendTriggerSelectionAtoms

@framework

本模块定义前端离散动作入口原子。它承接按钮、开关入口、菜单触发和文件选择这类“触发一次、产生一次结果”的原子，不把文本编辑混进同一模块。

## 1. 能力声明（Capability Statement）

- `C1` 触发入口承载能力：把点击、确认、切换与即时动作入口收敛为稳定触发结构。
- `C2` 选择结果承接能力：把候选项、选择结果与文件收纳入口收敛为稳定选择结构。
- `C3` 绑定反馈承接能力：把动作出口、状态出口与可访问提示收敛为稳定框架接口。
- `C4` 非能力项：不负责连续文本编辑、长文阅读与页面级流程编排。

## 2. 边界定义（Boundary / 参数）

- `TRIGGER` 触发边界：点击入口、确认入口与二次触发出口需稳定。来源：`C1 + C3`。
- `PICK` 选择边界：文件选择、候选项选择与结果收纳位需稳定。来源：`C2 + C3`。
- `OPTION` 选项边界：选项摘要、当前选中项与禁用选项需稳定。来源：`C1 + C2`。
- `ACTION` 动作边界：触发出口、切换出口与回退出口需稳定。来源：`C1 + C3`。
- `STATUS` 状态边界：默认、处理中、成功与失败状态需稳定。来源：`C2 + C3`。
- `A11Y` 可访问边界：键盘触发、当前项提示与错误提示需可达。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 触发入口结构基：L0.M0[R2] + L0.M1[R1,R2] + L0.M2[R1,R2]。来源：`C1 + TRIGGER + ACTION + STATUS`。
- `B2` 选择承接结构基：L0.M0[R2] + L0.M1[R1,R2] + L0.M2[R1,R2]。来源：`C2 + C3 + PICK + OPTION + ACTION + A11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 触发入口先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定点击、确认与回退顺序，形成稳定离散动作入口。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`TRIGGER + ACTION + STATUS`。
- `R2` 选择并轨
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在离散动作入口上接入候选项、文件收纳与当前选中关系，形成统一触发与选择原子。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`PICK + OPTION + A11Y + ACTION + STATUS`。
- `R3` 禁止组合
  - `R3.1` 参与基：`B1 + B2`。
  - `R3.2` 组合方式：动作出口不可解释、选择结果不附着稳定入口或提示关系缺位的方案均无效。
  - `R3.3` 输出能力：`C4`。
  - `R3.4` 边界绑定：`TRIGGER + PICK + OPTION + ACTION + STATUS + A11Y`。

## 5. 验证（Verification）

- `V1` 触发完整性：点击入口、确认出口与回退出口必须完整。
- `V2` 选择完整性：候选项、当前选中项与结果收纳必须满足 `PICK + OPTION` 约束。
- `V3` 绑定有效性：动作出口与提示关系必须满足 `STATUS + A11Y` 约束。
- `V4` 最小必要性：移除任一 `B*` 后，`V2~V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* + 边界绑定` 的通过或失败结论。
