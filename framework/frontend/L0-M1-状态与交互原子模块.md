# 前端状态与交互原子模块:FrontendStateInteractionAtoms

@framework

本模块定义前端最底层的状态与交互结构。它把值流、动作出口和状态相位压到同一原子层，不再把“值通道”“动作通道”“状态集合”机械拆成多份平行基。

## 1. 能力声明（Capability Statement）

- `C1` 值流组织能力：把默认值、当前值、提交出口与复位出口收敛为稳定交互流结构。
- `C2` 动作承接能力：把触发动作、切换动作与回退动作收敛为稳定动作结构。
- `C3` 状态相位表达能力：把加载、成功、失败、禁用与反馈相位收敛为稳定状态结构。
- `C4` 非能力项：不负责具体组件外形、宿主元素与主题令牌取值。

## 2. 边界定义（Boundary / 参数）

- `VALUE` 值流边界：默认值、当前值与值变化顺序需稳定。来源：`C1 + C2`。
- `ACTION` 动作边界：触发出口、提交出口与切换出口需稳定。来源：`C1 + C2`。
- `STATE` 状态边界：默认、加载、成功、失败与禁用相位需稳定。来源：`C2 + C3`。
- `RESET` 回退边界：清空、撤销与回退落点需稳定。来源：`C1 + C3`。
- `FEEDBACK` 反馈边界：成功提示、失败提示与处理中反馈位需稳定。来源：`C2 + C3`。
- `TRANSFER` 传递边界：流入顺序、流出顺序与外部同步位需稳定。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 值流动作结构基：由默认值、当前值、触发出口、提交出口与复位出口构成的交互流结构。来源：`C1 + C2 + VALUE + ACTION + RESET + TRANSFER`。
- `B2` 状态相位反馈结构基：由状态集合、相位切换、反馈出口、禁用位与对外暴露位构成的状态结构。来源：`C2 + C3 + STATE + FEEDBACK + TRANSFER`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 值流先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定输入、触发、提交与回退顺序，再允许更高层组件附着状态语义。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`VALUE/ACTION/RESET/TRANSFER`。
- `R2` 状态闭合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在值流动作结构上叠加状态相位与反馈出口，形成稳定前端交互闭环。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`STATE/FEEDBACK/TRANSFER/VALUE/ACTION/RESET`。
- `R3` 禁止组合
  - `R3.1` 参与基：`B1 + B2`。
  - `R3.2` 组合方式：没有值流顺序就声明状态相位、反馈出口不可解释或回退链断裂的方案均无效。
  - `R3.3` 输出能力：`C4`。
  - `R3.4` 边界绑定：`VALUE/ACTION/STATE/RESET/FEEDBACK/TRANSFER`。

## 5. 验证（Verification）

- `V1` 值流完整性：默认值、当前值与提交出口必须完整。
- `V2` 状态完整性：状态集合与相位切换必须满足 `STATE/FEEDBACK` 约束。
- `V3` 回退有效性：复位与回退路径必须满足 `RESET/TRANSFER` 约束。
- `V4` 最小必要性：移除任一 `B*` 后，`V2~V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
