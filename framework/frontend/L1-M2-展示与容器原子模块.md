# 前端展示与容器原子模块:FrontendDisplayContainerAtoms

@framework

本模块定义前端最小展示与容器原子。它把文本展示、面板、抽屉和阅读视口放在同一结构族中，因为这些对象都属于“承载内容而非主动修改内容”的展示面。

## 1. 能力声明（Capability Statement）

- `C1` 文本展示能力：把正文、标题、说明与元信息承载收敛为稳定展示结构。
- `C2` 容器视口能力：把面板、抽屉、分区视口与空态承载收敛为稳定容器结构。
- `C3` 阅读承接能力：把展示内容、容器分区与阅读路径收敛为稳定阅读结构。
- `C4` 非能力项：不负责文本编辑、集合导航与页面级流程。

## 2. 边界定义（Boundary / 参数）

- `TEXT` 文本边界：标题、正文、说明与截断策略需稳定。来源：`C1 + C3`。
- `PANEL` 容器边界：面板外壳、抽屉轮廓与分区边界需稳定。来源：`C2 + C3`。
- `VIEWPORT` 视口边界：滚动区、分节顺序与当前阅读视口需稳定。来源：`C2 + C3`。
- `META` 元信息边界：标签、更新时间、来源说明与辅助信息需稳定。来源：`C1 + C2`。
- `EMPTY` 空态边界：无内容、无选择与加载占位结构需稳定。来源：`C2 + C3`。
- `A11Y` 可访问边界：阅读顺序、标题关系与说明提示需可达。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 文本展示结构基：L0.M0[R1,R2] + L0.M2[R1,R2]。来源：`C1 + TEXT + META + A11Y`。
- `B2` 容器视口结构基：L0.M0[R1,R2] + L0.M1[R2] + L0.M2[R1,R2]。来源：`C2 + C3 + PANEL + VIEWPORT + EMPTY + A11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 文本展示先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定标题、正文、说明与元信息的阅读顺序，再允许容器结构附着其上。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`TEXT + META + A11Y`。
- `R2` 容器阅读闭合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：将面板轮廓、视口分节与空态承载并入文本展示结构，形成统一阅读容器。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`PANEL + VIEWPORT + EMPTY + TEXT + META + A11Y`。
- `R3` 禁止组合
  - `R3.1` 参与基：`B1 + B2`。
  - `R3.2` 组合方式：缺失文本阅读顺序、容器分区不可解释或空态无法落位的方案均无效。
  - `R3.3` 输出能力：`C4`。
  - `R3.4` 边界绑定：`TEXT + PANEL + VIEWPORT + META + EMPTY + A11Y`。

## 5. 验证（Verification）

- `V1` 展示完整性：标题、正文与说明关系必须完整。
- `V2` 容器完整性：面板轮廓、视口分节与空态承载必须满足 `PANEL + VIEWPORT + EMPTY` 约束。
- `V3` 阅读有效性：阅读顺序与说明关系必须满足 `TEXT + A11Y` 约束。
- `V4` 最小必要性：移除任一 `B*` 后，`V2~V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* + 边界绑定` 的通过或失败结论。
