# 前端页面与路由编排模块:FrontendPageRouteOrchestration

@framework

## 1. 能力声明（Capability Statement）

- `C1` 页面骨架编排能力：把聊天首页、聊天会话页、知识库列表页、知识库详情页与文档详情页等页面骨架编排为稳定页面集。
- `C2` 路由治理能力：把入口、深链、返回与守卫收敛为稳定路由结构。
- `C3` 模板复用能力：让同类页面在统一骨架和状态编排下复用。
- `C4` 非能力项：不负责领域字段语义与后端权限判定细节。

## 2. 边界定义（Boundary / 参数）

- `LAYOUT` 布局边界：头部、侧栏、主体、浮层与反馈区槽位需固定。来源：`C1 + C3`。
- `ROUTE` 路由边界：聊天页、知识库页、文档详情页与引用深链路径命名需统一。来源：`C1 + C2`。
- `NAV` 导航边界：主入口、返回入口与上下文切换入口必须可达。来源：`C1 + C2`。
- `PAGESET` 页面集合边界：至少覆盖聊天首页、聊天会话页、知识库列表页、知识库详情页与文档详情页五类页面骨架。来源：`C1 + C3`。
- `GUARD` 守卫边界：写入入口、敏感入口与空状态入口需具备稳定守卫。来源：`C2 + C3`。
- `META` 元信息边界：标题、摘要、面包屑与分享信息生成规则需统一。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 页面骨架结构基：L3.M0[R1,R3]。来源：`C1 + LAYOUT + PAGESET + META`。
- `B2` 路由入口结构基：L3.M0[R1,R2]。来源：`C1 + C2 + ROUTE + NAV + GUARD`。
- `B3` 导航复用结构基：L3.M0[R2,R3]。来源：`C2 + C3 + NAV + PAGESET + META`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 页面骨架先行
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先固定页面骨架与路由入口，再允许具体场景挂载数据与交互；对于知识问答客户端，聊天工作页优先承载 chat 主舞台，知识库列表、知识库详情与文档详情页作为其附属页面能力。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`LAYOUT/ROUTE/NAV/GUARD`。
- `R2` 页面集复用
  - `R2.1` 参与基：`B1 + B3`。
  - `R2.2` 组合方式：在统一元信息与导航结构下复用不同页面模板，形成稳定页面集合。
  - `R2.3` 输出能力：`C1 + C3`。
  - `R2.4` 边界绑定：`PAGESET/META/NAV/LAYOUT`。
- `R3` 路由页面闭环
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：联合页面骨架、路由入口与导航复用结构，形成可深链、可返回的页面系统。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`ROUTE/NAV/PAGESET/GUARD/META`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：缺失引用深链入口、知识库详情页、文档详情页或页面骨架与路径错配、守卫缺位的方案均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`LAYOUT/ROUTE/NAV/PAGESET/GUARD/META`。

## 5. 验证（Verification）

- `V1` 路由一致性：页面路径和布局槽位必须满足 `ROUTE/LAYOUT` 约束。
- `V2` 页面覆盖性：`PAGESET` 必须覆盖聊天页、知识库列表/详情页、文档详情页与引用深链页面。
- `V3` 导航正确性：返回、深链与守卫入口必须符合 `NAV/GUARD` 规则。
- `V4` 元信息稳定性：页面元信息必须满足 `META` 约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
