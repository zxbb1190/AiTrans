# 前端框架模块标准（L3）组件原子层
<!--@layer module=frontend; level=L3-->

## 1. 目标（Goal）

将中枢能力沉淀为可复用 UI 组件体系，形成稳定交互语义。

## 2. 边界定义（Boundary）

- `COMPONENT_SCOPE`：原子与复合组件边界。
- `EVENT_SCOPE`：组件事件契约边界。
- `A11Y_SCOPE`：组件可访问能力边界。

## 3. 最小可行基（Bases）

- `B1` 组件原子基  <!--@base id=B1; name=ATOM_COMPONENT_BASE; capability=atomic_component_contract-->
- `B2` 复合组件基  <!--@base id=B2; name=COMPOSITE_COMPONENT_BASE; capability=composite_component_contract-->
- `B3` 表单域组件基  <!--@base id=B3; name=FORM_COMPONENT_BASE; capability=form_component_contract-->
- `B4` 反馈组件基  <!--@base id=B4; name=FEEDBACK_COMPONENT_BASE; capability=feedback_component_contract-->
- `B5` 可访问适配基  <!--@base id=B5; name=A11Y_COMPONENT_BASE; capability=a11y_component_contract-->

## 4. 组合原则（Combination Principles）

仅允许下游向上游组合：`L3 -> L4`。禁止同层组合与反向组合。

组合约束集合：
- `C1` 相邻层约束
- `C2` 下游上行约束
- `C3` 组件到状态契约

- 组件原子基组合到数据查询基。  <!--@compose from=L3.Mfrontend.B1; to=L4.Mfrontend.B2; rule=adjacent_only; principle=P_DOWNSTREAM_UPSTREAM; constraint=C1_C2_C3-->
- 复合组件基组合到状态模型基。  <!--@compose from=L3.Mfrontend.B2; to=L4.Mfrontend.B1; rule=adjacent_only; principle=P_DOWNSTREAM_UPSTREAM; constraint=C1_C2_C3-->
- 复合组件基组合到数据查询基。  <!--@compose from=L3.Mfrontend.B2; to=L4.Mfrontend.B2; rule=adjacent_only; principle=P_DOWNSTREAM_UPSTREAM; constraint=C1_C2_C3-->
- 表单域组件基组合到变更事务基。  <!--@compose from=L3.Mfrontend.B3; to=L4.Mfrontend.B3; rule=adjacent_only; principle=P_DOWNSTREAM_UPSTREAM; constraint=C1_C2_C3-->
- 表单域组件基组合到数据查询基。  <!--@compose from=L3.Mfrontend.B3; to=L4.Mfrontend.B2; rule=adjacent_only; principle=P_DOWNSTREAM_UPSTREAM; constraint=C1_C2_C3-->
- 反馈组件基组合到副作用治理基。  <!--@compose from=L3.Mfrontend.B4; to=L4.Mfrontend.B4; rule=adjacent_only; principle=P_DOWNSTREAM_UPSTREAM; constraint=C1_C2_C3-->
- 可访问适配基组合到状态模型基。  <!--@compose from=L3.Mfrontend.B5; to=L4.Mfrontend.B1; rule=adjacent_only; principle=P_DOWNSTREAM_UPSTREAM; constraint=C1_C2_C3-->

## 5. 验证（Verification）

- 每个 L3 基必须至少指向一个 L4 父节点。
- 每个 L4 基必须至少由一个 L3 子节点支撑。
- 不允许 `L3 -> L3` 或 `L3 -> L5`。
