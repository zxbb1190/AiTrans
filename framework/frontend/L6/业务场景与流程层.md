# 前端框架模块标准（L6）业务场景与流程层
<!--@layer module=frontend; level=L6-->

## 1. 目标（Goal）

沉淀业务场景流程与领域语义，收敛为可验收的上游体验目标。

## 2. 边界定义（Boundary）

- `USECASE_SCOPE`：业务用例覆盖范围。
- `DOMAIN_SCOPE`：领域语义与跨域边界。
- `FAILURE_SCOPE`：失败分支与补偿边界。

## 3. 最小可行基（Bases）

- `B1` 业务用例基  <!--@base id=B1; name=USECASE_BASE; capability=business_usecase_orchestration-->
- `B2` 领域契约基  <!--@base id=B2; name=DOMAIN_CONTRACT_BASE; capability=domain_contract_alignment-->

## 4. 组合原则（Combination Principles）

仅允许下游向上游组合：`L6 -> L7`。禁止同层组合与反向组合。

组合约束集合：
- `C1` 相邻层约束
- `C2` 下游上行约束
- `C3` 业务语义到体验目标契约

组合映射（示例）：
- `U1 = {L6.B1, L6.B2} --[C1,C2,C3]--> L7.B1`

- 业务用例基组合到体验目标基。  <!--@compose from=L6.Mfrontend.B1; to=L7.Mfrontend.B1; rule=adjacent_only; principle=P_DOWNSTREAM_UPSTREAM; constraint=C1_C2_C3-->
- 领域契约基组合到体验目标基。  <!--@compose from=L6.Mfrontend.B2; to=L7.Mfrontend.B1; rule=adjacent_only; principle=P_DOWNSTREAM_UPSTREAM; constraint=C1_C2_C3-->

## 5. 验证（Verification）

- 每个 L6 基必须至少指向一个 L7 父节点。
- L7 基必须由至少一个 L6 子节点支撑。
- 不允许 `L6 -> L6` 或 `L7 -> L6`。
