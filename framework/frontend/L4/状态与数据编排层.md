# 前端框架模块标准（L4）状态与数据编排层
<!--@layer module=frontend; level=L4-->

## 1. 目标（Goal）

统一状态、数据与副作用编排，形成业务流程的可靠中间层。

## 2. 边界定义（Boundary）

- `STATE_SCOPE`：状态实体与生命周期边界。
- `DATA_SCOPE`：查询、缓存、失效边界。
- `EFFECT_SCOPE`：副作用触发与隔离边界。

## 3. 最小可行基（Bases）

- `B1` 状态模型基  <!--@base id=B1; name=STATE_MODEL_BASE; capability=state_modeling-->
- `B2` 数据查询基  <!--@base id=B2; name=DATA_QUERY_BASE; capability=query_cache_policy-->
- `B3` 变更事务基  <!--@base id=B3; name=MUTATION_BASE; capability=mutation_transaction-->
- `B4` 副作用治理基  <!--@base id=B4; name=EFFECT_GOV_BASE; capability=effect_governance-->

## 4. 组合原则（Combination Principles）

仅允许下游向上游组合：`L4 -> L5`。禁止同层组合与反向组合。

组合约束集合：
- `C1` 相邻层约束
- `C2` 下游上行约束
- `C3` 状态到页面流程契约

组合映射（示例）：
- `U1 = {L4.B1, L4.B2} --[C1,C2,C3]--> L5.B1`
- `U2 = {L4.B1, L4.B3} --[C1,C2,C3]--> L5.B2`
- `U3 = {L4.B3, L4.B4} --[C1,C2,C3]--> L5.B3`

- 状态模型基组合到路由流程基。  <!--@compose from=L4.Mfrontend.B1; to=L5.Mfrontend.B2; rule=adjacent_only; principle=P_DOWNSTREAM_UPSTREAM; constraint=C1_C2_C3-->
- 状态模型基组合到页面容器编排基。  <!--@compose from=L4.Mfrontend.B1; to=L5.Mfrontend.B1; rule=adjacent_only; principle=P_DOWNSTREAM_UPSTREAM; constraint=C1_C2_C3-->
- 数据查询基组合到页面容器编排基。  <!--@compose from=L4.Mfrontend.B2; to=L5.Mfrontend.B1; rule=adjacent_only; principle=P_DOWNSTREAM_UPSTREAM; constraint=C1_C2_C3-->
- 变更事务基组合到导航策略基。  <!--@compose from=L4.Mfrontend.B3; to=L5.Mfrontend.B3; rule=adjacent_only; principle=P_DOWNSTREAM_UPSTREAM; constraint=C1_C2_C3-->
- 变更事务基组合到路由流程基。  <!--@compose from=L4.Mfrontend.B3; to=L5.Mfrontend.B2; rule=adjacent_only; principle=P_DOWNSTREAM_UPSTREAM; constraint=C1_C2_C3-->
- 副作用治理基组合到导航策略基。  <!--@compose from=L4.Mfrontend.B4; to=L5.Mfrontend.B3; rule=adjacent_only; principle=P_DOWNSTREAM_UPSTREAM; constraint=C1_C2_C3-->

## 5. 验证（Verification）

- 每个 L4 基必须至少指向一个 L5 父节点。
- 每个 L5 基必须至少由一个 L4 子节点支撑。
- 不允许 `L4 -> L4` 或 `L4 -> L6`。
