# 前端承载与绑定原子模块:FrontendCarrierBindingAtoms

@framework

本模块定义前端框架最底层的承载与绑定结构。它只负责回答“界面如何承载内容、如何落到宿主、如何保持可达”，不再把壳体、槽位、宿主、焦点和可访问拆成多份横切模板。

## 1. 能力声明（Capability Statement）

- `C1` 承载面组织能力：把主承载区、辅助承载区与槽位顺序收敛为稳定视图壳结构。
- `C2` 宿主绑定能力：把宿主元素、属性入口、事件出口与引用句柄收敛为稳定绑定结构。
- `C3` 可达闭合能力：把焦点顺序、阅读提示与状态播报关系收敛为稳定可访问结构。
- `C4` 非能力项：不负责具体组件语义、主题值与项目实例布局方案。

## 2. 边界定义（Boundary / 参数）

- `SURFACE` 承载面边界：主承载区、辅助承载区与阅读分区需稳定。来源：`C1 + C3`。
- `SLOT` 槽位边界：前缀槽位、主体槽位、后缀槽位与空态占位需稳定。来源：`C1 + C2`。
- `HOST` 宿主边界：宿主元素、语义角色与宿主替换位需稳定。来源：`C2 + C3`。
- `PROP` 绑定边界：属性入口、事件出口与引用句柄需稳定。来源：`C2 + C3`。
- `FOCUS` 焦点边界：焦点顺序、焦点回路与回退落点需稳定。来源：`C1 + C3`。
- `A11Y` 可访问边界：阅读提示、关系说明与状态播报位需可达。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 视图壳与槽位结构基：由主承载区、辅助承载区、顺序槽位与空态占位构成的视图壳结构。来源：`C1 + SURFACE + SLOT`。
- `B2` 宿主绑定可达结构基：由宿主元素、属性入口、事件出口、焦点顺序与提示关系构成的绑定结构。来源：`C2 + C3 + HOST + PROP + FOCUS + A11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 承载壳先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定承载面与槽位顺序，再允许更高层结构附着其上。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`SURFACE/SLOT`。
- `R2` 绑定闭合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在承载壳上对齐宿主元素、属性出口、焦点顺序与提示关系，形成可直接被前端实现承接的基础结构。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`HOST/PROP/FOCUS/A11Y/SURFACE/SLOT`。
- `R3` 禁止组合
  - `R3.1` 参与基：`B1 + B2`。
  - `R3.2` 组合方式：跳过承载壳直接写宿主绑定、焦点回路不可解释或阅读提示缺位的方案均无效。
  - `R3.3` 输出能力：`C4`。
  - `R3.4` 边界绑定：`SURFACE/SLOT/HOST/PROP/FOCUS/A11Y`。

## 5. 验证（Verification）

- `V1` 承载完整性：主承载区、辅助承载区与槽位顺序必须完整。
- `V2` 绑定完整性：宿主元素、属性入口与事件出口必须满足 `HOST/PROP` 约束。
- `V3` 可达完整性：焦点回路与阅读提示必须满足 `FOCUS/A11Y` 约束。
- `V4` 最小必要性：移除任一 `B*` 后，`V2~V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
