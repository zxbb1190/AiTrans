# 前端文本输入原子模块:FrontendTextInputAtoms

@framework

本模块定义前端连续文本编辑原子。它承接输入框和文本域这类“持续编辑、持续同步”的文本输入结构，不把离散点击和文件选择混进同一模块。

## 1. 能力声明（Capability Statement）

- `C1` 单行编辑承载能力：把单值输入、占位文本与即时校验收敛为稳定编辑结构。
- `C2` 多行编辑承载能力：把多行内容、滚动策略与高度扩展收敛为稳定编辑结构。
- `C3` 绑定同步能力：把值流、提交出口与可访问提示收敛为稳定文本输入接口。
- `C4` 非能力项：不负责离散选择控件、集合导航与页面流程编排。

## 2. 边界定义（Boundary / 参数）

- `FIELD` 单行边界：单行输入框、占位文本与即时校验位需稳定。来源：`C1 + C3`。
- `AREA` 多行边界：多行内容区、滚动区与自动扩展策略需稳定。来源：`C2 + C3`。
- `VALUE` 值流边界：默认值、当前值与变更同步顺序需稳定。来源：`C1 + C2`。
- `SUBMIT` 提交边界：提交出口、快捷提交与清空出口需稳定。来源：`C1 + C3`。
- `STATUS` 状态边界：聚焦、禁用、处理中与错误状态需稳定。来源：`C2 + C3`。
- `A11Y` 可访问边界：标签关系、帮助信息与错误提示需可达。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 单行编辑结构基：L0.M0[R2] + L0.M1[R1,R2] + L0.M2[R1,R2]。来源：`C1 + FIELD + VALUE + SUBMIT + STATUS`。
- `B2` 多行编辑结构基：L0.M0[R2] + L0.M1[R1,R2] + L0.M2[R1,R2]。来源：`C2 + C3 + AREA + VALUE + SUBMIT + A11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 单行编辑先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定单值输入、占位提示与提交回退顺序，形成稳定单行编辑原子。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`FIELD + VALUE + SUBMIT + STATUS`。
- `R2` 多行编辑并轨
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在单行编辑结构上接入多行滚动、自动扩展与长文本提示，形成统一文本输入原子。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`AREA + VALUE + SUBMIT + A11Y + STATUS`。
- `R3` 禁止组合
  - `R3.1` 参与基：`B1 + B2`。
  - `R3.2` 组合方式：值流不可解释、提交出口缺位或多行滚动策略不附着稳定编辑壳的方案均无效。
  - `R3.3` 输出能力：`C4`。
  - `R3.4` 边界绑定：`FIELD + AREA + VALUE + SUBMIT + STATUS + A11Y`。

## 5. 验证（Verification）

- `V1` 单行完整性：输入壳、占位文本与提交出口必须完整。
- `V2` 多行完整性：滚动区、自动扩展与长文本提示必须满足 `AREA` 约束。
- `V3` 同步有效性：值流、状态出口与提示关系必须满足 `VALUE + STATUS + A11Y` 约束。
- `V4` 最小必要性：移除任一 `B*` 后，`V2~V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* + 边界绑定` 的通过或失败结论。
