# 窗帘抽象结构模块:CurtainAbstractStructure

@framework

## 1. 能力声明（Capability Statement）

- `C1` 结构抽象能力：定义窗帘系统最小稳定结构单元与组合接口。
- `C2` 运动承载能力：定义遮挡体在导向结构中的可控运动能力。
- `C3` 约束治理能力：定义安全、噪声和环境适配的约束边界。
- `C4` 非能力项：不负责具体布料花色、装饰样式和场景陈设。

## 2. 边界定义（Boundary / 参数）

- `SPAN` 跨度约束：有效覆盖范围和跨距上限需明确。来源：`C1 + C2`。
- `LOAD` 承载约束：承载质量和动态载荷上限需可验证。来源：`C1 + C2`。
- `TRAVEL` 行程约束：开合行程和端点缓冲范围需确定。来源：`C2`。
- `MOUNT` 安装约束：墙装、顶装和侧装条件需可区分。来源：`C1 + C3`。
- `SAFETY` 安全约束：夹持风险和失效保护边界需可执行。来源：`C2 + C3`。
- `ENV` 环境约束：温湿度与光照影响范围需可量化。来源：`C1 + C3`。
- `NOISE` 噪声约束：运行噪声阈值和抑制策略需明确。来源：`C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 承载导向基：由承载路径、导向轨迹和连接节点构成的结构骨架。来源：`C1 + SPAN + LOAD + MOUNT`。
- `B2` 运动传递基：由驱动链、行程限位和端点缓冲单元构成的运动结构。来源：`C2 + TRAVEL + LOAD + NOISE`。
- `B3` 安全约束基：由失效保护单元、环境适配层和风险隔离节点构成的治理结构。来源：`C3 + SAFETY + ENV + MOUNT`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 结构先行组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立承载导向结构，再接入运动与治理能力。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`SPAN + LOAD + MOUNT`。
- `R2` 运动承载组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在稳定导向结构上叠加运动驱动，形成可控开合能力。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`TRAVEL + LOAD + NOISE + SPAN`。
- `R3` 安全闭环组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将安全与环境约束绑定到运动链路，形成完整治理闭环。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`SAFETY + ENV + MOUNT + TRAVEL`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：缺失承载导向、忽略失效保护或超出环境噪声边界的方案均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`SPAN + LOAD + TRAVEL + MOUNT + SAFETY + ENV + NOISE`。

## 5. 验证（Verification）

- `V1` 结构完整性：导向结构与承载路径必须满足跨度和载荷约束。
- `V2` 运动有效性：开合行程必须可控且端点行为可验证。
- `V3` 治理有效性：安全保护与环境适配策略必须可执行。
- `V4` 边界符合性：所有有效组合必须满足 `NOISE + SAFETY + ENV` 阈值。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 边界绑定` 的通过或失败结论。
