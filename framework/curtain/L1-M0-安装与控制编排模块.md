# 安装控制编排模块:InstallControlOrchestration

@framework

## 1. 能力声明（Capability Statement）

- `C1` 安装编排能力：将抽象结构落地为可执行安装步骤与验收条件。
- `C2` 控制编排能力：将控制输入转换为稳定可预测的开合行为。
- `C3` 运维恢复能力：支持校准、故障诊断与回退恢复流程。
- `C4` 非能力项：不负责具体装饰风格和业务展示交互。

## 2. 边界定义（Boundary / 参数）

- `INSTALL` 安装流程约束：安装步骤顺序与工序依赖需明确。来源：`C1`。
- `ALIGN` 对齐精度约束：定位误差与水平垂直偏差需可测量。来源：`C1 + C2`。
- `POWER` 供电约束：供电规格、波动范围与保护策略需定义。来源：`C2 + C3`。
- `CTRL` 控制协议约束：本地控制与远程控制命令集合需一致。来源：`C2`。
- `CAL` 校准约束：行程与端点校准条件需可重复执行。来源：`C2 + C3`。
- `FAULT` 故障处理约束：故障分类和恢复优先级需可追踪。来源：`C2 + C3`。
- `MAINT` 运维约束：巡检周期和维护窗口需可计划。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 安装定位基：L0.M0[R1]。来源：`C1 + INSTALL + ALIGN + MAINT`。
- `B2` 控制执行基：L0.M0[R2]。来源：`C2 + CTRL + POWER + ALIGN`。
- `B3` 校准恢复基：L0.M0[R3]。来源：`C3 + CAL + FAULT + MAINT`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 安装基线组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先完成安装与定位基线，再进入控制编排阶段。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`INSTALL + ALIGN + MAINT`。
- `R2` 控制执行组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在安装基线之上接入控制协议与供电联动，形成可执行控制链路。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`CTRL + POWER + ALIGN + INSTALL`。
- `R3` 运维恢复组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将校准和故障恢复机制绑定到控制链路形成可维护闭环。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`CAL + FAULT + MAINT + CTRL`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：未完成定位验收即接入控制，或缺失故障恢复路径的方案均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`INSTALL + ALIGN + POWER + CTRL + CAL + FAULT + MAINT`。

## 5. 验证（Verification）

- `V1` 安装一致性：安装流程必须符合 `INSTALL` 和 `ALIGN` 要求。
- `V2` 控制一致性：同一控制命令在同条件下输出结果必须一致。
- `V3` 恢复有效性：故障场景必须存在可执行校准或回退路径。
- `V4` 边界符合性：系统运行必须满足 `POWER + FAULT + MAINT` 约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 边界绑定` 的通过或失败结论。
