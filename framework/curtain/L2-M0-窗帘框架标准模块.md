# 窗帘系统框架模块:CurtainSystemFramework

@framework

本模块只负责窗帘系统的共同结构语言、边界与组合规则，不得直接泄漏项目专属值、实例值或技术栈细节。

## 1. 能力声明（Capability Statement）

- `C1` 窗帘结构集成能力：集成承载、导向与连接结构形成稳定系统。
- `C2` 开合控制能力：实现可控开合、状态反馈和模式切换能力。
- `C3` 场景适配能力：在不同安装场景下保持安全、静音与可维护性。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责布料图案、空间软装风格和装饰偏好。

## 2. 边界定义（Boundary / 参数）

- `SPAN` 覆盖跨度约束：有效覆盖宽度区间需可验证。来源：`C1 + C3`。
- `LOAD` 载荷约束：系统可承载质量与动态余量需可计算。来源：`C1 + C2`。
- `TRAVEL` 开合行程约束：行程范围、端点缓冲和停靠精度需明确。来源：`C2`。
- `CTRL` 控制约束：控制指令集、反馈频率和状态同步需一致。来源：`C2 + C3`。
- `SAFE` 安全约束：异常阻塞、断电与误触场景保护需具备。来源：`C2 + C3`。
- `ENV` 环境约束：温湿度、粉尘和光照影响范围需量化。来源：`C1 + C3`。
- `NOISE` 运行噪声约束：噪声阈值和抑制策略需可验证。来源：`C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 轨道承载基：L1.M0[R1]。来源：`C1 + SPAN + LOAD + ENV`。
- `B2` 驱动传递基：L1.M0[R2]。来源：`C2 + TRAVEL + CTRL + NOISE`。
- `B3` 场景治理基：L1.M0[R3]。来源：`C3 + SAFE + ENV + LOAD`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 承载控制主链组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先确定承载轨道与安装接口，再接入驱动控制链路形成主链路。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`SPAN + LOAD + TRAVEL + CTRL`。
- `R2` 场景安全组合
  - `R2.1` 参与基：`B1 + B3`。
  - `R2.2` 组合方式：将安全约束与环境条件绑定到承载结构，形成场景化保护策略。
  - `R2.3` 输出能力：`C1 + C3`。
  - `R2.4` 边界绑定：`SAFE + ENV + LOAD + SPAN`。
- `R3` 完整系统闭环组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：合并主链控制与场景治理能力，形成可控、可维护、可回退系统。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`SPAN + LOAD + TRAVEL + CTRL + SAFE + ENV + NOISE`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：缺失轨道承载基、控制反馈链路或安全保护策略的系统组合均无效。
  - `R4.3` 失效结论：`N1`。
  - `R4.4` 边界绑定：`SPAN + LOAD + TRAVEL + CTRL + SAFE + ENV + NOISE`。

## 5. 验证（Verification）

- `V1` 主链完整性：`R1` 必须满足承载与控制主链路完整要求。
- `V2` 安全有效性：`R2` 必须覆盖异常阻塞、断电和误触场景。
- `V3` 闭环覆盖性：`R3` 输出能力并集必须覆盖 `C1~C3`。
- `V4` 边界符合性：所有有效组合必须满足 `SAFE + NOISE + ENV` 阈值。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 边界绑定` 或 `R* -> N* + 边界绑定` 的通过或失败结论。
