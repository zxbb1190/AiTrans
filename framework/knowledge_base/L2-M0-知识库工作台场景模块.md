# 知识库工作台场景模块:KnowledgeBaseWorkbenchScenarios

@framework

本模块是知识库领域通用框架的场景层。它建立在知识库原子结构和知识引用上下文结构之上，同时默认由 frontend `L1` 的触发选择、文本输入、展示容器、集合导航和标记反馈原子族提供更基础的渲染承接结构，而这些结构又建立在 frontend `L0` 基础原子层之上。项目实例应把本模块收敛成显式 workbench contract、上下文回路和规则校验，而不是拆散这条主链。默认工作台形态是 chat-first：左侧承接历史会话和知识库入口，中部承接对话，来源细节通过按需打开的引用抽屉与辅助详情页展开。对具体实例而言，`LIBRARY`、`PREVIEW`、`CHAT`、`CONTEXT`、`RETURN` 构成稳定的大类边界；界面外形与布局 recipe 继续由 frontend 的 `SURFACE / VISUAL / ROUTE / A11Y` 边界族承接。本模块只负责知识库工作台场景的共同结构语言，不规定具体实例值或实现路径。

## 1. 能力声明（Capability Statement）

- `C1` chat-first 工作台建模能力：将会话侧栏、知识库入口、对话主舞台与引用来源收敛为统一工作台场景。
- `C2` 上下文衔接能力：统一当前文件、当前章节、引用域与回合上下文在场景间的传递结构。
- `C3` 引用对话闭环能力：支持上传文件、选择引用域、基于知识库对话并回看引用来源。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责前端底座实现、后端索引算法与模型本身推理质量。

## 2. 边界定义（Boundary / 参数）

- `SURFACE` 场景边界：至少覆盖会话侧栏、对话主舞台、引用抽屉与辅助详情页；其布局 recipe 继续由 frontend 的 `SURFACE` 边界族承接。来源：`C1 + C2`。
- `LIBRARY` 文件库边界：创建、移除、聚焦、筛选与选择引用域动作需统一。来源：`C1 + C3`。
- `PREVIEW` 预览边界：引用抽屉、文档正文、TOC 导航与章节焦点切换需稳定。来源：`C1 + C2`。
- `CHAT` 对话边界：输入、回合顺序、引用附着与回看路径需稳定。来源：`C1 + C3`。
- `CONTEXT` 上下文边界：当前文件、当前章节、当前引用域与当前回合需稳定传递。来源：`C2 + C3`。
- `RETURN` 返回边界：从引用回合返回文件、目录与文档锚点的路径需明确；返回卡片变体与 anchor restore 策略需收敛在 `RETURN` 边界内。来源：`C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 会话与知识入口场景基：L1.M0[R1,R2]。来源：`C1 + SURFACE + LIBRARY + PREVIEW`。
- `B2` 对话引用场景基：L1.M0[R1,R3] + L1.M1[R1,R2]。来源：`C3 + CHAT + CONTEXT + RETURN`。
- `B3` chat-first 客户端闭环场景基：L1.M0[R3] + L1.M1[R2,R3]。来源：`C2 + SURFACE + CONTEXT + RETURN`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 会话知识主链
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定会话侧栏、知识库入口和来源详情展开之间的主链关系，再允许文件库与预览边界内的实例差异在该主链内变化。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`SURFACE + LIBRARY + PREVIEW`。
- `R2` 对话引用并轨
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在会话与来源主链上接入对话回合、引用域、引用抽屉与文档详情返回路径，形成知识库对话场景；对话、上下文与返回相关实例差异只能在既定闭环内变化。
  - `R2.3` 输出能力：`C1 + C3`。
  - `R2.4` 边界绑定：`CHAT + CONTEXT + RETURN + LIBRARY + PREVIEW`。
- `R3` 客户端上下文闭环
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：联合会话与来源主链、对话引用并轨和上下文回路，形成稳定知识库 chat-first 客户端场景。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`SURFACE + LIBRARY + PREVIEW + CHAT + CONTEXT + RETURN`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：缺失会话侧栏、引用抽屉与对话主舞台的稳定关系，或边界归属被拆散、引用无法返回来源抽屉和文档详情页的方案均无效。
  - `R4.3` 失效结论：`N1`。
  - `R4.4` 边界绑定：`SURFACE + LIBRARY + PREVIEW + CHAT + CONTEXT + RETURN`。

## 5. 验证（Verification）

- `V1` 场景完整性：会话侧栏、引用抽屉、辅助详情页与对话区必须全部存在。
- `V2` 上下文一致性：当前文件、当前章节、引用域与回合上下文必须可追踪且一致。
- `V3` 闭环有效性：知识库切换、对话引用与返回来源路径必须形成稳定闭环。
- `V4` 最小必要性：移除任一 `B*` 后，`V1~V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* + 边界绑定` 或 `R* -> N* + 边界绑定` 的通过或失败结论。
- `V6` 边界归属稳定性：文件库、预览、对话、上下文与返回相关实例差异必须严格归属于对应边界大类。
- `V7` 承接闭合性：承接本场景的上传、预览、对话、上下文与返回结构必须进入稳定工作台主链，不允许形成悬空承接面。
