# 知识库界面骨架模块:KnowledgeBaseWorkbenchSkeleton

@framework

本模块在知识库领域内部装配会话侧栏、对话主舞台、来源抽屉和辅助详情页四类原子结构；其直接渲染面由 frontend `L1` 组件原子提供，而这些组件原子又建立在 frontend `L0` 共相结构之上，本模块不重复定义按钮、输入框、文本域、文件选择、文本块、面板、列表、树、标签与气泡这些前端原子。实例配置在本层保持“大类边界 -> 大类 section”约定：骨架和视觉 recipe 由 frontend `[surface]/[visual]` 承接，知识库内部只在 `[library]/[preview]/[chat]/[context]/[return]` 中填充小类字段。

## 1. 能力声明（Capability Statement）

- `C1` chat-first 骨架装配能力：把会话侧栏、对话主舞台、来源抽屉与辅助详情页装配为稳定知识库界面骨架。
- `C2` 多区焦点联动能力：把文件焦点、章节焦点、引用焦点与对话焦点组织为稳定联动结构。
- `C3` 响应式骨架复用能力：让桌面多区与移动分区共享同一骨架约束。
- `C4` 非能力项：不负责项目级路由策略、后端接口实现与模型推理细节。

## 2. 边界定义（Boundary / 参数）

- `REGION` 区域边界：会话侧栏、对话主舞台、来源抽屉与辅助详情页职责需明确。来源：`C1 + C3`。
- `FOCUS` 焦点边界：文件焦点、章节焦点、引用焦点与对话焦点传递关系需稳定。来源：`C1 + C2`。
- `EMPTY` 空态边界：无文件、无文档与无对话场景需有统一承载结构。来源：`C1 + C3`。
- `RESPONSIVE` 响应式边界：桌面多区与移动折叠结构需满足同一骨架规则。来源：`C1 + C3`。
- `ENTRY` 入口边界：文件进入、对话进入与来源进入的默认落点需明确。来源：`C1 + C2`。
- `A11Y` 可访问边界：多区阅读顺序与焦点回路需达标。来源：`C1 + C2`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 会话骨架结构基：L0.M0[R1,R2] + L0.M1[R1,R2]。来源：`C1 + REGION + ENTRY + EMPTY`。
- `B2` 来源抽屉结构基：L0.M1[R2,R3] + L0.M2[R1,R2]。来源：`C1 + C2 + FOCUS + REGION + A11Y`。
- `B3` chat-first 客户端骨架基：L0.M0[R3] + L0.M1[R3] + L0.M2[R3]。来源：`C1 + C2 + C3 + RESPONSIVE + EMPTY + FOCUS`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 骨架区块先行
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先固定会话侧栏、对话主舞台、来源抽屉和辅助详情页之间的区块关系，再建立焦点联动。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`REGION/FOCUS/ENTRY/A11Y`。
- `R2` 空态与响应式收敛
  - `R2.1` 参与基：`B1 + B3`。
  - `R2.2` 组合方式：将空态承载与桌面/移动切换规则并入骨架，保证不同设备共享同一 chat-first 界面骨架。
  - `R2.3` 输出能力：`C1 + C3`。
  - `R2.4` 边界绑定：`EMPTY/RESPONSIVE/REGION/ENTRY`。
- `R3` 工作台骨架闭环
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：联合区块关系、焦点联动和响应式承载，形成稳定知识库 chat-first 客户端界面骨架。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`REGION/FOCUS/EMPTY/RESPONSIVE/ENTRY/A11Y`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：缺失会话侧栏、对话主舞台与来源抽屉之间的稳定关系，或焦点回路不可解释的方案均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`REGION/FOCUS/EMPTY/RESPONSIVE/ENTRY/A11Y`。

## 5. 验证（Verification）

- `V1` 骨架完整性：会话侧栏、来源抽屉和对话主舞台必须全部存在。
- `V2` 联动有效性：文件焦点、章节焦点、引用焦点与对话焦点必须稳定联动。
- `V3` 响应式一致性：桌面多区与移动分区必须共享同一骨架约束。
- `V4` 可访问符合性：有效组合必须满足 `A11Y` 约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
