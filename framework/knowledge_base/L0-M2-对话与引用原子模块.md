# 知识库对话与引用原子模块:KnowledgeConversationCitationAtoms

@framework

本模块定义知识库领域的最小对话与引用结构。推导链为：`framework/frontend/L1-M0-触发与选择原子模块.md` + `framework/frontend/L1-M1-文本输入原子模块.md` + `framework/frontend/L1-M2-展示与容器原子模块.md` + `framework/frontend/L1-M4-标记与反馈原子模块.md` -> 本模块。知识库对话与引用由触发选择、文本输入、展示容器与标记反馈这些更基础的前端原子族组成，而这些原子族又建立在更底层的前端 `L0` 基础原子之上。该模块默认服务于“chat-first + inline refs + citation drawer”这类知识问答客户端，而不是无来源的纯聊天界面。项目实例不会在本层直接暴露碎片化参数，而是通过上层 `[chat]/[context]/[return]` section 在大类边界内取值。

## 1. 能力声明（Capability Statement）

- `C1` 对话舞台承载能力：把用户回合、助手回合、系统提示与回合顺序抽象为稳定结构。
- `C2` 行内引用附着承载能力：把回合与知识文件片段、章节锚点、行内引用编号、来源抽屉和返回动作之间的关系抽象为稳定结构。
- `C3` 知识域约束能力：把当前选中文件、全库引用范围与回合上下文外化为稳定边界。
- `C4` 非能力项：不负责文件摄取、文档解析与模型推理实现。

## 2. 边界定义（Boundary / 参数）

- `TURN` 回合边界：用户回合、助手回合、系统回合与时间顺序需稳定。来源：`C1 + C3`。
- `INPUT` 输入边界：当前输入草稿、提交出口、快捷提交与清空规则需稳定。来源：`C1 + C3`。
- `CITATION` 引用边界：引用文件、章节锚点、片段、行内引用编号、来源抽屉与返回路径字段需完整。来源：`C2 + C3`。
- `SCOPE` 引用域边界：当前选中文件、全库默认域与临时扩展域需可区分。来源：`C2 + C3`。
- `STATUS` 状态边界：生成中、成功、失败与“无可用引用”回合需有稳定承载结构。来源：`C1 + C2`。
- `A11Y` 可访问边界：消息阅读、输入提交与引用展开需可达。来源：`C1 + C2`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 对话舞台结构基：frontend.L1.M2[R1,R2] + frontend.L1.M4[R1,R2]。来源：`C1 + TURN + STATUS + A11Y`。
- `B2` 提问提交结构基：frontend.L1.M0[R1,R2] + frontend.L1.M1[R1,R2]。来源：`INPUT + A11Y + STATUS`。
- `B3` 行内引用结构基：frontend.L1.M2[R1,R2] + frontend.L1.M4[R1,R2]。来源：`C2 + C3 + CITATION + SCOPE`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 对话舞台先行
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先固定回合顺序与输入提交结构，再允许行内引用编号、来源抽屉和来源动作挂接到助手回合上。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`TURN + INPUT + STATUS + A11Y`。
- `R2` 引用卡片并轨
  - `R2.1` 参与基：`B1 + B3`。
  - `R2.2` 组合方式：将引用文件、章节锚点、片段摘要、行内引用编号与返回动作附着到助手回合上，形成稳定可回溯关系。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`CITATION + SCOPE + TURN + STATUS`。
- `R3` 对话引用闭环
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：联合回合顺序、输入提交和引用卡片结构，形成可提问、可引用、可回看来源的对话结构。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`TURN + INPUT + CITATION + SCOPE + STATUS + A11Y`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：缺失输入出口、引用不附着到稳定回合、返回动作缺位或引用域不可解释的方案均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`TURN + INPUT + CITATION + SCOPE + STATUS + A11Y`。

## 5. 验证（Verification）

- `V1` 回合完整性：用户回合、助手回合与顺序结构必须完整。
- `V2` 输入完整性：输入草稿与提交出口必须满足 `INPUT` 约束。
- `V3` 引用有效性：引用条目必须满足 `CITATION + SCOPE` 约束，并能附着到稳定回合、打开来源抽屉与跳到文档详情。
- `V4` 可访问符合性：有效组合必须满足 `A11Y` 约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 边界绑定` 的通过或失败结论。
