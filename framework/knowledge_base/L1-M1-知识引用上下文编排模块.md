# 知识引用上下文编排模块:KnowledgeCitationContextOrchestration

@framework

本模块定义知识库领域内部的引用域、文档锚点和对话回合之间的上下文关系。它建立在知识库原子结构之上，并以 `frontend/L1-M0 ~ L1-M4` 的触发选择、文本输入、展示容器、集合导航和标记反馈原子族为直接承接，再以下层的 frontend `L0` 基础原子作为支撑。该模块默认服务于“默认轻量回答、按需展开来源抽屉、可进入文档详情”的知识对话客户端。实例配置在本层只开放 `[chat]/[context]/[return]` 中的小类字段，不把边界拆成过细配置轴。

## 1. 能力声明（Capability Statement）

- `C1` 知识域编排能力：把当前允许引用的文件集、当前聚焦文件与回合上下文收敛为稳定结构。
- `C2` 来源锚点衔接能力：把文件焦点、章节锚点、引用抽屉与文档详情衔接为稳定关系结构。
- `C3` 回合回溯能力：把行内引用附着、回合输出与返回文档路径收敛为稳定回路。
- `C4` 非能力项：不负责文件摄取入口、项目路由策略与模型推理实现。

## 2. 边界定义（Boundary / 参数）

- `SCOPE` 引用域边界：当前允许引用的文件集合、当前聚焦文件、默认域与切换规则需明确。来源：`C1 + C3`。
- `ANCHOR` 锚点边界：文件锚点、章节锚点、来源抽屉定位与片段边界需稳定。来源：`C2 + C3`。
- `TURNLINK` 回合链接边界：行内引用必须挂接到稳定回合并可回看。来源：`C1 + C3`。
- `RETURN` 返回边界：从引用回合返回来源抽屉、文档详情与聊天页的路径需明确。来源：`C2 + C3`。
- `TRACE` 追踪边界：引用来源、回合关系与切换动作需可追踪。来源：`C1 + C3`。
- `STATUS` 状态边界：引用生成中、引用成功与引用失败需有稳定表达。来源：`C1 + C2`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 知识域装配基：L0.M0[R3] + L0.M2[R2,R3]。来源：`C1 + SCOPE + STATUS + TRACE`。
- `B2` 来源抽屉衔接基：L0.M1[R1,R3] + L0.M2[R2,R3]。来源：`C2 + ANCHOR + RETURN + TURNLINK`。
- `B3` 引用上下文回路基：L0.M0[R1,R3] + L0.M1[R2,R3] + L0.M2[R1,R3]。来源：`C1 + C2 + C3 + SCOPE + ANCHOR + TRACE`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 知识域先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定允许引用的文件域、当前聚焦文件与回合挂接关系，再允许文档锚点接入。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`SCOPE/TURNLINK/STATUS/TRACE`。
- `R2` 来源锚点并轨
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：将文件焦点、章节锚点、引用片段、来源抽屉和文档详情入口对齐到稳定回合上，形成可回看的来源结构。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`ANCHOR/RETURN/TURNLINK/SCOPE`。
- `R3` 上下文回路闭合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：联合引用域、锚点衔接与回合回路，形成稳定知识引用上下文结构。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`SCOPE/ANCHOR/TURNLINK/RETURN/TRACE/STATUS`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：引用域不可解释、当前聚焦文件不可判定、锚点无法返回文档或回合关系不可追踪的方案均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`SCOPE/ANCHOR/TURNLINK/RETURN/TRACE/STATUS`。

## 5. 验证（Verification）

- `V1` 引用域完整性：允许引用的文件集与默认域必须完整。
- `V2` 锚点有效性：引用文件、章节锚点与片段必须满足 `ANCHOR` 约束。
- `V3` 回路有效性：引用必须附着到稳定回合，并具备来源抽屉和文档详情页返回路径。
- `V4` 追踪符合性：有效组合必须满足 `TRACE/STATUS` 约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
