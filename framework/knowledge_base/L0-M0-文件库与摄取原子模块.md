# 知识库文件库与摄取原子模块:KnowledgeFileLibraryIngestAtoms

@framework

本模块定义知识库领域的最小文件库结构。推导链为：`framework/frontend/L1-M0-触发与选择原子模块.md` + `framework/frontend/L1-M2-展示与容器原子模块.md` + `framework/frontend/L1-M3-集合与导航原子模块.md` + `framework/frontend/L1-M4-标记与反馈原子模块.md` -> 本模块。知识库文件库由触发选择、展示容器、集合导航与标记反馈这些更基础的前端原子族组成，而这些原子族又建立在更底层的前端 `L0` 基础原子之上。

## 1. 能力声明（Capability Statement）

- `C1` 文件集合承载能力：把知识库中的文件集合、当前焦点与分组关系抽象为稳定结构。
- `C2` 文件摄取承接能力：把上传、导入与移除入口抽象为稳定摄取结构。
- `C3` 目录约束外化能力：把数量、类型、排序与可见性约束外化为稳定边界。
- `C4` 非能力项：不负责文档正文展示、对话引用与后端存储实现。

## 2. 边界定义（Boundary / 参数）

- `FILESET` 文件集边界：文件标识、文件名、分组与当前焦点需稳定。来源：`C1 + C3`。
- `INGEST` 摄取边界：上传入口、导入入口与移除入口需稳定。来源：`C2 + C3`。
- `CLASSIFY` 分类边界：标签、类型与排序规则需统一。来源：`C1 + C3`。
- `LIMIT` 数量边界：文件数量、文件名长度与内容体量约束需可校验。来源：`C2 + C3`。
- `VISIBILITY` 可见边界：当前焦点、可管理集与可引用集需可区分。来源：`C1 + C2`。
- `A11Y` 可访问边界：文件列表、树节点与摄取入口需键盘可达。来源：`C1 + C2`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 文件目录结构基：frontend.L1.M2[R2] + frontend.L1.M3[R1,R2]。来源：`C1 + FILESET + VISIBILITY + A11Y`。
- `B2` 文件摄取结构基：frontend.L1.M0[R2] + frontend.L1.M4[R1,R2]。来源：`C2 + INGEST + LIMIT + A11Y`。
- `B3` 目录约束结构基：frontend.L1.M2[R1,R2] + frontend.L1.M3[R2] + frontend.L1.M4[R1,R2]。来源：`C1 + C3 + CLASSIFY + LIMIT + VISIBILITY`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 文件集合先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定文件集合、分组关系与当前焦点，再允许摄取与管理动作挂接。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`FILESET/VISIBILITY/A11Y`。
- `R2` 摄取并轨
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：将上传、导入与移除入口并入文件目录结构，形成稳定文件库入口面。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`INGEST/LIMIT/FILESET/VISIBILITY`。
- `R3` 目录治理闭环
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：在文件目录与摄取入口上叠加分类、数量与可见域约束，形成可治理文件库结构。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`CLASSIFY/LIMIT/VISIBILITY/INGEST/A11Y`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：缺失当前焦点、摄取入口不可解释或目录约束缺位的方案均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`FILESET/INGEST/CLASSIFY/LIMIT/VISIBILITY/A11Y`。

## 5. 验证（Verification）

- `V1` 目录完整性：文件集合、分组关系与当前焦点必须同时存在。
- `V2` 摄取完整性：上传、导入与移除入口必须满足 `INGEST/LIMIT` 约束。
- `V3` 治理有效性：分类、数量与可见域约束必须满足 `CLASSIFY/VISIBILITY` 规则。
- `V4` 可访问符合性：有效组合必须满足 `A11Y` 约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
