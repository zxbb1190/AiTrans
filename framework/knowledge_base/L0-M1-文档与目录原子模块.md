# 知识库文档与目录原子模块:KnowledgeDocumentTocAtoms

@framework

本模块定义知识库领域的最小文档结构。推导链为：`framework/frontend/L1-M2-展示与容器原子模块.md` + `framework/frontend/L1-M3-集合与导航原子模块.md` + `framework/frontend/L1-M4-标记与反馈原子模块.md` -> 本模块。知识库文档与目录由展示容器、集合导航与标记反馈这些更基础的前端原子族组成，而这些原子族又建立在更底层的前端 `L0` 基础原子之上。

## 1. 能力声明（Capability Statement）

- `C1` 文档承载能力：把当前选中文件的正文、元信息与阅读焦点抽象为稳定文档结构。
- `C2` TOC 层级承载能力：把当前文档中的章节树抽象为稳定目录结构。
- `C3` 阅读路径约束能力：把文档、目录与焦点的对应关系外化为稳定边界。
- `C4` 非能力项：不负责文件摄取、对话回合与后端解析实现。

## 2. 边界定义（Boundary / 参数）

- `DOCVIEW` 文档视图边界：正文区、元信息区与当前焦点需稳定。来源：`C1 + C3`。
- `TOC` 目录边界：目录节点、层级深度与当前章节焦点需稳定。来源：`C2 + C3`。
- `META` 元信息边界：标题、更新时间、标签与来源信息需完整。来源：`C1 + C2`。
- `FOCUS` 焦点边界：文件焦点、章节焦点与滚动焦点关系需明确。来源：`C1 + C3`。
- `EMPTY` 空态边界：无文档与无目录场景需有稳定承载结构。来源：`C1 + C2`。
- `A11Y` 可访问边界：目录跳转、正文阅读与焦点回路需可达。来源：`C1 + C2`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 文档视口结构基：frontend.L1.M2[R1,R2]。来源：`C1 + DOCVIEW + META + A11Y`。
- `B2` TOC 树结构基：frontend.L1.M2[R1] + frontend.L1.M3[R2]。来源：`C2 + TOC + FOCUS + A11Y`。
- `B3` 阅读锚点结构基：frontend.L1.M2[R2] + frontend.L1.M4[R1,R2]。来源：`C1 + C3 + FOCUS + EMPTY + A11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 文档目录对齐
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先对齐文档正文与 TOC 层级，再建立章节焦点对应关系。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`DOCVIEW/TOC/META/FOCUS`。
- `R2` 焦点路径闭合
  - `R2.1` 参与基：`B1 + B3`。
  - `R2.2` 组合方式：将文件焦点、章节锚点与空态承载并入阅读路径，形成稳定阅读回路。
  - `R2.3` 输出能力：`C1 + C3`。
  - `R2.4` 边界绑定：`FOCUS/EMPTY/A11Y/DOCVIEW`。
- `R3` 文档阅读闭环
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：联合正文视口、TOC 树和阅读锚点结构，形成可导航的文档阅读结构。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`DOCVIEW/TOC/META/FOCUS/EMPTY/A11Y`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：缺失正文承载、目录层级不可解释或焦点路径中断的方案均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`DOCVIEW/TOC/META/FOCUS/EMPTY/A11Y`。

## 5. 验证（Verification）

- `V1` 文档完整性：正文区、元信息区与当前焦点必须完整。
- `V2` 目录完整性：目录节点与章节层级必须满足 `TOC/FOCUS` 约束。
- `V3` 阅读有效性：文档与目录组合后必须形成稳定阅读路径。
- `V4` 可访问符合性：有效组合必须满足 `A11Y` 约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
