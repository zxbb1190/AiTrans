# 悬浮面板布局管理器:XuanFuMianBanBuJuGuanLiQi

@framework

## 1. 能力声明（Capability Statement）

- `C1` 面板定位能力：提供结果悬浮面板初始定位与相对锚定能力。
- `C2` 布局控制能力：提供面板尺寸、边界约束与布局调整能力。
- `C3` 拖拽移动能力：提供面板拖拽、移动与释放定位能力。
- `C4` 显隐控制能力：提供显示、隐藏、自动隐藏与恢复显示能力。
- `C5` 避障适配能力：提供屏幕边界、遮挡风险与多窗口环境下的避障布局能力。
- `C6` 非能力项：不负责面板内部内容渲染与交互动作执行。

## 2. 边界定义（Boundary / 参数）

- `POSITION` 定位约束：面板必须具备明确初始定位规则。来源：`C1`。
- `LAYOUT` 布局约束：面板布局必须满足尺寸、边界与最小显示要求。来源：`C2`。
- `DRAG` 拖拽约束：拖拽过程中必须保持位置更新与边界校正。来源：`C3`。
- `VISIBILITY` 显隐约束：显示与隐藏必须受统一状态控制。来源：`C4`。
- `AVOID` 避障约束：面板位置必须尽量避免超出屏幕或遮挡关键区域。来源：`C5`。
- `UX` 体验约束：浮层展示必须低打扰、可恢复、可控。来源：`C1 + C2 + C3 + C4 + C5`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 布局避障骨架：L0.M23[面板布局约束器]。来源：`C1 + C2 + C5 + POSITION + LAYOUT + AVOID + UX`。
- `B2` 拖拽显隐骨架：L0.M24[面板拖拽显隐器]。来源：`C3 + C4 + DRAG + VISIBILITY`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 布局避障组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立悬浮面板初始定位规则、布局约束与避障回退，形成可显示布局结果。
  - `R1.3` 输出能力：`C1 + C2 + C5`。
  - `R1.4` 边界绑定：`POSITION / LAYOUT / AVOID / UX`。

- `R2` 拖拽显隐组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在布局约束上接入拖拽与显隐控制，形成可操作悬浮面板。
  - `R2.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R2.4` 边界绑定：`DRAG / VISIBILITY / UX`。

- `R3` 禁止组合
  - `R3.1` 参与基：`B1 + B2`。
  - `R3.2` 组合方式：无布局避障骨架直接显示面板，或无避障策略直接固定覆盖关键区域的方案均无效。
  - `R3.3` 输出能力：`C6`。
  - `R3.4` 边界绑定：`LAYOUT / AVOID / UX`。

## 5. 验证（Verification）

- `V1` 布局避障有效性：面板初始定位、相对锚定、尺寸边界、避障回退与最小显示要求必须满足。
- `V2` 拖拽显隐有效性：面板拖拽、移动、显示、隐藏、自动隐藏与恢复显示必须可执行。
- `V3` 避障适配有效性：屏幕边界与遮挡风险必须被识别并规避。
- `V4` 体验达标性：面板布局必须满足 `UX` 约束。
- `V5` 最小必要性：移除任一 `B1~B2` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
