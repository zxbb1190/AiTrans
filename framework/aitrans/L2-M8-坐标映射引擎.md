# 模块框架：L2-M8-坐标映射引擎

@module

## 1. 能力声明（Capability Statement）

- `C1` 显示器描述能力：提供显示器拓扑、边界与缩放信息读取能力。
- `C2` 坐标转换能力：提供逻辑坐标到设备像素坐标的转换能力。
- `C3` 多屏映射能力：提供跨显示器场景下的选区归属与偏移映射能力。
- `C4` 边界校正能力：提供越界修正、最小区域限制与非法区域拦截能力。
- `C5` 标准输出能力：提供统一的裁剪坐标结果输出能力。
- `C6` 非能力项：不负责图像读取与图像裁剪执行。

## 2. 边界定义（Boundary / 参数）

- `DISPLAY` 显示器约束：必须可读取显示器边界、序号与缩放比例。来源：`C1`。
- `TRANSFORM` 转换约束：坐标转换必须具备确定性与可重复性。来源：`C2`。
- `MULTI` 多屏约束：跨屏场景必须正确识别目标显示器与偏移关系。来源：`C3`。
- `BOUNDARY` 边界约束：所有输出区域必须在有效屏幕范围内。来源：`C4`。
- `OUTPUT` 输出约束：映射结果必须标准化并可直接用于裁剪。来源：`C5`。
- `COMPAT` 兼容约束：需支持高分屏与不同缩放因子场景。来源：`C1 + C2 + C3 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 显示器模型基：framework.L3.M20[显示器拓扑提供器]。来源：`C1 + DISPLAY`。
- `B2` 转换计算基：framework.L3.M21[缩放因子解析器] + framework.L3.M22[坐标转换器]。来源：`C2 + TRANSFORM`。
- `B3` 多屏判定基：framework.L3.M23[多屏区域解析器]。来源：`C3 + MULTI + COMPAT`。
- `B4` 边界修正基：framework.L3.M24[区域边界修正器]。来源：`C4 + BOUNDARY`。
- `B5` 结果封装基：由结果模型、输出封装器、裁剪输入适配器构成。来源：`C5 + OUTPUT`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 显示器先行组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立显示器拓扑与缩放模型，再执行坐标转换。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`DISPLAY`。

- `R2` 转换映射组合
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：基于显示器模型完成逻辑坐标到设备像素坐标的转换与多屏归属判定。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`TRANSFORM / MULTI / COMPAT`。

- `R3` 边界修正组合
  - `R3.1` 参与基：`B2 + B3 + B4`。
  - `R3.2` 组合方式：在转换和多屏映射后叠加边界校正，形成有效裁剪区域。
  - `R3.3` 输出能力：`C2 + C3 + C4`。
  - `R3.4` 边界绑定：`BOUNDARY / MULTI`。

- `R4` 输出标准化组合
  - `R4.1` 参与基：`B2 + B4 + B5`。
  - `R4.2` 组合方式：将转换结果与边界修正结果封装为统一裁剪坐标输出。
  - `R4.3` 输出能力：`C2 + C4 + C5`。
  - `R4.4` 边界绑定：`OUTPUT / BOUNDARY`。

- `R5` 完整映射模块组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R5.2` 组合方式：将显示器描述、坐标转换、多屏映射、边界修正与结果封装整合为完整映射模块。
  - `R5.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R5.4` 边界绑定：`DISPLAY / TRANSFORM / MULTI / BOUNDARY / OUTPUT / COMPAT`。

- `R6` 禁止组合
  - `R6.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R6.2` 组合方式：忽略缩放比例直接裁剪，或无边界修正直接输出坐标的方案均无效。
  - `R6.3` 输出能力：`C6`。
  - `R6.4` 边界绑定：`TRANSFORM / BOUNDARY / COMPAT`。

## 5. 验证（Verification）

- `V1` 描述有效性：显示器边界与缩放信息必须可读取。
- `V2` 转换有效性：逻辑坐标到设备像素坐标转换必须正确。
- `V3` 多屏有效性：跨屏或高分屏场景下归属判定必须正确。
- `V4` 校正有效性：越界与非法区域必须被校正或拦截。
- `V5` 输出有效性：结果必须满足 `OUTPUT` 约束并可供裁剪模块直接使用。
- `V6` 兼容符合性：所有有效组合必须满足 `COMPAT` 约束。
- `V7` 最小必要性：移除任一 `B1~B5` 后，`V2~V6` 至少一项失败。
- `V8` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
