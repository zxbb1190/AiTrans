# 模块框架：L2-M7-屏幕帧获取器

@module

## 1. 能力声明（Capability Statement）

- `C1` 屏幕源发现能力：提供可用屏幕源识别与目标源选择能力。
- `C2` 帧获取能力：提供当前屏幕帧快照获取能力。
- `C3` 图像裁剪能力：提供基于选区的图像裁剪能力。
- `C4` 图像编码能力：提供截图结果的位图、Buffer 或 DataURL 编码能力。
- `C5` 异常回退能力：提供源不可用、读取失败与裁剪失败时的回退能力。
- `C6` 非能力项：不负责选区坐标换算逻辑。

## 2. 边界定义（Boundary / 参数）

- `SOURCE` 屏幕源约束：必须支持源发现、目标选取与不可用判定。来源：`C1`。
- `FRAME` 帧读取约束：帧快照必须具备时效性与可读取性。来源：`C2`。
- `CROP` 裁剪约束：裁剪必须接受标准选区输入并输出有效图像。来源：`C3`。
- `ENCODE` 编码约束：图像输出必须支持统一编码格式。来源：`C4`。
- `FALLBACK` 回退约束：失败场景必须可返回明确错误或回退结果。来源：`C5`。
- `PERF` 性能约束：帧获取与裁剪操作必须满足可接受响应时间。来源：`C2 + C3 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 源发现基：由源枚举器、目标匹配器、可用性检查器构成。来源：`C1 + SOURCE`。
- `B2` 帧快照基：由帧读取器、快照器、读取校验器构成。来源：`C2 + FRAME`。
- `B3` 裁剪执行基：由裁剪器、边界校验器、区域提取器构成。来源：`C3 + CROP`。
- `B4` 编码输出基：由格式转换器、编码器、输出包装器构成。来源：`C4 + ENCODE`。
- `B5` 故障回退基：由错误分类器、失败处理器、回退器与错误对象构成。来源：`C5 + FALLBACK + PERF`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 源发现先行组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立屏幕源发现与目标选择机制，再执行帧获取。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`SOURCE`。

- `R2` 快照裁剪组合
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：在目标源选定后完成帧快照获取与区域裁剪。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`SOURCE / FRAME / CROP`。

- `R3` 输出编码组合
  - `R3.1` 参与基：`B2 + B3 + B4`。
  - `R3.2` 组合方式：将帧快照与裁剪结果统一编码为标准图像输出。
  - `R3.3` 输出能力：`C2 + C3 + C4`。
  - `R3.4` 边界绑定：`FRAME / CROP / ENCODE`。

- `R4` 回退保护组合
  - `R4.1` 参与基：`B1 + B2 + B5`。
  - `R4.2` 组合方式：将源发现与帧获取过程接入故障回退机制，形成受控读取路径。
  - `R4.3` 输出能力：`C1 + C2 + C5`。
  - `R4.4` 边界绑定：`FALLBACK / PERF`。

- `R5` 完整图像提供模块组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R5.2` 组合方式：将源发现、帧获取、裁剪、编码与故障回退整合为完整屏幕帧提供模块。
  - `R5.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R5.4` 边界绑定：`SOURCE / FRAME / CROP / ENCODE / FALLBACK / PERF`。

- `R6` 禁止组合
  - `R6.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R6.2` 组合方式：无源发现直接读取，或裁剪失败后无错误分流的方案均无效。
  - `R6.3` 输出能力：`C6`。
  - `R6.4` 边界绑定：`SOURCE / CROP / FALLBACK`。

## 5. 验证（Verification）

- `V1` 源发现有效性：可用屏幕源必须可识别与选择。
- `V2` 帧获取有效性：帧快照读取必须可执行。
- `V3` 裁剪有效性：标准选区输入必须可转换为有效裁剪结果。
- `V4` 编码有效性：裁剪结果必须可编码为统一输出格式。
- `V5` 回退有效性：源不可用、读取失败与裁剪失败必须可分流处理。
- `V6` 性能达标性：读取与裁剪必须满足 `PERF` 约束。
- `V7` 最小必要性：移除任一 `B1~B5` 后，`V2~V6` 至少一项失败。
- `V8` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
