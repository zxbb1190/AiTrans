# 坐标映射引擎:ZuoBiaoYingSheYinQing

@framework

## 1. 能力声明（Capability Statement）

- `C1` 显示器描述能力：提供显示器拓扑、边界与缩放信息读取能力。
- `C2` 坐标转换能力：提供逻辑坐标到设备像素坐标的转换能力。
- `C3` 多屏映射能力：提供跨显示器场景下的选区归属与偏移映射能力。
- `C4` 边界校正能力：提供越界修正、最小区域限制与非法区域拦截能力。
- `C5` 标准输出能力：提供统一的裁剪坐标结果输出能力。
- `C6` 非能力项：不负责图像读取与图像裁剪执行。

## 2. 边界定义（Boundary / 参数）

- `DISPLAY` 显示器约束：必须可读取显示器边界、序号与缩放比例。来源：`C1`。
- `TRANSFORM` 转换约束：坐标转换必须具备确定性与可重复性。来源：`C2`。
- `MULTI` 多屏约束：跨屏场景必须正确识别目标显示器与偏移关系。来源：`C3`。
- `BOUNDARY` 边界约束：所有输出区域必须在有效屏幕范围内。来源：`C4`。
- `OUTPUT` 输出约束：映射结果必须标准化并可直接用于裁剪。来源：`C5`。
- `COMPAT` 兼容约束：需支持高分屏与不同缩放因子场景。来源：`C1 + C2 + C3 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 显示拓扑骨架：L0.M15[显示器拓扑提供器]。来源：`C1 + DISPLAY`。
- `B2` 变换校正骨架：L0.M16[缩放坐标变换器] + L0.M17[多屏区域校正器]。来源：`C2 + C4 + TRANSFORM + BOUNDARY`。
- `B3` 多屏输出骨架：L0.M17[多屏区域校正器] + L0.M8[标准输出装配器]。来源：`C3 + C5 + MULTI + OUTPUT + COMPAT`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 显示拓扑组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立显示器拓扑与缩放模型，再执行坐标转换。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`DISPLAY`。

- `R2` 变换校正组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：基于显示器拓扑完成逻辑坐标到设备像素坐标的转换与边界修正。
  - `R2.3` 输出能力：`C1 + C2 + C4`。
  - `R2.4` 边界绑定：`TRANSFORM / BOUNDARY`。

- `R3` 多屏输出组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：在变换校正结果上叠加多屏归属判定与标准输出装配，形成统一裁剪坐标结果。
  - `R3.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R3.4` 边界绑定：`MULTI / OUTPUT / COMPAT / BOUNDARY`。

- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：忽略缩放比例直接裁剪，或无边界校正与多屏输出收敛直接输出坐标的方案均无效。
  - `R4.3` 输出能力：`C6`。
  - `R4.4` 边界绑定：`TRANSFORM / BOUNDARY / COMPAT`。

## 5. 验证（Verification）

- `V1` 显示拓扑有效性：显示器边界与缩放信息必须可读取。
- `V2` 变换校正有效性：逻辑坐标到设备像素坐标转换与越界拦截必须正确。
- `V3` 多屏输出有效性：跨屏或高分屏场景下归属判定与结果输出必须正确。
- `V4` 兼容符合性：所有有效组合必须满足 `COMPAT` 约束。
- `V5` 最小必要性：移除任一 `B1~B3` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
