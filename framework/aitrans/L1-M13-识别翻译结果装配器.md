# 识别翻译结果装配器:ShiBieFanYiJieGuoZhuangPeiQi

@framework

## 1. 能力声明（Capability Statement）

- `C1` 任务结果归并能力：提供 OCR 结果、翻译结果、回退结果与任务元信息的统一归并能力。
- `C2` 字段归一能力：提供原文、译文、语言信息与来源字段的统一归一能力。
- `C3` 状态错误收敛能力：提供成功、部分成功、失败、降级状态与错误来源的统一收敛能力。
- `C4` 输出契约能力：提供面向展示层与治理层的统一结果契约输出能力。
- `C5` 非能力项：不负责具体识别与翻译执行。

## 2. 边界定义（Boundary / 参数）

- `AGGREGATE` 聚合约束：OCR、翻译、回退与任务元信息必须在统一任务标识下归并。来源：`C1`。
- `NORMALIZE` 归一约束：原文、译文、语言对、提供方标识与来源字段必须完成统一归一。来源：`C2`。
- `STATUS` 状态约束：输出状态必须显式区分成功、部分成功、失败、降级。来源：`C3`。
- `ERROR` 错误约束：错误信息必须保留来源阶段、标准错误码与可追踪引用。来源：`C3`。
- `OUTPUT` 输出约束：结果契约必须至少包含任务标识、原文、译文、语言信息、状态与错误束。来源：`C4`。
- `CONSISTENCY` 一致性约束：同一输入与同一阶段结果必须导出同一输出契约。来源：`C1 + C2 + C3 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 字段归一壳基：L0.M6[结果解析器] + L0.M8[标准输出装配器]。来源：`C1 + C2 + AGGREGATE + NORMALIZE`。
- `B2` 状态错误收敛基：L0.M7[错误分流器] + L0.M22[降级结果构造器] + L0.M8[标准输出装配器]。来源：`C3 + C4 + STATUS + ERROR + OUTPUT + CONSISTENCY`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 字段归一组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先按任务维度归并阶段结果，再完成原文、译文、语言与来源字段的统一归一。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`AGGREGATE / NORMALIZE`。

- `R2` 状态错误收敛组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在字段归一结果上叠加状态封装、错误来源保留与降级结果收敛，形成统一结果契约。
  - `R2.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R2.4` 边界绑定：`STATUS / ERROR / OUTPUT / CONSISTENCY`。

- `R3` 禁止组合
  - `R3.1` 参与基：`B1 + B2`。
  - `R3.2` 组合方式：无统一状态错误收敛直接输出阶段碎片结果，或错误信息未标注来源阶段的方案均无效。
  - `R3.3` 输出能力：`C5`。
  - `R3.4` 边界绑定：`STATUS / ERROR / OUTPUT`。

## 5. 验证（Verification）

- `V1` 归并有效性：OCR、翻译、回退与任务元信息必须可在同一任务标识下归并。
- `V2` 归一有效性：原文、译文、语言信息与来源字段必须完成统一归一。
- `V3` 收敛有效性：状态、错误来源与降级结果必须被正确收敛为统一表达。
- `V4` 输出有效性：输出契约必须满足 `OUTPUT` 约束并可被展示层直接消费。
- `V5` 一致性符合性：同一输入与同一阶段结果必须导出确定性输出。
- `V6` 最小必要性：移除任一 `B1~B2` 后，`V2~V5` 至少一项失败。
- `V7` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
