# framework L0-L3 结构收敛说明

## 1. 本次收敛原则

- 以你提供的“框架设计标准（核心）”为准，只保留五段式框架核心：能力声明、边界定义、最小可行基、基组合原则、验证。
- `B*` 优先表达结构，不再把动作清单、状态清单或实现对象直接当作高层 `B*` 主句。
- `L0/L1` 作为显式组合层，`B*` 统一改为直接内联下层模块引用。
- `L2` 仅对已经明确下钻或共享吸收到 `L3` 的模块补充显式 `L3` 引用；其余 `L2` 继续保持叶子自足，不强行下钻。
- `L3` 统一收敛为单叶子基模块，每个文件只保留一个 `B1`，避免继续裂成实现级文档。

## 2. 当前层级语义

- `L0`：框架总根，仅承接五大能力面，不再写实现性细节。
- `L1`：领域组合层，显式组合 `L2` 模块。
- `L2`：控制面与领域模块层；部分模块继续显式下钻到 `L3`，其余保留为叶子模块。
- `L3`：叶子结构层，只保留单一稳定结构单元，不再继续机械下钻到 `L4`。

## 3. 本次具体收敛结果

- `L0-M0` 与全部 `L1` 模块的 `B*` 已统一改成 `framework.Lx.My[...]` 形式。
- 继续下钻或共享吸收到 `L3` 的 `L2` 模块，已把对应 `L3` 引用直接写入 `B*` 主句。
- 全部 `L3-M0 ~ L3-M59` 文档已收敛成五段式标准模块，并统一为单叶子基表达。
- `L2-M0 / M1 / M2 / M4 / M7 / M9 / M16 / M17 / M21` 继续停在 `L2`，因为再拆只会退化为实现对象、函数集合或 UI 子组件树。

## 4. 后续建议

- 下一步如果继续收敛，不应再机械增加层级，而应优先检查同层模块是否在重复同一拆分轴。
- 若发现多个 `L3` 只是重复同一组织形式，应优先合并模块、减少 `B*` 数量，而不是继续下钻。
