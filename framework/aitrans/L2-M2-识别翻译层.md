# 识别翻译层:RecognitionTranslationLayer

@framework

## 1. 能力声明（Capability Statement）

- `C1` 任务承接能力：承接截图结果并建立可追踪的识别翻译任务上下文。
- `C2` 文本识别能力：提供 OCR Provider 执行、文字提取与识别元信息回收能力。
- `C3` 翻译转换能力：提供翻译 Provider 执行、译文生成与翻译元信息回收能力。
- `C4` 恢复执行能力：提供失败分类、超时预算、重试与候选引擎切换能力。
- `C5` 结果契约能力：提供原文、译文、语言信息、阶段状态与错误来源的统一结果契约。
- `C6` 非能力项：不负责截图采集与 UI 结果渲染。

## 2. 边界定义（Boundary / 参数）

- `TASK` 任务约束：每次处理必须绑定唯一任务标识、阶段游标与独立上下文。来源：`C1`。
- `OCR` 识别约束：OCR Provider 必须经统一适配边界访问，并返回可归一的文本片段与识别元信息。来源：`C2`。
- `TRANS` 翻译约束：翻译 Provider 必须经统一适配边界访问，并返回可归一的译文与翻译元信息。来源：`C3`。
- `FALLBACK` 回退约束：失败场景必须支持分类、超时预算、重试上限与候选引擎切换。来源：`C4`。
- `RESULT` 输出约束：输出结果必须同时携带原文、译文、语言信息、阶段状态与错误来源。来源：`C5`。
- `PERF` 性能约束：链路必须显式定义阶段超时预算与总时延预算。来源：`C2 + C3 + C4`。
- `SEC` 安全约束：离开任务上下文的图像、文本与密钥暴露面必须最小化，输出中不得泄露原始敏感配置。来源：`C3 + C4 + C5`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 任务承接基：L1.M10[识别翻译任务编排器]。来源：`C1 + TASK`。
- `B2` 提供方执行基：L1.M11[提供方执行适配器]。来源：`C2 + C3 + OCR + TRANS`。
- `B3` 恢复执行基：L1.M12[回退策略控制器]。来源：`C4 + FALLBACK + PERF`。
- `B4` 结果契约基：L1.M13[识别翻译结果装配器]。来源：`C5 + RESULT + SEC`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 任务承接组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立任务上下文、阶段游标与执行入口，再接入下游执行链路。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`TASK`。

- `R2` 提供方执行组合
  - `R2.1` 参与基：`B1 + B2`。
- `R2.2` 组合方式：在任务承接基础上绑定统一提供方执行壳，在壳内分别承接 OCR 与翻译 Provider，形成主处理链路。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`OCR / TRANS / PERF`。

- `R3` 恢复执行组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将失败分类、超时预算、重试与候选切换叠加到提供方执行链路，形成可恢复执行路径。
  - `R3.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R3.4` 边界绑定：`FALLBACK / PERF / SEC`。

- `R4` 结果契约组合
  - `R4.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R4.2` 组合方式：将任务链路结果收敛为统一契约，输出面向展示层与治理层的稳定结果结构。
  - `R4.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R4.4` 边界绑定：`RESULT / OCR / TRANS / FALLBACK / SEC`。

- `R5` 禁止组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R5.2` 组合方式：绕过 Provider 执行边界直接耦合具体引擎，或跳过结果契约直接输出阶段碎片结果的方案均无效。
  - `R5.3` 输出能力：`C6`。
  - `R5.4` 边界绑定：`OCR / TRANS / RESULT / SEC`。

## 5. 验证（Verification）

- `V1` 任务一致性：每次处理都必须建立唯一任务标识、阶段游标与独立上下文。
- `V2` 提供方执行有效性：OCR 与翻译调用必须均经统一适配边界进入主链路。
- `V3` 恢复执行有效性：失败分类、超时预算、重试上限与候选切换必须覆盖关键失败场景。
- `V4` 结果契约有效性：输出结果必须同时包含 `RESULT` 约束要求的核心字段。
- `V5` 性能达标性：阶段超时预算与总时延预算必须被显式声明并可校验。
- `V6` 安全符合性：外部服务调用与结果输出必须满足 `SEC` 约束。
- `V7` 最小必要性：移除任一 `B1~B4` 后，`V2~V6` 至少一项失败。
- `V8` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
