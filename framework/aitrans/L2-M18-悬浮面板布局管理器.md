# 模块框架：L2-M18-悬浮面板布局管理器

@module

## 1. 能力声明（Capability Statement）

- `C1` 面板定位能力：提供结果悬浮面板初始定位与相对锚定能力。
- `C2` 布局控制能力：提供面板尺寸、边界约束与布局调整能力。
- `C3` 拖拽移动能力：提供面板拖拽、移动与释放定位能力。
- `C4` 显隐控制能力：提供显示、隐藏、自动隐藏与恢复显示能力。
- `C5` 避障适配能力：提供屏幕边界、遮挡风险与多窗口环境下的避障布局能力。
- `C6` 非能力项：不负责面板内部内容渲染与交互动作执行。

## 2. 边界定义（Boundary / 参数）

- `POSITION` 定位约束：面板必须具备明确初始定位规则。来源：`C1`。
- `LAYOUT` 布局约束：面板布局必须满足尺寸、边界与最小显示要求。来源：`C2`。
- `DRAG` 拖拽约束：拖拽过程中必须保持位置更新与边界校正。来源：`C3`。
- `VISIBILITY` 显隐约束：显示与隐藏必须受统一状态控制。来源：`C4`。
- `AVOID` 避障约束：面板位置必须尽量避免超出屏幕或遮挡关键区域。来源：`C5`。
- `UX` 体验约束：浮层展示必须低打扰、可恢复、可控。来源：`C1 + C2 + C3 + C4 + C5`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 定位计算基：framework.L3.M32[面板锚点定位器]。来源：`C1 + POSITION`。
- `B2` 布局约束基：framework.L3.M33[面板布局约束引擎]。来源：`C2 + LAYOUT`。
- `B3` 拖拽移动基：framework.L3.M34[面板拖拽控制器]。来源：`C3 + DRAG`。
- `B4` 显隐控制基：framework.L3.M35[面板显隐控制器]。来源：`C4 + VISIBILITY`。
- `B5` 避障适配基：framework.L3.M36[面板避障引擎]。来源：`C5 + AVOID + UX`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 定位先行组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立悬浮面板初始定位规则，再接入布局与显隐能力。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`POSITION`。

- `R2` 定位布局组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：将定位计算与布局约束绑定，形成可显示的面板布局结果。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`POSITION / LAYOUT`。

- `R3` 拖拽显隐组合
  - `R3.1` 参与基：`B2 + B3 + B4`。
  - `R3.2` 组合方式：在布局约束上接入拖拽与显隐控制，形成可操作悬浮面板。
  - `R3.3` 输出能力：`C2 + C3 + C4`。
  - `R3.4` 边界绑定：`DRAG / VISIBILITY / UX`。

- `R4` 避障适配组合
  - `R4.1` 参与基：`B1 + B2 + B5`。
  - `R4.2` 组合方式：将初始定位与布局控制接入边界避让与遮挡规避能力。
  - `R4.3` 输出能力：`C1 + C2 + C5`。
  - `R4.4` 边界绑定：`AVOID / LAYOUT / UX`。

- `R5` 完整浮层布局模块组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R5.2` 组合方式：将面板定位、布局控制、拖拽移动、显隐控制与避障适配整合为完整浮层布局模块。
  - `R5.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R5.4` 边界绑定：`POSITION / LAYOUT / DRAG / VISIBILITY / AVOID / UX`。

- `R6` 禁止组合
  - `R6.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R6.2` 组合方式：无边界约束直接显示面板，或无避障策略直接固定覆盖关键区域的方案均无效。
  - `R6.3` 输出能力：`C6`。
  - `R6.4` 边界绑定：`LAYOUT / AVOID / UX`。

## 5. 验证（Verification）

- `V1` 定位有效性：面板初始定位与相对锚定必须可执行。
- `V2` 布局有效性：尺寸、边界与最小显示要求必须满足。
- `V3` 拖拽有效性：面板拖拽、移动与释放定位必须可执行。
- `V4` 显隐有效性：显示、隐藏、自动隐藏与恢复显示必须可执行。
- `V5` 避障有效性：屏幕边界与遮挡风险必须被识别并规避。
- `V6` 体验达标性：面板布局必须满足 `UX` 约束。
- `V7` 最小必要性：移除任一 `B1~B5` 后，`V2~V6` 至少一项失败。
- `V8` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
