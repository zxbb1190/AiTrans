# 遮罩选区控制器:ZheZhaoXuanQuKongZhiQi

@framework

## 1. 能力声明（Capability Statement）

- `C1` 遮罩显示能力：提供截图遮罩层创建、显示与隐藏能力。
- `C2` 选区拖拽能力：提供鼠标拖拽选区、调整选区与确认选区能力。
- `C3` 可视反馈能力：提供选区边框、尺寸提示与交互反馈能力。
- `C4` 取消回退能力：提供 ESC 取消、空白点击退出与异常回退能力。
- `C5` 事件上报能力：提供选区变更、确认与取消事件上报能力。
- `C6` 非能力项：不负责实际图像裁剪与像素处理。

## 2. 边界定义（Boundary / 参数）

- `OVERLAY` 遮罩约束：遮罩必须全屏覆盖、低延迟显示且可受控关闭。来源：`C1`。
- `SELECTION` 选区约束：选区必须支持开始、拖拽、结束与调整。来源：`C2`。
- `FEEDBACK` 反馈约束：交互过程必须具备清晰的视觉反馈。来源：`C3`。
- `CANCEL` 取消约束：取消操作必须立即中断并释放遮罩。来源：`C4`。
- `EVENT` 事件约束：选区交互事件必须可上报与追踪。来源：`C5`。
- `PERF` 性能约束：拖拽与反馈过程必须低延迟。来源：`C1 + C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 交互承载骨架：L0.M13[选区交互承载器]。来源：`C1 + C2 + OVERLAY + SELECTION`。
- `B2` 反馈回退骨架：L0.M14[选区反馈回退器]。来源：`C3 + C4 + FEEDBACK + CANCEL + PERF`。
- `B3` 事件上报骨架：L0.M3[事件总线] + L0.M9[审计追踪器]。来源：`C5 + EVENT`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 交互承载组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立遮罩宿主、输入跟踪与区域建模，形成统一选区交互承接面。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`OVERLAY / SELECTION`。

- `R2` 反馈回退组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在交互承接面上绑定反馈绘制与取消回退，形成受控选区交互闭环。
  - `R2.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R2.4` 边界绑定：`FEEDBACK / CANCEL / PERF`。

- `R3` 上报同步组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将选区变化、确认与取消动作绑定到事件上报链路，形成对外同步能力。
  - `R3.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R3.4` 边界绑定：`EVENT / FEEDBACK / CANCEL`。

- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：无遮罩交互承载直接绘制选区，或无反馈回退机制的拖拽方案均无效。
  - `R4.3` 输出能力：`C6`。
  - `R4.4` 边界绑定：`OVERLAY / CANCEL / PERF`。

## 5. 验证（Verification）

- `V1` 交互承载有效性：遮罩创建、显示关闭、拖拽调整与确认必须可执行。
- `V2` 反馈回退有效性：边框、尺寸与光标反馈必须清晰可见，ESC 与空白退出必须立即中断交互。
- `V3` 上报同步有效性：选区变化与确认取消事件必须可上报并可追踪。
- `V4` 性能达标性：拖拽与反馈必须满足 `PERF` 约束。
- `V5` 最小必要性：移除任一 `B1~B3` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
