# 模块框架：L2-M9-截图结果标准化器

@module

## 1. 能力声明（Capability Statement）

- `C1` 结果建模能力：提供截图结果对象与元数据模型构建能力。
- `C2` 数据统一能力：提供图像数据、坐标信息与来源信息统一封装能力。
- `C3` 结果校验能力：提供截图结果完整性与合法性校验能力。
- `C4` 缓存句柄能力：提供临时缓存标识、引用管理与释放能力。
- `C5` 下游适配能力：提供面向 OCR 与后续模块的标准输入适配能力。
- `C6` 非能力项：不负责图像生成与业务处理逻辑。

## 2. 边界定义（Boundary / 参数）

- `MODEL` 模型约束：截图结果必须具有统一数据模型。来源：`C1`。
- `UNIFY` 统一约束：图像与元信息必须在单一结构中表达。来源：`C2`。
- `VALIDATE` 校验约束：结果必须在输出前完成完整性校验。来源：`C3`。
- `CACHE` 缓存约束：缓存句柄必须可创建、查询与释放。来源：`C4`。
- `ADAPTER` 适配约束：输出结果必须被 OCR/翻译链路直接消费。来源：`C5`。
- `SEC` 安全约束：临时数据与缓存引用需受控管理。来源：`C4 + C5`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 结果模型基：由结果对象、元数据模型、字段映射器构成。来源：`C1 + MODEL`。
- `B2` 数据封装基：由图像封装器、坐标封装器、来源信息封装器构成。来源：`C2 + UNIFY`。
- `B3` 完整性校验基：由字段校验器、格式校验器、空值检查器构成。来源：`C3 + VALIDATE`。
- `B4` 缓存引用基：由句柄生成器、缓存索引器、释放器构成。来源：`C4 + CACHE + SEC`。
- `B5` 下游适配基：由 OCR 输入适配器、任务输入构造器、标准输出器构成。来源：`C5 + ADAPTER`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 建模先行组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立统一截图结果模型，再接入封装与校验。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`MODEL`。

- `R2` 封装校验组合
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：将图像与元信息封装到统一结果中，并在输出前完成合法性校验。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`UNIFY / VALIDATE`。

- `R3` 缓存适配组合
  - `R3.1` 参与基：`B2 + B4 + B5`。
  - `R3.2` 组合方式：将封装后的图像结果绑定缓存句柄与下游输入适配，形成可传递结果结构。
  - `R3.3` 输出能力：`C2 + C4 + C5`。
  - `R3.4` 边界绑定：`CACHE / ADAPTER / SEC`。

- `R4` 校验适配组合
  - `R4.1` 参与基：`B3 + B5`。
  - `R4.2` 组合方式：以合法性校验结果作为下游适配前置条件，形成受控输出路径。
  - `R4.3` 输出能力：`C3 + C5`。
  - `R4.4` 边界绑定：`VALIDATE / ADAPTER`。

- `R5` 完整结果模块组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R5.2` 组合方式：将结果建模、数据统一、合法性校验、缓存引用与下游适配整合为完整结果标准化模块。
  - `R5.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R5.4` 边界绑定：`MODEL / UNIFY / VALIDATE / CACHE / ADAPTER / SEC`。

- `R6` 禁止组合
  - `R6.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R6.2` 组合方式：无统一结果模型直接向下游传递原始数据，或跳过完整性校验直接输出的方案均无效。
  - `R6.3` 输出能力：`C6`。
  - `R6.4` 边界绑定：`MODEL / VALIDATE / ADAPTER`。

## 5. 验证（Verification）

- `V1` 建模有效性：截图结果对象与元数据模型必须统一可用。
- `V2` 封装有效性：图像、坐标与来源信息必须统一封装。
- `V3` 校验有效性：输出前必须完成合法性与完整性校验。
- `V4` 缓存有效性：缓存句柄必须可创建、可查询、可释放。
- `V5` 适配有效性：结果必须能直接供 OCR 与后续模块消费。
- `V6` 安全符合性：临时数据与缓存引用必须满足 `SEC` 约束。
- `V7` 最小必要性：移除任一 `B1~B5` 后，`V2~V6` 至少一项失败。
- `V8` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
