# 模块框架：L2-M22-可观测性中心

@module

## 1. 能力声明（Capability Statement）

- `C1` 日志记录能力：提供关键链路日志记录、分级与结构化输出能力。
- `C2` 事件埋点能力：提供业务事件埋点、阶段事件记录与统计聚合能力。
- `C3` 错误采集能力：提供异常捕获、错误聚合与失败追踪能力。
- `C4` 链路关联能力：提供任务、会话与调用链路关联标识能力。
- `C5` 诊断输出能力：提供调试定位、问题回放与诊断上下文输出能力。
- `C6` 非能力项：不负责业务逻辑恢复与具体告警系统实现。

## 2. 边界定义（Boundary / 参数）

- `LOG` 日志约束：日志必须具备级别、结构化字段与输出策略。来源：`C1`。
- `METRIC` 事件约束：关键事件必须可埋点、统计与聚合。来源：`C2`。
- `ERROR` 错误约束：错误必须具备捕获、归类与上下文绑定能力。来源：`C3`。
- `TRACE` 链路约束：关键调用链路必须具备统一关联标识。来源：`C4`。
- `DIAG` 诊断约束：问题定位必须可输出足够诊断上下文。来源：`C5`。
- `PRIVACY` 隐私约束：日志与诊断信息不得泄露不必要敏感数据。来源：`C1 + C3 + C5`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 日志输出基：framework.L3.M45[结构化日志器]。来源：`C1 + LOG`。
- `B2` 事件埋点基：framework.L3.M46[事件指标采集器]。来源：`C2 + METRIC`。
- `B3` 错误采集基：framework.L3.M47[错误采集中心]。来源：`C3 + ERROR`。
- `B4` 链路关联基：framework.L3.M48[链路上下文管理器]。来源：`C4 + TRACE`。
- `B5` 诊断上下文基：framework.L3.M49[诊断快照提供器]。来源：`C5 + DIAG + PRIVACY`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 日志先行组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立统一日志级别与结构化输出能力，再接入事件与错误采集。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`LOG`。

- `R2` 事件错误组合
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：将日志输出、事件埋点与错误采集绑定，形成基础观测面。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`METRIC / ERROR / LOG`。

- `R3` 链路诊断组合
  - `R3.1` 参与基：`B3 + B4 + B5`。
  - `R3.2` 组合方式：将错误上下文与链路标识、诊断快照绑定，形成定位闭环。
  - `R3.3` 输出能力：`C3 + C4 + C5`。
  - `R3.4` 边界绑定：`TRACE / DIAG / PRIVACY`。

- `R4` 关联统计组合
  - `R4.1` 参与基：`B2 + B4`。
  - `R4.2` 组合方式：将事件埋点与链路标识绑定，形成跨阶段统计与分析能力。
  - `R4.3` 输出能力：`C2 + C4`。
  - `R4.4` 边界绑定：`METRIC / TRACE`。

- `R5` 完整观测中心模块组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R5.2` 组合方式：将日志记录、事件埋点、错误采集、链路关联与诊断输出整合为完整观测中心模块。
  - `R5.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R5.4` 边界绑定：`LOG / METRIC / ERROR / TRACE / DIAG / PRIVACY`。

- `R6` 禁止组合
  - `R6.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R6.2` 组合方式：无链路标识直接拼接跨阶段日志，或诊断信息不做隐私裁剪直接输出的方案均无效。
  - `R6.3` 输出能力：`C6`。
  - `R6.4` 边界绑定：`TRACE / DIAG / PRIVACY`。

## 5. 验证（Verification）

- `V1` 日志有效性：日志级别、结构化字段与输出策略必须正确执行。
- `V2` 事件有效性：关键业务事件必须可埋点、统计与聚合。
- `V3` 错误有效性：异常必须被正确捕获、归类与追踪。
- `V4` 链路有效性：关键流程必须具备统一关联标识。
- `V5` 诊断有效性：问题定位必须可输出足够诊断上下文且不泄露敏感数据。
- `V6` 隐私符合性：所有有效组合必须满足 `PRIVACY` 约束。
- `V7` 最小必要性：移除任一 `B1~B5` 后，`V2~V6` 至少一项失败。
- `V8` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
