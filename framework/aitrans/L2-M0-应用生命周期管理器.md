# 模块框架：L2-M0-应用生命周期管理器

@module

## 1. 能力声明（Capability Statement）

- `C1` 启动管理能力：提供应用启动、初始化编排与就绪状态建立能力。
- `C2` 单实例管理能力：提供单实例锁获取、重复启动识别与实例切换协调能力。
- `C3` 退出清理能力：提供退出前资源释放、状态持久化与清理执行能力。
- `C4` 异常恢复能力：提供未捕获异常监听、崩溃恢复与安全退出能力。
- `C5` 生命周期事件分发能力：提供生命周期事件注册、广播与监听协调能力。
- `C6` 非能力项：不负责托盘交互、窗口业务调度与权限探测细节。

## 2. 边界定义（Boundary / 参数）

- `BOOT` 启动约束：必须支持冷启动、热激活与就绪回调。来源：`C1`。
- `INSTANCE` 单实例约束：必须保证单实例唯一性并支持重复启动处理。来源：`C2`。
- `EXIT` 退出约束：必须支持主动退出、被动退出与异常退出三类路径。来源：`C3`。
- `RECOVERY` 恢复约束：必须对异常事件具备记录、分流与恢复策略。来源：`C4`。
- `EVENT` 生命周期事件约束：生命周期事件必须可注册、可取消、可追踪。来源：`C5`。
- `RUNTIME` 运行边界约束：生命周期状态源应由主进程统一管理。来源：`C1 + C2 + C3 + C4 + C5`。
- `COMPAT` 兼容约束：生命周期行为在目标平台上应保持一致语义。来源：`C1 + C3 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 启动编排基：由启动器、初始化队列、就绪标记器与失败中断器构成。来源：`C1 + BOOT`。
- `B2` 单实例控制基：由实例锁获取器、重复实例检测器与实例切换处理器构成。来源：`C2 + INSTANCE`。
- `B3` 退出处理基：由退出控制器、资源释放器、状态持久器与清理执行器构成。来源：`C3 + EXIT`。
- `B4` 异常恢复基：由异常监听器、恢复策略器、崩溃记录器与安全退出器构成。来源：`C4 + RECOVERY`。
- `B5` 事件总线基：由生命周期事件注册器、订阅器、分发器与注销器构成。来源：`C5 + EVENT + RUNTIME`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 启动先行组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先完成启动编排与就绪状态建立，再接入其他生命周期能力。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`BOOT / RUNTIME`。

- `R2` 单实例约束组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在启动编排基础上叠加单实例控制，形成唯一实例运行语义。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`BOOT / INSTANCE / COMPAT`。

- `R3` 退出恢复组合
  - `R3.1` 参与基：`B3 + B4`。
  - `R3.2` 组合方式：将退出处理与异常恢复绑定，形成完整终止与恢复路径。
  - `R3.3` 输出能力：`C3 + C4`。
  - `R3.4` 边界绑定：`EXIT / RECOVERY`。

- `R4` 生命周期广播组合
  - `R4.1` 参与基：`B1 + B3 + B5`。
  - `R4.2` 组合方式：在启动与退出链路上绑定生命周期事件分发，形成统一事件语义。
  - `R4.3` 输出能力：`C1 + C3 + C5`。
  - `R4.4` 边界绑定：`EVENT / RUNTIME`。

- `R5` 完整生命周期组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R5.2` 组合方式：将启动、单实例、退出、恢复与事件分发整合为完整生命周期管理模块。
  - `R5.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R5.4` 边界绑定：`BOOT / INSTANCE / EXIT / RECOVERY / EVENT / RUNTIME / COMPAT`。

- `R6` 禁止组合
  - `R6.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R6.2` 组合方式：绕过单实例控制直接启动，或无退出清理直接强制终止的方案均无效。
  - `R6.3` 输出能力：`C6`。
  - `R6.4` 边界绑定：`INSTANCE / EXIT / RUNTIME`。

## 5. 验证（Verification）

- `V1` 启动有效性：应用启动、初始化与就绪标记必须完整可执行。
- `V2` 单实例有效性：重复启动必须被识别并按规则处理。
- `V3` 退出有效性：退出前清理与状态持久化必须执行。
- `V4` 恢复有效性：异常监听与恢复分流必须覆盖关键崩溃路径。
- `V5` 事件有效性：生命周期事件必须可注册、可分发、可追踪。
- `V6` 边界符合性：所有有效组合必须满足 `RUNTIME / COMPAT` 约束。
- `V7` 最小必要性：移除任一 `B1~B5` 后，`V2~V6` 至少一项失败。
- `V8` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
