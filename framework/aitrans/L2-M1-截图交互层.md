# 截图交互层:ScreenshotInteractionLayer

@framework

## 1. 能力声明（Capability Statement）

- `C1` 截图入口承接能力：承接来自宿主模块的截图触发并建立当前会话上下文。
- `C2` 选区交互能力：提供遮罩绘制、鼠标拖拽、区域调整与取消操作能力。
- `C3` 屏幕采集能力：提供屏幕源选择、帧获取、像素裁剪与图像输出能力。
- `C4` 多屏映射能力：提供多显示器、缩放比例与坐标换算能力。
- `C5` 截图结果标准化能力：提供截图元数据、位图对象与后续处理输入格式统一能力。
- `C6` 非能力项：不负责 OCR 文本识别、翻译与结果展示。

## 2. 边界定义（Boundary / 参数）

- `SESSION` 截图会话约束：每次截图必须生成唯一会话并支持完成、取消、失败三类状态。来源：`C1`。
- `OVERLAY` 交互遮罩约束：遮罩层必须全屏、低延迟、可取消且不干扰选区反馈。来源：`C2`。
- `SOURCE` 屏幕源约束：必须支持屏幕源识别、目标屏选取与当前帧读取。来源：`C3`。
- `COORD` 坐标换算约束：必须正确处理逻辑坐标、设备像素比与多屏相对坐标。来源：`C4`。
- `IMAGE` 图像输出约束：截图结果必须统一为可传递、可序列化或可缓存的图像对象。来源：`C5`。
- `PERF` 性能约束：需显式声明遮罩显示、拖拽反馈、坐标换算与截图输出的 `p95` 时延预算。来源：`C2 + C3 + C4 + C5`。
- `COMPAT` 兼容约束：需支持目标平台下的多屏与高分屏场景。来源：`C3 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 会话承接基：L1.M5[交互会话管理器]。来源：`C1 + SESSION`。
- `B2` 交互采集基：L1.M6[遮罩选区控制器] + L1.M7[屏幕帧获取器]。来源：`C2 + C3 + OVERLAY + SOURCE + PERF`。
- `B3` 坐标校正基：L1.M8[坐标映射引擎]。来源：`C4 + COORD + COMPAT`。
- `B4` 结果标准化基：L1.M9[截图结果标准化器]。来源：`C5 + IMAGE + SESSION`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 会话承接组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立截图会话状态与上下文，再进入遮罩交互与图像采集阶段。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`SESSION`。

- `R2` 交互采集组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：将截图会话与选区交互、屏幕帧采集绑定，形成从拖拽到原始截图结果的执行闭环。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`OVERLAY / SOURCE / PERF`。

- `R3` 坐标校正组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将选区逻辑坐标与实际设备像素映射绑定，保证多屏与缩放场景下的裁剪正确性。
  - `R3.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R3.4` 边界绑定：`COORD / COMPAT / PERF`。

- `R4` 输出标准化组合
  - `R4.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R4.2` 组合方式：在采集和坐标校正后统一截图输出格式，形成下游可直接消费的标准结果。
  - `R4.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R4.4` 边界绑定：`IMAGE / SESSION / COORD / PERF / COMPAT`。

- `R5` 禁止组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R5.2` 组合方式：跳过会话承接直接输出截图结果，或跳过坐标校正直接裁剪图像的方案均无效。
  - `R5.3` 输出能力：`C6`。
  - `R5.4` 边界绑定：`SESSION / COORD / IMAGE / COMPAT`。

## 5. 验证（Verification）

- `V1` 会话承接有效性：截图会话必须具备创建、取消、完成、失败的明确状态流转。
- `V2` 交互采集有效性：遮罩显示、拖拽选区、取消确认、屏幕源读取与帧获取链路必须全链路可执行。
- `V3` 坐标校正有效性：多屏与高分屏场景下的坐标换算结果必须正确。
- `V4` 输出标准化有效性：截图结果必须满足 `IMAGE` 约束并可被下游模块直接消费。
- `V5` 性能达标性：交互反馈与截图输出必须满足 `PERF` 约束。
- `V6` 最小必要性：移除任一 `B1~B4` 后，`V2~V5` 至少一项失败。
- `V7` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
