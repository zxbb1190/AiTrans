# 模块框架：L2-M6-遮罩选区控制器

@module

## 1. 能力声明（Capability Statement）

- `C1` 遮罩显示能力：提供截图遮罩层创建、显示与隐藏能力。
- `C2` 选区拖拽能力：提供鼠标拖拽选区、调整选区与确认选区能力。
- `C3` 可视反馈能力：提供选区边框、尺寸提示与交互反馈能力。
- `C4` 取消回退能力：提供 ESC 取消、空白点击退出与异常回退能力。
- `C5` 事件上报能力：提供选区变更、确认与取消事件上报能力。
- `C6` 非能力项：不负责实际图像裁剪与像素处理。

## 2. 边界定义（Boundary / 参数）

- `OVERLAY` 遮罩约束：遮罩必须全屏覆盖、低延迟显示且可受控关闭。来源：`C1`。
- `SELECTION` 选区约束：选区必须支持开始、拖拽、结束与调整。来源：`C2`。
- `FEEDBACK` 反馈约束：交互过程必须具备清晰的视觉反馈。来源：`C3`。
- `CANCEL` 取消约束：取消操作必须立即中断并释放遮罩。来源：`C4`。
- `EVENT` 事件约束：选区交互事件必须可上报与追踪。来源：`C5`。
- `PERF` 性能约束：拖拽与反馈过程必须低延迟。来源：`C1 + C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 遮罩容器基：framework.L3.M15[遮罩生命周期管理器]。来源：`C1 + OVERLAY`。
- `B2` 选区交互基：framework.L3.M16[选区输入跟踪器] + framework.L3.M17[选区区域模型]。来源：`C2 + SELECTION`。
- `B3` 反馈绘制基：framework.L3.M18[选区反馈渲染器]。来源：`C3 + FEEDBACK + PERF`。
- `B4` 取消回退基：framework.L3.M19[选区确认取消控制器]。来源：`C4 + CANCEL`。
- `B5` 事件上报基：由事件发布器、变更节流器、确认通知器与取消通知器构成。来源：`C5 + EVENT`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 遮罩先行组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立遮罩显示与关闭能力，再接入交互与反馈能力。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`OVERLAY`。

- `R2` 交互反馈组合
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：在遮罩容器上绑定拖拽交互与反馈绘制，形成选区交互闭环。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`SELECTION / FEEDBACK / PERF`。

- `R3` 取消中断组合
  - `R3.1` 参与基：`B1 + B2 + B4`。
  - `R3.2` 组合方式：将选区交互与取消回退绑定，形成受控中断路径。
  - `R3.3` 输出能力：`C1 + C2 + C4`。
  - `R3.4` 边界绑定：`CANCEL / OVERLAY`。

- `R4` 上报同步组合
  - `R4.1` 参与基：`B2 + B3 + B5`。
  - `R4.2` 组合方式：将选区变化与视觉反馈绑定到事件上报链路，形成对外同步能力。
  - `R4.3` 输出能力：`C2 + C3 + C5`。
  - `R4.4` 边界绑定：`EVENT / FEEDBACK`。

- `R5` 完整选区模块组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R5.2` 组合方式：将遮罩显示、选区拖拽、反馈绘制、取消回退与事件上报整合为完整选区控制模块。
  - `R5.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R5.4` 边界绑定：`OVERLAY / SELECTION / FEEDBACK / CANCEL / EVENT / PERF`。

- `R6` 禁止组合
  - `R6.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R6.2` 组合方式：无遮罩容器直接绘制选区，或无取消回退机制的拖拽方案均无效。
  - `R6.3` 输出能力：`C6`。
  - `R6.4` 边界绑定：`OVERLAY / CANCEL / PERF`。

## 5. 验证（Verification）

- `V1` 遮罩有效性：遮罩创建、显示与关闭必须可执行。
- `V2` 选区有效性：拖拽、调整与确认必须可执行。
- `V3` 反馈有效性：边框、尺寸与光标反馈必须清晰可见。
- `V4` 取消有效性：ESC 与空白退出必须立即中断交互。
- `V5` 上报有效性：选区变化与确认取消事件必须可上报。
- `V6` 性能达标性：拖拽与反馈必须满足 `PERF` 约束。
- `V7` 最小必要性：移除任一 `B1~B5` 后，`V2~V6` 至少一项失败。
- `V8` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
