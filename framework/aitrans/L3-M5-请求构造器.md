# 模块框架：L3-M5-请求构造器

@module

## 1. 能力声明（Capability Statement）

- `C1` 叶子结构承接能力：统一完成输入适配、参数装配、请求模型归一与调用前约束绑定。
- `C2` 适用承接能力：在 `L2-M11-文字识别提供方适配器`、`L2-M12-翻译提供方适配器` 边界内作为 `通用可复用` 结构单元被稳定复用。
- `C3` 非能力项：只负责请求表达，不负责远端调用、结果解析与错误处理。

## 2. 边界定义（Boundary / 参数）

- `PARENT` 上级适用边界：`L2-M11-文字识别提供方适配器`、`L2-M12-翻译提供方适配器`
- `REUSE` 复用边界：`通用可复用`
- `STRUCT` 结构边界：只负责请求表达，不负责远端调用、结果解析与错误处理。
- `STOP` 停止边界：继续下钻只会变成字段映射器与参数默认值实现，已不属于框架级子层。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 请求构造器结构基：由该叶子模块的承载壳、输入绑定位与输出绑定位构成。来源：`C1 + C2 + STRUCT`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 单基自足组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：在满足 `PARENT / REUSE / STRUCT` 边界时，`B1` 独立成立并向上层输出单一叶子结构。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`PARENT / REUSE / STRUCT`。

- `R2` 停止下钻组合
  - `R2.1` 参与基：`B1`。
  - `R2.2` 组合方式：若继续把 `B1` 细化为纯实现类、函数集合或 `Handler / Builder / Mapper`，则该展开无效。
  - `R2.3` 输出能力：`C3`。
  - `R2.4` 边界绑定：`STOP`。

## 5. 验证（Verification）

- `V1` 叶子成立性：`B1` 必须在声明边界内作为独立叶子结构成立。
- `V2` 复用成立性：该模块必须可被上级模块直接引用，而不依赖更细的本层子模块。
- `V3` 非能力项成立性：越过 `只负责请求表达，不负责远端调用、结果解析与错误处理。` 的扩张必须判为结构越界。
- `V4` 停止性：继续下钻后若仅剩实现对象或函数级单元，应判为不适用。
- `V5` 结论表达：逐条输出 `R1/R2 -> C* / 边界` 的通过或失败结论。
