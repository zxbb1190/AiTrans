# 模块框架：L2-M16-结果视图渲染器

@module

## 1. 能力声明（Capability Statement）

- `C1` 视图装配能力：提供原文区、译文区、状态区与错误区的视图装配能力。
- `C2` 内容渲染能力：提供文本内容、语言标签与辅助信息渲染能力。
- `C3` 视图切换能力：提供成功态、失败态、空结果态与加载态视图切换能力。
- `C4` 局部刷新能力：提供原文、译文与状态区域的局部刷新能力。
- `C5` 渲染一致性能力：提供视图结构、字段绑定与显示规则一致性能力。
- `C6` 非能力项：不负责交互动作执行与悬浮布局控制。

## 2. 边界定义（Boundary / 参数）

- `VIEW` 视图约束：视图必须包含原文、译文、状态与错误表达区。来源：`C1`。
- `CONTENT` 内容约束：展示内容必须正确绑定到结果字段。来源：`C2`。
- `SWITCH` 切换约束：不同展示状态必须可切换且互斥。来源：`C3`。
- `REFRESH` 刷新约束：局部更新不得破坏整体视图一致性。来源：`C4`。
- `CONSISTENCY` 一致性约束：字段绑定与显示规则必须确定可复现。来源：`C5`。
- `UX` 体验约束：渲染结果必须清晰、稳定、低干扰。来源：`C1 + C2 + C3 + C4 + C5`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 视图结构基：由原文面板、译文面板、状态区域与错误区域构成。来源：`C1 + VIEW`。
- `B2` 内容绑定基：由字段绑定器、文本渲染器、语言标签渲染器与辅助信息渲染器构成。来源：`C2 + CONTENT`。
- `B3` 视图切换基：由状态视图选择器、条件渲染器、互斥切换器构成。来源：`C3 + SWITCH`。
- `B4` 局部刷新基：由差量更新器、区域刷新器、一致性校验器构成。来源：`C4 + REFRESH`。
- `B5` 渲染规则基：由显示规则器、结构校验器、字段映射器与体验约束器构成。来源：`C5 + CONSISTENCY + UX`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 结构先行组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立视图结构骨架，再接入内容绑定与状态切换。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`VIEW`。

- `R2` 内容渲染组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：将字段绑定与内容渲染叠加到视图骨架，形成基础展示能力。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`CONTENT / UX`。

- `R3` 状态切换组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：在内容渲染基础上接入成功、失败、空结果与加载态切换。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`SWITCH / UX`。

- `R4` 刷新一致性组合
  - `R4.1` 参与基：`B2 + B4 + B5`。
  - `R4.2` 组合方式：将局部刷新与渲染规则绑定，形成可更新且一致的视图输出。
  - `R4.3` 输出能力：`C2 + C4 + C5`。
  - `R4.4` 边界绑定：`REFRESH / CONSISTENCY`。

- `R5` 完整结果渲染模块组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R5.2` 组合方式：将视图结构、内容渲染、状态切换、局部刷新与渲染规则整合为完整结果渲染模块。
  - `R5.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R5.4` 边界绑定：`VIEW / CONTENT / SWITCH / REFRESH / CONSISTENCY / UX`。

- `R6` 禁止组合
  - `R6.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R6.2` 组合方式：无统一视图结构直接拼接文本区域，或局部刷新不经一致性校验直接替换的方案均无效。
  - `R6.3` 输出能力：`C6`。
  - `R6.4` 边界绑定：`VIEW / REFRESH / CONSISTENCY`。

## 5. 验证（Verification）

- `V1` 结构有效性：原文、译文、状态与错误区域必须完整存在。
- `V2` 渲染有效性：文本、语言标签与辅助信息必须正确绑定并渲染。
- `V3` 切换有效性：成功态、失败态、空态与加载态必须可切换。
- `V4` 刷新有效性：局部更新必须不破坏整体视图一致性。
- `V5` 一致性有效性：字段绑定与显示规则必须满足 `CONSISTENCY` 约束。
- `V6` 体验达标性：渲染结果必须满足 `UX` 约束。
- `V7` 最小必要性：移除任一 `B1~B5` 后，`V2~V6` 至少一项失败。
- `V8` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
