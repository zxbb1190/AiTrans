# framework 最小可行基去重与中文命名分析报告

## 1. 结论摘要

- `framework` 目录当前共有 `31` 个模块文档，覆盖 `L0 / L1 / L2` 三层。
- 全目录共定义 `155` 个“最小可行基”条目；其中“完全同名重复”的基有 `11` 组，涉及 `22` 个条目。
- 当前问题不在于“大量完全重复模块”，而在于“多个模块在用不同业务名词重复建模相同的基础骨架”，尤其集中在会话、状态、事件、错误、输出封装这几类能力上。
- 结论是：不建议直接删掉这些模块；更合理的方向是保留模块边界，同时把横向共性的基础骨架抽成可复用的通用基。

## 3. 高重叠能力簇

### 3.1 会话 / 任务 / 状态骨架

涉及模块：

- `L2-M0` 应用生命周期管理器
- `L2-M5` 截图会话管理器
- `L2-M10` 识别翻译任务编排器
- `L2-M15` 展示会话管理器
- `L2-M19` 展示状态机

重复特征：

- 都在做“标识生成 + 初始状态建立 + 状态流转 + 上下文持有 + 中断/关闭 + 事件广播”。
- 命名不同，但骨架模式高度一致。

建议：

- 抽一个 `SessionTaskKernel` 或“会话任务通用骨架”。
- 各业务模块只保留自己的领域状态与专有事件，不再重复描述通用初始化、通用广播、通用上下文仓储。

### 3.2 Provider 适配骨架

涉及模块：

- `L2-M11` 文字识别提供方适配器
- `L2-M12` 翻译提供方适配器

重复特征：

- `B1-B5` 几乎是镜像结构，只是输入对象从图像切换到文本，输出对象从识别结果切换到译文结果。

建议：

- 这两个模块最适合优先去重。
- 可抽成统一 `ProviderAdapterBase`，下沉接口定义、请求封装、结果解析、错误分流、兼容映射五类通用能力。

### 3.3 错误 / 回退 / 输出收敛骨架

涉及模块：

- `L2-M11` 文字识别提供方适配器
- `L2-M12` 翻译提供方适配器
- `L2-M13` 回退策略控制器
- `L2-M14` 识别翻译结果装配器
- `L2-M20` 进程间通信桥接网关
- `L2-M22` 可观测性中心

重复特征：

- 多个模块都包含“错误分类、失败回传、状态封装、结果输出、审计追踪”。
- 这些能力散落在链路不同位置，容易导致错误对象模型和失败语义不统一。

建议：

- 抽出统一错误模型、统一失败分类枚举、统一响应封装协议。
- `L2-M13` 只负责决策，`L2-M14` 只负责结果装配，`L2-M22` 只负责观测，不再各自重新定义一套失败表达。

### 3.4 展示壳层骨架

涉及模块：

- `L1-M3` 结果展示层
- `L2-M16` 结果视图渲染器
- `L2-M17` 交互动作控制器
- `L2-M18` 悬浮面板布局管理器
- `L2-M19` 展示状态机

重复特征：

- 都在围绕“展示会话、显示状态、交互反馈、显隐控制、局部刷新”展开。
- 它们不是完全重复，但已经出现了展示层对 L2 细分模块的再次抽象，导致部分描述有层间重复。

建议：

- 保留这几个模块，不建议直接合并。
- 但建议在 `L1-M3` 中只保留组合职责，不再重复定义过多与 `L2-M16/17/18/19` 完全对应的基名称。

### 3.5 治理基础设施骨架

涉及模块：

- `L1-M4` 运行治理层
- `L2-M20` 进程间通信桥接网关
- `L2-M21` 配置中心
- `L2-M22` 可观测性中心
- `L2-M23` 安全策略守卫
- `L2-M24` 构建发布管理器

重复特征：

- 多个模块都携带“校验、审计、变更通知、环境隔离、风险约束”。
- 这些能力是治理领域的共通底座，而不是单一业务模块特有能力。

建议：

- 后续可抽出治理共享底座，例如：统一事件总线、统一配置变更模型、统一审计接口、统一风险校验协议。

## 4. 不建议强行合并的边界

- `L2-M7` 屏幕帧获取器、`L2-M8` 坐标映射引擎、`L2-M9` 截图结果标准化器：虽然都属于截图链路，但三者分别对应“采集”“坐标计算”“结果统一”，变化原因不同，边界合理。
- `L2-M16` 结果视图渲染器、`L2-M17` 交互动作控制器、`L2-M18` 悬浮面板布局管理器：虽然都在 UI 侧，但分别对应“内容渲染”“动作处理”“空间布局”，建议继续分离。
- `L2-M20` 进程间通信桥接网关、`L2-M23` 安全策略守卫、`L2-M22` 可观测性中心：虽然都有审计/约束属性，但它们分别负责通信、安全、观测，不能简单合并成一个大模块。

## 6. 建议的后续整理顺序

1. 先抽 `ProviderAdapterBase`，因为 `L2-M11` 与 `L2-M12` 的镜像度最高，收益最大。
2. 再抽“会话任务通用骨架”，覆盖 `L2-M5 / L2-M10 / L2-M15 / L2-M19`。
3. 最后统一错误模型、审计接口与结果输出协议，收敛 `L2-M13 / L2-M14 / L2-M20 / L2-M22` 的重叠表达。
