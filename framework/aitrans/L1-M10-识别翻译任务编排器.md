# 识别翻译任务编排器:ShiBieFanYiRenWuBianPaiQi

@framework

## 1. 能力声明（Capability Statement）

- `C1` 任务会话承接能力：提供任务标识生成、上下文装配与初始状态建立能力。
- `C2` 阶段调度能力：提供识别、翻译与结果装配阶段的顺序调度能力。
- `C3` 阶段收敛能力：提供阶段结果记录、状态推进与阶段游标收敛能力。
- `C4` 终止控制能力：提供取消、超时与异常中断的统一终止能力。
- `C5` 事件同步能力：提供任务创建、阶段变化、终态结果的事件广播与审计同步能力。
- `C6` 非能力项：不负责具体 OCR 引擎与翻译引擎的内部调用实现。

## 2. 边界定义（Boundary / 参数）

- `TASK` 任务约束：每次处理必须绑定唯一任务标识、输入句柄与可回收上下文。来源：`C1`。
- `PIPELINE` 流程约束：阶段顺序必须显式定义为识别、翻译、结果装配，且只能在已声明阶段边界插入恢复动作。来源：`C2`。
- `STATE` 状态约束：任务状态必须支持待执行、执行中、部分完成、完成、失败、取消。来源：`C3`。
- `INTERRUPT` 中断约束：取消、超时与异常必须终止未开始的下游阶段并写入终态。来源：`C4`。
- `EVENT` 事件约束：任务必须发出创建、阶段变化、完成、失败、取消五类标准事件，并留下审计引用。来源：`C5`。
- `RUNTIME` 运行约束：同一任务上下文与阶段游标必须由单一编排控制器独占持有。来源：`C1 + C2 + C3 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 任务会话骨架：L0.M0[会话初始化器] + L0.M1[上下文存储器] + L0.M2[状态机内核]。来源：`C1 + TASK + STATE`。
- `B2` 阶段调度骨架：L0.M18[识别流水线调度器] + L0.M19[阶段结果记录器]。来源：`C2 + C3 + PIPELINE`。
- `B3` 终止同步骨架：L0.M3[事件总线] + L0.M4[生命周期终止控制器] + L0.M9[审计追踪器]。来源：`C4 + C5 + INTERRUPT + EVENT + RUNTIME`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 会话承接组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立任务会话、上下文与初始状态，再开放阶段调度入口。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`TASK / STATE`。

- `R2` 调度推进组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：将阶段顺序定义、阶段结果记录与状态推进绑定，形成完整任务执行路径。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`PIPELINE / STATE`。

- `R3` 终止同步组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：在任务调度链路上叠加终止控制、事件广播与审计同步，形成完整运行收敛面。
  - `R3.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R3.4` 边界绑定：`INTERRUPT / EVENT / RUNTIME`。

- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：无任务会话直接调度阶段，或无终止同步直接放行终态结果的方案均无效。
  - `R4.3` 输出能力：`C6`。
  - `R4.4` 边界绑定：`TASK / PIPELINE / INTERRUPT / EVENT`。

## 5. 验证（Verification）

- `V1` 会话有效性：任务标识、上下文装配与初始状态建立必须可执行。
- `V2` 调度有效性：识别、翻译与结果装配必须按声明阶段顺序推进。
- `V3` 收敛有效性：阶段结果记录、状态推进与阶段游标收敛必须一致。
- `V4` 终止有效性：取消、超时与异常必须阻断未开始的下游阶段并写入终态。
- `V5` 同步有效性：创建、阶段变化与终态事件必须可广播、可审计。
- `V6` 边界符合性：所有有效组合必须满足 `RUNTIME` 约束。
- `V7` 最小必要性：移除任一 `B1~B3` 后，`V2~V6` 至少一项失败。
- `V8` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
