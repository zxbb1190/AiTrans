# 模块框架：L1-M2-识别翻译层

@module

## 1. 能力声明（Capability Statement）

- `C1` 输入编排能力：承接截图结果并形成可处理的识别翻译任务。
- `C2` 文本识别能力：提供 OCR 引擎适配、文字提取与语言初判能力。
- `C3` 翻译转换能力：提供翻译引擎适配、译文生成与结果清洗能力。
- `C4` 策略回退能力：提供引擎切换、失败回退、超时控制与重试能力。
- `C5` 结果标准化能力：提供原文、译文、语言信息与错误状态的统一输出能力。
- `C6` 非能力项：不负责截图采集与 UI 结果渲染。

## 2. 边界定义（Boundary / 参数）

- `TASK` 任务约束：每次处理必须绑定唯一任务上下文并具备状态流转。来源：`C1`。
- `OCR` 识别约束：OCR 引擎必须通过统一接口访问，支持基础中英文识别。来源：`C2`。
- `TRANS` 翻译约束：翻译引擎必须通过统一接口访问，支持超时与错误返回。来源：`C3`。
- `FALLBACK` 回退约束：处理链路必须支持失败分流、重试与候选引擎切换。来源：`C4`。
- `RESULT` 输出约束：处理结果必须标准化并携带必要元信息。来源：`C5`。
- `PERF` 性能约束：识别与翻译链路需满足可接受时延与资源使用边界。来源：`C2 + C3 + C4`。
- `SEC` 安全约束：调用外部服务时需控制敏感信息暴露与密钥访问范围。来源：`C3 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 任务编排基：framework.L2.M10[识别翻译任务编排器]。来源：`C1 + TASK`。
- `B2` OCR 适配基：framework.L2.M11[文字识别提供方适配器]。来源：`C2 + OCR`。
- `B3` 翻译适配基：framework.L2.M12[翻译提供方适配器]。来源：`C3 + TRANS`。
- `B4` 回退控制基：framework.L2.M13[回退策略控制器]。来源：`C4 + FALLBACK + PERF`。
- `B5` 标准输出基：framework.L2.M14[识别翻译结果装配器]。来源：`C5 + RESULT + SEC`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 任务先行组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立识别翻译任务上下文与状态机，再接入引擎处理链路。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`TASK`。

- `R2` 识别处理组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在任务编排基础上接入 OCR 引擎适配与文本提取能力，形成识别链路。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`OCR / PERF`。

- `R3` 翻译转换组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：在识别输出基础上接入翻译引擎适配，形成从图像到译文的处理链路。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`OCR / TRANS / PERF`。

- `R4` 回退闭环组合
  - `R4.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R4.2` 组合方式：将错误分类、超时与重试策略绑定到识别翻译链路，形成可恢复执行路径。
  - `R4.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R4.4` 边界绑定：`FALLBACK / PERF / SEC`。

- `R5` 完整处理模块组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R5.2` 组合方式：将任务编排、OCR、翻译、回退与标准输出整合为完整识别翻译模块。
  - `R5.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R5.4` 边界绑定：`TASK / OCR / TRANS / FALLBACK / RESULT / PERF / SEC`。

- `R6` 禁止组合
  - `R6.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R6.2` 组合方式：绕过统一接口直接耦合具体 OCR 或翻译引擎，或无回退机制直接暴露失败结果的方案均无效。
  - `R6.3` 输出能力：`C6`。
  - `R6.4` 边界绑定：`OCR / TRANS / FALLBACK / RESULT / SEC`。

## 5. 验证（Verification）

- `V1` 任务一致性：处理任务必须具备明确的输入、状态与结果输出。
- `V2` 识别有效性：OCR 接口调用、文本提取与语言初判必须可执行。
- `V3` 翻译有效性：翻译调用、译文生成与结果清洗必须可执行。
- `V4` 回退有效性：超时、重试、切换与失败分类机制必须覆盖关键处理路径。
- `V5` 输出标准化：所有处理结果必须满足 `RESULT` 约束并可被展示模块直接消费。
- `V6` 安全符合性：外部服务调用必须满足 `SEC` 约束。
- `V7` 最小必要性：移除任一 `B1~B5` 后，`V2~V6` 至少一项失败。
- `V8` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
