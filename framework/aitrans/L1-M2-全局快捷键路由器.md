# 全局快捷键路由器:QuanJuKuaiJieJianLuYouQi

@framework

## 1. 能力声明（Capability Statement）

- `C1` 快捷键注册能力：提供全局快捷键注册、注销与重注册能力。
- `C2` 冲突识别能力：提供快捷键冲突检测、冲突提示与冲突规避能力。
- `C3` 动作路由能力：提供快捷键触发到具体系统动作的分发能力。
- `C4` 配置更新能力：提供快捷键动态修改、持久化与热更新能力。
- `C5` 触发审计能力：提供快捷键触发日志记录与失败追踪能力。
- `C6` 非能力项：不负责动作执行目标的内部实现。

## 2. 边界定义（Boundary / 参数）

- `REGISTER` 注册约束：快捷键必须支持注册成功、失败与注销状态。来源：`C1`。
- `CONFLICT` 冲突约束：冲突必须可识别、可提示、可回退。来源：`C2`。
- `ROUTE` 路由约束：触发事件必须映射到唯一有效动作。来源：`C3`。
- `CONFIG` 配置约束：快捷键配置必须支持读取、变更与持久化。来源：`C4`。
- `AUDIT` 审计约束：每次注册与触发必须可追踪。来源：`C5`。
- `COMPAT` 兼容约束：快捷键语义需适配不同平台键位约定。来源：`C1 + C2 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 注册配置骨架：L0.M1[上下文存储器] + L0.M2[状态机内核] + L0.M9[审计追踪器]。来源：`C1 + C4 + C5 + REGISTER + CONFIG + AUDIT`。
- `B2` 冲突收敛骨架：L0.M7[错误分流器] + L0.M9[审计追踪器]。来源：`C2 + CONFLICT + COMPAT`。
- `B3` 动作路由骨架：L0.M3[事件总线]。来源：`C3 + ROUTE`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 注册配置组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立快捷键注册、配置持久化与触发审计的统一状态源，再接入冲突收敛与动作路由。
  - `R1.3` 输出能力：`C1 + C4 + C5`。
  - `R1.4` 边界绑定：`REGISTER / CONFIG / AUDIT`。

- `R2` 冲突收敛组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在注册配置状态源上叠加冲突检测、提示与回退，形成受控注册路径。
  - `R2.3` 输出能力：`C1 + C2 + C4 + C5`。
  - `R2.4` 边界绑定：`REGISTER / CONFLICT / CONFIG / COMPAT`。

- `R3` 动作路由组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将已注册且经冲突收敛的快捷键绑定到唯一动作路由，形成稳定输入分发链路。
  - `R3.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R3.4` 边界绑定：`ROUTE / REGISTER / CONFLICT / AUDIT`。

- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：无冲突收敛直接覆盖注册，或无动作路由直接消费触发事件的方案均无效。
  - `R4.3` 输出能力：`C6`。
  - `R4.4` 边界绑定：`CONFLICT / ROUTE / CONFIG`。

## 5. 验证（Verification）

- `V1` 注册配置有效性：快捷键注册、注销、重注册、配置修改与持久化必须可执行。
- `V2` 冲突收敛有效性：冲突检测、提示与回退必须可执行。
- `V3` 动作路由有效性：触发事件必须可分发到唯一有效动作。
- `V4` 兼容符合性：不同平台快捷键语义必须满足 `COMPAT` 约束。
- `V5` 最小必要性：移除任一 `B1~B3` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
