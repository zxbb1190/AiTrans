# 模块框架：L1-M0-桌面宿主层

@module

## 1. 能力声明（Capability Statement）

- `C1` 生命周期承载能力：提供应用初始化、退出、异常恢复与单实例管理能力。
- `C2` 系统入口能力：提供托盘驻留、菜单入口与前后台切换能力。
- `C3` 快捷触发能力：提供全局快捷键注册、冲突处理与触发路由能力。
- `C4` 窗口调度能力：提供主窗口、截图窗口、结果窗口的创建、显示、隐藏与销毁能力。
- `C5` 权限协同能力：提供屏幕录制、辅助功能等桌面权限检测与引导能力。
- `C6` 非能力项：不负责截图选区绘制、OCR 识别与翻译处理。

## 2. 边界定义（Boundary / 参数）

- `APP` 应用生命周期约束：必须支持启动、就绪、激活、退出与异常恢复等状态切换。来源：`C1`。
- `TRAY` 托盘入口约束：托盘必须可驻留、可交互、可映射到核心操作入口。来源：`C2`。
- `SHORTCUT` 快捷键约束：全局快捷键必须支持注册、注销、冲突检测与重绑。来源：`C3`。
- `WINDOW` 窗口调度约束：窗口必须具备分工明确、状态可控、生命周期可追踪特性。来源：`C4`。
- `PERM` 权限约束：需识别系统权限状态并支持提示、降级与跳转引导。来源：`C5`。
- `RUNTIME` 运行边界约束：仅主进程可直接访问系统级桌面能力。来源：`C1 + C4 + C5`。
- `COMPAT` 兼容约束：需支持目标平台下的基础桌面行为一致性。来源：`C1 + C5`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 生命周期基：framework.L2.M0[应用生命周期管理器]。来源：`C1 + APP + COMPAT`。
- `B2` 系统入口基：framework.L2.M1[托盘入口管理器]。来源：`C2 + TRAY`。
- `B3` 快捷路由基：framework.L2.M2[全局快捷键路由器]。来源：`C3 + SHORTCUT + COMPAT`。
- `B4` 窗口调度基：framework.L2.M3[窗口调度器]。来源：`C4 + WINDOW + RUNTIME`。
- `B5` 权限协同基：framework.L2.M4[权限协调器]。来源：`C5 + PERM + COMPAT`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 生命周期先行组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立应用启动、单实例与退出清理规则，再接入其他入口能力。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`APP / COMPAT`。

- `R2` 系统入口组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在生命周期基上叠加托盘与菜单入口，形成稳定的系统驻留能力。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`TRAY / APP`。

- `R3` 触发调度组合
  - `R3.1` 参与基：`B1 + B3 + B4`。
  - `R3.2` 组合方式：将快捷键触发与窗口调度绑定，形成从系统输入到窗口行为的执行链路。
  - `R3.3` 输出能力：`C1 + C3 + C4`。
  - `R3.4` 边界绑定：`SHORTCUT / WINDOW / RUNTIME`。

- `R4` 权限闭环组合
  - `R4.1` 参与基：`B1 + B4 + B5`。
  - `R4.2` 组合方式：在窗口调度前绑定权限探测与降级策略，形成受控桌面能力访问路径。
  - `R4.3` 输出能力：`C1 + C4 + C5`。
  - `R4.4` 边界绑定：`PERM / WINDOW / COMPAT`。

- `R5` 完整宿主组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R5.2` 组合方式：将生命周期、入口、快捷触发、窗口调度与权限协同整合为完整桌面宿主模块。
  - `R5.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R5.4` 边界绑定：`APP / TRAY / SHORTCUT / WINDOW / PERM / RUNTIME / COMPAT`。

- `R6` 禁止组合
  - `R6.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R6.2` 组合方式：绕过生命周期或权限协同直接创建窗口与注册系统能力的方案均无效。
  - `R6.3` 输出能力：`C6`。
  - `R6.4` 边界绑定：`APP / PERM / RUNTIME`。

## 5. 验证（Verification）

- `V1` 生命周期一致性：应用启动、激活、退出、异常恢复链路必须完整可追踪。
- `V2` 系统入口有效性：托盘入口必须可驻留、可交互且状态同步正确。
- `V3` 快捷路由有效性：快捷键注册、冲突处理与触发分发必须可执行。
- `V4` 窗口调度一致性：各窗口的创建、显示、隐藏、销毁必须满足 `WINDOW` 约束。
- `V5` 权限协同有效性：权限检测、提示与降级路径必须覆盖关键桌面能力调用。
- `V6` 边界符合性：所有有效组合必须满足 `RUNTIME / COMPAT` 约束。
- `V7` 最小必要性：移除任一 `B1~B5` 后，`V2~V6` 至少一项失败。
- `V8` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
