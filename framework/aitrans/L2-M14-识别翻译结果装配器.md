# 模块框架：L2-M14-识别翻译结果装配器

@module

## 1. 能力声明（Capability Statement）

- `C1` 结果聚合能力：提供 OCR 结果、翻译结果与任务元信息聚合能力。
- `C2` 结果清洗能力：提供原文、译文与语言信息的统一清洗能力。
- `C3` 状态封装能力：提供成功、失败、部分成功与降级状态封装能力。
- `C4` 错误整合能力：提供 OCR/翻译阶段错误信息合并与标准表达能力。
- `C5` 标准输出能力：提供面向展示层的统一结果模型输出能力。
- `C6` 非能力项：不负责具体识别与翻译执行。

## 2. 边界定义（Boundary / 参数）

- `AGGREGATE` 聚合约束：不同阶段结果必须在统一任务维度上聚合。来源：`C1`。
- `CLEAN` 清洗约束：原文、译文与语言字段必须完成规范化清洗。来源：`C2`。
- `STATUS` 状态约束：输出结果必须具有明确处理状态。来源：`C3`。
- `ERROR` 错误约束：错误信息必须标准化并可追踪来源阶段。来源：`C4`。
- `OUTPUT` 输出约束：结果必须可直接被展示层消费。来源：`C5`。
- `CONSISTENCY` 一致性约束：结果字段必须具备确定性与可复现性。来源：`C1 + C2 + C3 + C4 + C5`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 结果聚合基：由聚合器、阶段结果绑定器、任务元信息收集器构成。来源：`C1 + AGGREGATE`。
- `B2` 字段清洗基：由原文清洗器、译文清洗器、语言字段规范器构成。来源：`C2 + CLEAN`。
- `B3` 状态封装基：由状态判断器、状态封装器、降级状态构造器构成。来源：`C3 + STATUS`。
- `B4` 错误整合基：framework.L3.M7[错误分流器]。来源：`C4 + ERROR`。
- `B5` 标准输出基：framework.L3.M8[标准输出装配器]。来源：`C5 + OUTPUT + CONSISTENCY`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 聚合先行组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先按任务维度聚合识别、翻译与元信息结果，再接入清洗与状态封装。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`AGGREGATE`。

- `R2` 清洗封装组合
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：将阶段结果聚合后进行字段清洗与状态封装，形成统一处理结果。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`CLEAN / STATUS`。

- `R3` 错误状态组合
  - `R3.1` 参与基：`B3 + B4`。
  - `R3.2` 组合方式：将错误整合结果与状态封装绑定，形成统一失败态与部分成功态表达。
  - `R3.3` 输出能力：`C3 + C4`。
  - `R3.4` 边界绑定：`ERROR / STATUS`。

- `R4` 输出一致性组合
  - `R4.1` 参与基：`B2 + B4 + B5`。
  - `R4.2` 组合方式：在字段清洗和错误整合后统一适配为展示层可消费输出模型。
  - `R4.3` 输出能力：`C2 + C4 + C5`。
  - `R4.4` 边界绑定：`OUTPUT / CONSISTENCY`。

- `R5` 完整结果装配模块组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R5.2` 组合方式：将结果聚合、字段清洗、状态封装、错误整合与标准输出整合为完整结果装配模块。
  - `R5.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R5.4` 边界绑定：`AGGREGATE / CLEAN / STATUS / ERROR / OUTPUT / CONSISTENCY`。

- `R6` 禁止组合
  - `R6.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R6.2` 组合方式：无统一状态封装直接输出阶段碎片结果，或错误信息未标注来源阶段的方案均无效。
  - `R6.3` 输出能力：`C6`。
  - `R6.4` 边界绑定：`STATUS / ERROR / OUTPUT`。

## 5. 验证（Verification）

- `V1` 聚合有效性：OCR、翻译与任务元信息必须可按任务维度聚合。
- `V2` 清洗有效性：原文、译文与语言信息必须完成规范化清洗。
- `V3` 状态有效性：成功、失败、部分成功与降级状态必须被正确封装。
- `V4` 错误有效性：错误信息必须被正确整合并标注来源阶段。
- `V5` 输出有效性：结果必须能被展示层直接消费。
- `V6` 一致性符合性：输出字段必须满足 `CONSISTENCY` 约束。
- `V7` 最小必要性：移除任一 `B1~B5` 后，`V2~V6` 至少一项失败。
- `V8` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
