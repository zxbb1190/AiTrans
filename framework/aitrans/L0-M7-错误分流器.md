# 错误分流器:CuoWuFenLiuQi

@framework

## 1. 能力声明（Capability Statement）

- `C1` 叶子结构承接能力：统一完成错误分类、失败标签归并、可重试性判断与回传方向分流。
- `C2` 适用承接能力：在 `L1-M11-提供方执行适配器`、`L1-M12-回退策略控制器`、`L1-M13-识别翻译结果装配器` 边界内作为 `通用可复用` 结构单元被稳定复用。
- `C3` 非能力项：只负责错误归口，不负责重试执行、降级构造与结果最终输出。

## 2. 边界定义（Boundary / 参数）

- `PARENT` 上级适用边界：`L1-M11-提供方执行适配器`、`L1-M12-回退策略控制器`、`L1-M13-识别翻译结果装配器`。来源：`C2`。
- `REUSE` 复用边界：`通用可复用`。来源：`C2`。
- `STRUCT` 结构边界：只负责错误归口，不负责重试执行、降级构造与结果最终输出。来源：`C1 + C3`。
- `STOP` 停止边界：继续下钻会变成枚举表、映射器和条件函数实现，不再是稳定架构边界。来源：`C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 错误分流器结构基：由该叶子模块的承载壳、输入绑定位与输出绑定位构成。来源：`C1 + C2 + STRUCT`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 单基自足组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：在满足 `PARENT / REUSE / STRUCT` 边界时，`B1` 独立成立并向上层输出单一叶子结构。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`PARENT / REUSE / STRUCT`。

- `R2` 停止下钻组合
  - `R2.1` 参与基：`B1`。
  - `R2.2` 组合方式：若继续把 `B1` 细化为纯实现类、函数集合或处理器/构造器/映射器级实现细节，则该展开无效。
  - `R2.3` 输出能力：`C3`。
  - `R2.4` 边界绑定：`STOP`。

## 5. 验证（Verification）

- `V1` 叶子成立性：`B1` 必须在声明边界内作为独立叶子结构成立。
- `V2` 复用成立性：该模块必须可被上级模块直接引用，而不依赖更细的本层子模块。
- `V3` 非能力项成立性：越过 `只负责错误归口，不负责重试执行、降级构造与结果最终输出。` 的扩张必须判为结构越界。
- `V4` 停止性：继续下钻后若仅剩实现对象或函数级单元，应判为不适用。
- `V5` 结论表达：逐条输出 `R1/R2 -> C* / 边界` 的通过或失败结论。
