# 模块框架：L2-M4-权限协调器

@module

## 1. 能力声明（Capability Statement）

- `C1` 权限探测能力：提供系统权限状态识别与查询能力。
- `C2` 权限缓存能力：提供权限状态缓存、刷新与失效管理能力。
- `C3` 权限引导能力：提供权限缺失提示、引导说明与跳转协助能力。
- `C4` 降级决策能力：提供权限不足场景下的降级与替代路径能力。
- `C5` 权限事件通知能力：提供权限变化事件广播与依赖模块同步能力。
- `C6` 非能力项：不负责具体系统权限申请界面的实现。

## 2. 边界定义（Boundary / 参数）

- `CHECK` 探测约束：权限检查必须可重复执行且结果可判定。来源：`C1`。
- `CACHE` 缓存约束：缓存必须支持时效性、手动刷新与失效策略。来源：`C2`。
- `GUIDE` 引导约束：权限不足时必须向用户提供明确引导。来源：`C3`。
- `FALLBACK` 降级约束：关键权限缺失时必须具备可执行降级路径。来源：`C4`。
- `NOTIFY` 通知约束：权限状态变化必须可广播与可消费。来源：`C5`。
- `COMPAT` 兼容约束：不同平台权限语义与检查方式需可适配。来源：`C1 + C3 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 权限探测基：由权限检查器、状态解析器、结果标准化器构成。来源：`C1 + CHECK`。
- `B2` 状态缓存基：由缓存仓库、刷新器、失效器与一致性检查器构成。来源：`C2 + CACHE`。
- `B3` 引导交互基：由提示器、说明生成器、设置跳转器与帮助链接器构成。来源：`C3 + GUIDE`。
- `B4` 降级决策基：由能力评估器、降级规则器、替代路径选择器与阻断器构成。来源：`C4 + FALLBACK`。
- `B5` 通知广播基：由事件发布器、订阅器、变化监听器与同步器构成。来源：`C5 + NOTIFY + COMPAT`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 探测先行组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立权限探测与结果标准化，再接入缓存与引导能力。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`CHECK`。

- `R2` 探测缓存组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在权限检查基础上建立缓存与刷新机制，形成稳定权限状态源。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`CHECK / CACHE`。

- `R3` 引导降级组合
  - `R3.1` 参与基：`B1 + B3 + B4`。
  - `R3.2` 组合方式：将权限结果与用户引导、降级路径绑定，形成权限不足处理闭环。
  - `R3.3` 输出能力：`C1 + C3 + C4`。
  - `R3.4` 边界绑定：`GUIDE / FALLBACK / COMPAT`。

- `R4` 同步广播组合
  - `R4.1` 参与基：`B2 + B5`。
  - `R4.2` 组合方式：将缓存状态变化与事件通知绑定，形成权限状态同步链路。
  - `R4.3` 输出能力：`C2 + C5`。
  - `R4.4` 边界绑定：`CACHE / NOTIFY`。

- `R5` 完整权限模块组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R5.2` 组合方式：将权限探测、缓存、引导、降级与通知整合为完整权限协同模块。
  - `R5.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R5.4` 边界绑定：`CHECK / CACHE / GUIDE / FALLBACK / NOTIFY / COMPAT`。

- `R6` 禁止组合
  - `R6.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R6.2` 组合方式：无权限探测直接执行关键能力，或权限缺失时无引导与降级路径的方案均无效。
  - `R6.3` 输出能力：`C6`。
  - `R6.4` 边界绑定：`CHECK / GUIDE / FALLBACK`。

## 5. 验证（Verification）

- `V1` 探测有效性：权限检查与结果标准化必须可执行。
- `V2` 缓存有效性：状态缓存、刷新与失效必须可执行。
- `V3` 引导有效性：权限不足时必须提供明确提示与引导。
- `V4` 降级有效性：关键权限缺失时必须具备可执行降级路径。
- `V5` 通知有效性：权限变化必须可广播并被依赖模块消费。
- `V6` 兼容符合性：权限语义与处理路径必须满足 `COMPAT` 约束。
- `V7` 最小必要性：移除任一 `B1~B5` 后，`V2~V6` 至少一项失败。
- `V8` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
