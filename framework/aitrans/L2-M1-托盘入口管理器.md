# 模块框架：L2-M1-托盘入口管理器

@module

## 1. 能力声明（Capability Statement）

- `C1` 托盘驻留能力：提供应用托盘图标创建、驻留与销毁能力。
- `C2` 菜单装配能力：提供托盘菜单构建、菜单项绑定与动态更新能力。
- `C3` 状态映射能力：提供托盘状态与应用状态同步能力。
- `C4` 快捷入口能力：提供从托盘到截图、设置、退出等核心操作的入口能力。
- `C5` 交互反馈能力：提供点击、双击、右键与状态提示反馈能力。
- `C6` 非能力项：不负责全局快捷键注册与窗口创建实现细节。

## 2. 边界定义（Boundary / 参数）

- `TRAY` 托盘驻留约束：托盘必须在应用运行期间可见且可操作。来源：`C1`。
- `MENU` 菜单约束：菜单项必须具备唯一标识、动作绑定与动态可更新能力。来源：`C2`。
- `STATE` 状态约束：托盘状态必须与应用主状态保持一致。来源：`C3`。
- `ENTRY` 入口约束：托盘必须能映射到核心系统操作入口。来源：`C4`。
- `FEEDBACK` 反馈约束：用户交互后必须具备明确反馈。来源：`C5`。
- `COMPAT` 兼容约束：托盘行为需适配不同平台桌面语义。来源：`C1 + C5`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 托盘实例基：由托盘创建器、图标加载器、驻留控制器与销毁器构成。来源：`C1 + TRAY`。
- `B2` 菜单构建基：由菜单模型、菜单构建器、菜单动作绑定器与刷新器构成。来源：`C2 + MENU`。
- `B3` 状态同步基：由状态观察器、状态映射器、图标切换器与标题更新器构成。来源：`C3 + STATE`。
- `B4` 入口路由基：由入口动作注册器、操作分发器与导航触发器构成。来源：`C4 + ENTRY`。
- `B5` 反馈处理基：由点击处理器、悬停提示器、消息通知器与事件记录器构成。来源：`C5 + FEEDBACK + COMPAT`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 驻留先行组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立托盘实例与驻留状态，再接入菜单与动作。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`TRAY`。

- `R2` 菜单入口组合
  - `R2.1` 参与基：`B1 + B2 + B4`。
  - `R2.2` 组合方式：在托盘驻留基础上挂载菜单模型与入口路由，形成可操作托盘入口。
  - `R2.3` 输出能力：`C1 + C2 + C4`。
  - `R2.4` 边界绑定：`MENU / ENTRY`。

- `R3` 状态反馈组合
  - `R3.1` 参与基：`B1 + B3 + B5`。
  - `R3.2` 组合方式：将状态同步与交互反馈绑定到托盘实例，形成可感知状态入口。
  - `R3.3` 输出能力：`C1 + C3 + C5`。
  - `R3.4` 边界绑定：`STATE / FEEDBACK / COMPAT`。

- `R4` 动态菜单组合
  - `R4.1` 参与基：`B2 + B3 + B4`。
  - `R4.2` 组合方式：根据应用状态动态刷新菜单项与入口可用性。
  - `R4.3` 输出能力：`C2 + C3 + C4`。
  - `R4.4` 边界绑定：`MENU / STATE / ENTRY`。

- `R5` 完整托盘模块组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R5.2` 组合方式：将托盘驻留、菜单装配、状态同步、入口路由与反馈处理整合为完整托盘模块。
  - `R5.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R5.4` 边界绑定：`TRAY / MENU / STATE / ENTRY / FEEDBACK / COMPAT`。

- `R6` 禁止组合
  - `R6.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R6.2` 组合方式：无托盘实例直接注入菜单，或菜单动作无状态同步与反馈的方案均无效。
  - `R6.3` 输出能力：`C6`。
  - `R6.4` 边界绑定：`TRAY / MENU / STATE / FEEDBACK`。

## 5. 验证（Verification）

- `V1` 驻留有效性：托盘图标必须可创建、驻留与销毁。
- `V2` 菜单有效性：菜单项必须可构建、绑定与动态更新。
- `V3` 状态有效性：托盘状态与应用状态必须同步正确。
- `V4` 入口有效性：截图、设置、退出等入口必须可触发。
- `V5` 反馈有效性：交互动作必须具备明确反馈。
- `V6` 兼容符合性：托盘行为必须满足 `COMPAT` 约束。
- `V7` 最小必要性：移除任一 `B1~B5` 后，`V2~V6` 至少一项失败。
- `V8` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
