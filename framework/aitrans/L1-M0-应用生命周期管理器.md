# 应用生命周期管理器:YingYongShengMingZhouQiGuanLiQi

@framework

## 1. 能力声明（Capability Statement）

- `C1` 启动管理能力：提供应用启动、初始化编排与就绪状态建立能力。
- `C2` 单实例管理能力：提供单实例锁获取、重复启动识别与实例切换协调能力。
- `C3` 退出清理能力：提供退出前资源释放、状态持久化与清理执行能力。
- `C4` 异常恢复能力：提供未捕获异常监听、崩溃恢复与安全退出能力。
- `C5` 生命周期事件分发能力：提供生命周期事件注册、广播与监听协调能力。
- `C6` 非能力项：不负责托盘交互、窗口业务调度与权限探测细节。

## 2. 边界定义（Boundary / 参数）

- `BOOT` 启动约束：必须支持冷启动、热激活与就绪回调。来源：`C1`。
- `INSTANCE` 单实例约束：必须保证单实例唯一性并支持重复启动处理。来源：`C2`。
- `EXIT` 退出约束：必须支持主动退出、被动退出与异常退出三类路径。来源：`C3`。
- `RECOVERY` 恢复约束：必须对异常事件具备记录、分流与恢复策略。来源：`C4`。
- `EVENT` 生命周期事件约束：生命周期事件必须可注册、可取消、可追踪。来源：`C5`。
- `RUNTIME` 运行边界约束：生命周期状态源应由主进程统一管理。来源：`C1 + C2 + C3 + C4 + C5`。
- `COMPAT` 兼容约束：生命周期行为在目标平台上应保持一致语义。来源：`C1 + C3 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 生命周期会话骨架：L0.M0[会话初始化器] + L0.M1[上下文存储器] + L0.M2[状态机内核]。来源：`C1 + C2 + BOOT + INSTANCE`。
- `B2` 退出恢复广播骨架：L0.M3[事件总线] + L0.M4[生命周期终止控制器] + L0.M7[错误分流器]。来源：`C3 + C4 + C5 + EXIT + RECOVERY + EVENT + RUNTIME`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 生命周期承接组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先完成启动编排、就绪状态建立与单实例控制，形成统一生命周期状态源。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`BOOT / INSTANCE / RUNTIME / COMPAT`。

- `R2` 终止恢复广播组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：将退出处理、异常恢复与生命周期事件广播绑定到统一状态源，形成完整终止与恢复路径。
  - `R2.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R2.4` 边界绑定：`EXIT / RECOVERY / EVENT / RUNTIME / COMPAT`。

- `R3` 禁止组合
  - `R3.1` 参与基：`B1 + B2`。
  - `R3.2` 组合方式：绕过生命周期状态源直接启动，或无退出恢复广播骨架直接强制终止的方案均无效。
  - `R3.3` 输出能力：`C6`。
  - `R3.4` 边界绑定：`INSTANCE / EXIT / RECOVERY / RUNTIME`。

## 5. 验证（Verification）

- `V1` 生命周期承接有效性：应用启动、初始化、就绪与单实例控制必须完整可执行。
- `V2` 终止恢复广播有效性：退出清理、异常恢复与生命周期事件分发必须覆盖关键终止路径。
- `V3` 边界符合性：所有有效组合必须满足 `RUNTIME / COMPAT` 约束。
- `V4` 最小必要性：移除任一 `B1~B2` 后，`V1~V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
