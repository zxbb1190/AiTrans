# 交互会话管理器:JiaoHuHuiHuaGuanLiQi

@framework

## 1. 能力声明（Capability Statement）

- `C1` 会话创建能力：提供交互会话初始化与唯一标识生成能力。
- `C2` 上下文承接能力：提供截图选区、展示数据、窗口上下文与临时交互数据管理能力。
- `C3` 会话更新能力：提供状态推进、内容刷新与二次触发复用能力。
- `C4` 会话终止能力：提供取消、关闭、超时、异常终止与资源回收能力。
- `C5` 会话事件通知能力：提供会话创建、更新、结束与失败事件广播能力。
- `C6` 非能力项：不负责截图采集执行、识别翻译任务编排与视图渲染实现。

## 2. 边界定义（Boundary / 参数）

- `SESSION` 会话约束：每次交互必须绑定唯一会话标识。来源：`C1`。
- `CONTEXT` 上下文约束：会话上下文必须支持读写、更新与销毁。来源：`C2`。
- `UPDATE` 更新约束：会话内容或阶段更新必须保持上下文一致性。来源：`C3`。
- `TERMINATE` 终止约束：取消、关闭、超时与异常终止必须可中断后续链路并回收资源。来源：`C4`。
- `EVENT` 事件约束：会话事件必须可追踪与订阅。来源：`C5`。
- `RUNTIME` 运行约束：同一受控交互面仅允许存在唯一活动会话。来源：`C1 + C2 + C3 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 会话上下文骨架：L0.M0[会话初始化器] + L0.M1[上下文存储器] + L0.M2[状态机内核]。来源：`C1 + C2 + C3 + SESSION + CONTEXT + UPDATE`。
- `B2` 终止广播骨架：L0.M3[事件总线] + L0.M4[生命周期终止控制器]。来源：`C4 + C5 + TERMINATE + EVENT + RUNTIME`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 会话上下文组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先完成交互会话初始化、上下文承接与更新控制，形成统一交互状态源。
  - `R1.3` 输出能力：`C1 + C2 + C3`。
  - `R1.4` 边界绑定：`SESSION / CONTEXT / UPDATE`。

- `R2` 终止广播组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：将取消、关闭、超时与异常终止路径绑定到统一状态源，形成可同步回收的会话闭环。
  - `R2.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R2.4` 边界绑定：`TERMINATE / EVENT / RUNTIME`。

- `R3` 禁止组合
  - `R3.1` 参与基：`B1 + B2`。
  - `R3.2` 组合方式：无会话上下文骨架直接进入截图或展示链路，或结束后不触发终止广播的方案均无效。
  - `R3.3` 输出能力：`C6`。
  - `R3.4` 边界绑定：`SESSION / CONTEXT / TERMINATE / EVENT`。

## 5. 验证（Verification）

- `V1` 会话上下文有效性：会话创建、上下文读写、更新与销毁必须可执行。
- `V2` 终止广播有效性：取消、关闭、超时、异常终止与会话事件广播必须中断后续链路并对外可追踪。
- `V3` 边界符合性：同一时刻会话控制必须满足 `RUNTIME` 约束。
- `V4` 最小必要性：移除任一 `B1~B2` 后，`V1~V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
