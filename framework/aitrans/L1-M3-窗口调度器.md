# 窗口调度器:ChuangKouDiaoDuQi

@framework

## 1. 能力声明（Capability Statement）

- `C1` 窗口创建能力：提供不同类型窗口的创建与初始化能力。
- `C2` 窗口注册能力：提供窗口标识注册、索引维护与实例查询能力。
- `C3` 窗口可见性控制能力：提供显示、隐藏、聚焦、置顶与恢复能力。
- `C4` 窗口销毁能力：提供窗口关闭、资源回收与安全销毁能力。
- `C5` 窗口协同能力：提供多窗口之间的切换、互斥与联动调度能力。
- `C6` 非能力项：不负责具体窗口内部 UI 渲染逻辑。

## 2. 边界定义（Boundary / 参数）

- `CREATE` 创建约束：窗口创建必须支持模板化配置与初始化钩子。来源：`C1`。
- `REGISTRY` 注册约束：窗口实例必须具备唯一标识与可查询性。来源：`C2`。
- `VISIBILITY` 可见性约束：窗口显示与隐藏行为必须受统一控制。来源：`C3`。
- `DESTROY` 销毁约束：窗口关闭后必须完成资源回收与注册解绑。来源：`C4`。
- `COORD` 协同约束：多窗口调度必须满足顺序、互斥与依赖关系。来源：`C5`。
- `RUNTIME` 运行边界约束：窗口调度权应统一收敛于主进程。来源：`C1 + C2 + C3 + C4 + C5`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 窗口实例骨架：L0.M10[窗口实例承载器]。来源：`C1 + C2 + CREATE + REGISTRY`。
- `B2` 可见回收骨架：L0.M11[窗口生命周期控制器]。来源：`C3 + C4 + VISIBILITY + DESTROY`。
- `B3` 协同调度骨架：L0.M12[多窗口协调器]。来源：`C5 + COORD + RUNTIME`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 窗口实例组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先统一窗口模板、创建策略与注册索引，形成可追踪窗口实例面。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`CREATE / REGISTRY`。

- `R2` 生命周期控制组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：对已注册窗口统一实施显示隐藏、聚焦恢复与销毁回收，形成完整实例生命周期。
  - `R2.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R2.4` 边界绑定：`VISIBILITY / DESTROY / REGISTRY`。

- `R3` 协同调度组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将窗口实例面、生命周期控制与多窗口协同策略绑定，形成受控窗口联动。
  - `R3.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R3.4` 边界绑定：`COORD / VISIBILITY / DESTROY / RUNTIME`。

- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：无窗口实例骨架直接控制窗口，或销毁后不解绑索引的方案均无效。
  - `R4.3` 输出能力：`C6`。
  - `R4.4` 边界绑定：`REGISTRY / DESTROY / RUNTIME`。

## 5. 验证（Verification）

- `V1` 窗口实例有效性：窗口模板化创建、注册、索引与解绑必须可执行。
- `V2` 生命周期控制有效性：显示、隐藏、聚焦、恢复、关闭与销毁回收必须可执行。
- `V3` 协同调度有效性：多窗口互斥、切换与联动必须满足 `COORD` 约束。
- `V4` 边界符合性：所有有效组合必须满足 `RUNTIME` 约束。
- `V5` 最小必要性：移除任一 `B1~B3` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
