# 模块框架：L1-M3-结果展示层

@module

## 1. 能力声明（Capability Statement）

- `C1` 结果承接能力：承接识别翻译结果并建立当前展示会话。
- `C2` 结果渲染能力：提供原文、译文、状态与错误信息的可视化展示能力。
- `C3` 交互反馈能力：提供复制、关闭、重试、展开与折叠等基础交互能力。
- `C4` 悬浮展示能力：提供低打扰、可拖动、可定位与自动隐藏的浮层展示能力。
- `C5` 展示状态管理能力：提供展示中、失败、空结果与重试等状态切换能力。
- `C6` 非能力项：不负责系统托盘、截图采集与 OCR/翻译执行。

## 2. 边界定义（Boundary / 参数）

- `SESSION` 展示会话约束：每次结果展示必须绑定唯一会话并支持关闭与更新。来源：`C1`。
- `VIEW` 视图约束：视图必须清晰区分原文、译文、状态提示与错误信息。来源：`C2`。
- `ACTION` 交互约束：交互动作必须可追踪、可反馈且不得阻塞主链路。来源：`C3`。
- `FLOAT` 悬浮约束：悬浮窗口需低打扰、可定位、可关闭并与截图上下文相协调。来源：`C4`。
- `STATE` 状态约束：展示状态必须支持加载、成功、失败、空结果与重试。来源：`C5`。
- `PERF` 性能约束：展示与交互反馈必须满足即时可感知响应。来源：`C2 + C3 + C4 + C5`。
- `UX` 体验约束：交付结果必须简洁、可读、可复制，不影响下一次操作。来源：`C2 + C3 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 会话承接基：framework.L2.M15[展示会话管理器]。来源：`C1 + SESSION`。
- `B2` 视图渲染基：framework.L2.M16[结果视图渲染器]。来源：`C2 + VIEW`。
- `B3` 交互动作基：framework.L2.M17[交互动作控制器]。来源：`C3 + ACTION + UX`。
- `B4` 悬浮布局基：framework.L2.M18[悬浮面板布局管理器]。来源：`C4 + FLOAT + PERF`。
- `B5` 状态管理基：framework.L2.M19[展示状态机]。来源：`C5 + STATE`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 会话先行组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立展示会话与上下文，再接入视图与交互能力。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`SESSION`。

- `R2` 视图交付组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：将会话结果绑定到原文译文视图，形成基础结果交付能力。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`VIEW / UX`。

- `R3` 交互反馈组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：在结果视图上叠加复制、关闭、重试与反馈提示，形成可操作结果面板。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`ACTION / UX / PERF`。

- `R4` 悬浮状态组合
  - `R4.1` 参与基：`B2 + B4 + B5`。
  - `R4.2` 组合方式：将悬浮布局与状态管理绑定到结果视图，形成低打扰且具备状态切换能力的展示层。
  - `R4.3` 输出能力：`C2 + C4 + C5`。
  - `R4.4` 边界绑定：`FLOAT / STATE / PERF`。

- `R5` 完整展示模块组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R5.2` 组合方式：将会话承接、视图渲染、交互动作、悬浮布局与状态管理整合为完整结果展示模块。
  - `R5.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R5.4` 边界绑定：`SESSION / VIEW / ACTION / FLOAT / STATE / PERF / UX`。

- `R6` 禁止组合
  - `R6.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R6.2` 组合方式：无展示会话直接渲染结果，或缺少状态管理与交互反馈的结果窗口方案均无效。
  - `R6.3` 输出能力：`C6`。
  - `R6.4` 边界绑定：`SESSION / ACTION / STATE / UX`。

## 5. 验证（Verification）

- `V1` 会话一致性：展示会话必须支持创建、更新、关闭与销毁。
- `V2` 渲染有效性：原文、译文、状态与错误信息必须可视化呈现。
- `V3` 交互有效性：复制、关闭、重试与反馈提示必须可执行。
- `V4` 悬浮有效性：悬浮展示、拖拽定位与显隐控制必须满足 `FLOAT` 约束。
- `V5` 状态管理有效性：加载、成功、失败、空结果与重试状态必须可切换。
- `V6` 体验达标性：展示与交互反馈必须满足 `PERF / UX` 约束。
- `V7` 最小必要性：移除任一 `B1~B5` 后，`V2~V6` 至少一项失败。
- `V8` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
