# 结果展示层:ResultDisplayLayer

@framework

## 1. 能力声明（Capability Statement）

- `C1` 结果承接能力：承接识别翻译结果并建立当前展示会话；展示会话可表现为单结果交付面或对话线程。
- `C2` 结果渲染能力：提供原文、译文、状态与错误信息的可视化展示能力；在对话样式下，应能区分用户输入与系统回复。
- `C3` 交互反馈能力：提供复制、关闭、重试、原文修订重译、展开与折叠等基础交互能力；若采用对话样式，可扩展会话切换与清空类动作。
- `C4` 悬浮展示能力：提供低打扰、可拖动、可定位与自动隐藏的浮层展示能力。
- `C5` 展示状态管理能力：提供展示中、失败、空结果与重试等状态切换能力。
- `C6` 非能力项：不负责系统托盘、截图采集与 OCR/翻译执行。

## 2. 边界定义（Boundary / 参数）

- `SESSION` 展示会话约束：每次结果展示必须绑定唯一会话并支持关闭与更新。来源：`C1`。
- `VIEW` 视图约束：视图必须清晰区分原文、译文、状态提示与错误信息；若采用对话样式，需区分用户消息与系统回复。来源：`C2`。
- `ACTION` 交互约束：交互动作必须可追踪、可反馈且不得阻塞主链路。来源：`C3`。
- `FLOAT` 悬浮约束：悬浮窗口需低打扰、可定位、可关闭并与截图上下文相协调；首屏应优先承接结果与动作，而非大面积装饰或诊断信息。来源：`C4`。
- `STATE` 状态约束：展示状态必须支持加载、成功、失败、空结果与重试。来源：`C5`。
- `PERF` 性能约束：需显式声明首屏展示与交互反馈的 `p95` 时延预算。来源：`C2 + C3 + C4 + C5`。
- `UX` 体验约束：交付结果必须支持复制、关闭、再次触发与原文修订后重译，且不影响下一次操作；原文、译文、状态与动作需位于首屏主阅读路径。若采用对话样式，截图可作为用户消息进入线程，系统回复需承接原文与译文，预览与诊断信息默认降级为次级详情。来源：`C2 + C3 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 展示会话承接基：L1.M5[交互会话管理器]。来源：`C1 + SESSION`。
- `B2` 视图交付基：L1.M14[结果视图渲染器] + L1.M15[交互动作控制器]。来源：`C2 + C3 + VIEW + ACTION + UX`。
- `B3` 浮层状态基：L1.M16[悬浮面板布局管理器] + L1.M17[展示状态机]。来源：`C4 + C5 + FLOAT + STATE + PERF`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 会话承接组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立展示会话与上下文，再接入视图与交互能力。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`SESSION`。

- `R2` 视图交付组合
  - `R2.1` 参与基：`B1 + B2`。
- `R2.2` 组合方式：将会话结果绑定到原文译文视图与基础交互动作，形成可感知、可操作的结果交付面；该交付面可表现为单结果面板或对话线程中的回复卡片。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`VIEW / ACTION / UX`。

- `R3` 浮层状态组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将悬浮布局与状态管理绑定到结果视图与交互动作，形成低打扰且具备状态切换能力的展示层。
  - `R3.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R3.4` 边界绑定：`FLOAT / STATE / PERF / UX`。

- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：无展示会话直接渲染结果，或缺少浮层状态收敛的结果窗口方案均无效。
  - `R4.3` 输出能力：`C6`。
  - `R4.4` 边界绑定：`SESSION / FLOAT / STATE / UX`。

## 5. 验证（Verification）

- `V1` 会话承接有效性：展示会话必须支持创建、更新、关闭与销毁。
- `V2` 视图交付有效性：原文、译文、状态与错误信息必须可视化呈现，复制、关闭、重试与原文修订重译必须可执行；若采用对话样式，截图消息与系统回复必须形成稳定线程关系，且结果与动作需优先于装饰与诊断信息进入首屏。
- `V3` 浮层状态有效性：悬浮展示、拖拽定位、显隐控制与状态切换必须满足 `FLOAT / STATE` 约束。
- `V4` 体验达标性：展示与交互反馈必须满足 `PERF / UX` 约束。
- `V5` 最小必要性：移除任一 `B1~B3` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
