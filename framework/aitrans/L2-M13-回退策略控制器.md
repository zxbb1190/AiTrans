# 模块框架：L2-M13-回退策略控制器

@module

## 1. 能力声明（Capability Statement）

- `C1` 失败分类能力：提供 OCR 与翻译链路失败场景分类能力。
- `C2` 超时控制能力：提供处理阶段超时检测、终止与分流能力。
- `C3` 重试决策能力：提供可重试场景判断、重试次数控制与重试调度能力。
- `C4` 引擎切换能力：提供候选 OCR/翻译引擎切换与替代路径选择能力。
- `C5` 回退结果输出能力：提供失败态、部分成功态与降级结果统一输出能力。
- `C6` 非能力项：不负责具体 OCR/翻译 Provider 的实现细节。

## 2. 边界定义（Boundary / 参数）

- `CLASSIFY` 分类约束：错误必须可被归类为网络、权限、配置、数据或服务失败等类型。来源：`C1`。
- `TIMEOUT` 超时约束：每个处理阶段必须具备独立超时规则。来源：`C2`。
- `RETRY` 重试约束：重试必须有上限、有条件并具备幂等控制。来源：`C3`。
- `SWITCH` 切换约束：候选引擎切换必须具备优先级与适用范围。来源：`C4`。
- `RESULT` 输出约束：回退结果必须统一建模并可被上游理解。来源：`C5`。
- `PERF` 性能约束：回退与重试决策不能无限放大总耗时。来源：`C2 + C3 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 失败分类基：framework.L3.M27[失败分类器]。来源：`C1 + CLASSIFY`。
- `B2` 超时控制基：framework.L3.M28[超时策略评估器]。来源：`C2 + TIMEOUT`。
- `B3` 重试决策基：framework.L3.M29[重试策略执行器]。来源：`C3 + RETRY`。
- `B4` 引擎切换基：framework.L3.M30[提供方切换策略]。来源：`C4 + SWITCH + PERF`。
- `B5` 回退输出基：framework.L3.M31[降级结果构造器]。来源：`C5 + RESULT`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 分类先行组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先完成失败场景分类，再进入超时、重试与切换决策。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`CLASSIFY`。

- `R2` 超时重试组合
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：将失败分类与超时检测、重试决策绑定，形成基础恢复路径。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`TIMEOUT / RETRY / PERF`。

- `R3` 切换恢复组合
  - `R3.1` 参与基：`B1 + B3 + B4`。
  - `R3.2` 组合方式：在失败分类与重试策略基础上叠加候选引擎切换，形成替代恢复路径。
  - `R3.3` 输出能力：`C1 + C3 + C4`。
  - `R3.4` 边界绑定：`RETRY / SWITCH / PERF`。

- `R4` 结果封装组合
  - `R4.1` 参与基：`B2 + B4 + B5`。
  - `R4.2` 组合方式：将超时、切换与降级结果统一封装为标准回退输出。
  - `R4.3` 输出能力：`C2 + C4 + C5`。
  - `R4.4` 边界绑定：`RESULT / TIMEOUT / SWITCH`。

- `R5` 完整回退控制模块组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R5.2` 组合方式：将失败分类、超时控制、重试决策、引擎切换与回退结果输出整合为完整回退控制模块。
  - `R5.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R5.4` 边界绑定：`CLASSIFY / TIMEOUT / RETRY / SWITCH / RESULT / PERF`。

- `R6` 禁止组合
  - `R6.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R6.2` 组合方式：无失败分类直接重试，或无上限地重复切换与重试的方案均无效。
  - `R6.3` 输出能力：`C6`。
  - `R6.4` 边界绑定：`CLASSIFY / RETRY / PERF`。

## 5. 验证（Verification）

- `V1` 分类有效性：失败类型必须可被正确识别与归类。
- `V2` 超时有效性：阶段超时检测与终止必须可执行。
- `V3` 重试有效性：重试条件、次数控制与幂等约束必须正确。
- `V4` 切换有效性：候选引擎切换与替代路径选择必须可执行。
- `V5` 输出有效性：失败态、部分成功态与降级结果必须统一封装。
- `V6` 性能达标性：回退策略必须满足 `PERF` 约束。
- `V7` 最小必要性：移除任一 `B1~B5` 后，`V2~V6` 至少一项失败。
- `V8` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
