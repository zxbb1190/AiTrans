# 桌面宿主层:DesktopHostLayer

@framework

## 1. 能力声明（Capability Statement）

- `C1` 生命周期承载能力：提供应用初始化、退出、异常恢复与单实例管理能力。
- `C2` 系统入口能力：提供托盘驻留、菜单入口与前后台切换能力。
- `C3` 快捷触发能力：提供全局快捷键注册、冲突处理与触发路由能力。
- `C4` 窗口调度能力：提供主窗口、截图窗口、结果窗口的创建、显示、隐藏与销毁能力。
- `C5` 权限协同能力：提供屏幕录制、辅助功能等桌面权限检测与引导能力。
- `C6` 非能力项：不负责截图选区绘制、OCR 识别与翻译处理。

## 2. 边界定义（Boundary / 参数）

- `APP` 应用生命周期约束：必须支持启动、就绪、激活、退出与异常恢复等状态切换。来源：`C1`。
- `TRAY` 托盘入口约束：托盘必须可驻留、可交互、可映射到核心操作入口。来源：`C2`。
- `SHORTCUT` 快捷键约束：全局快捷键必须支持注册、注销、冲突检测与重绑。来源：`C3`。
- `WINDOW` 窗口调度约束：窗口必须具备分工明确、状态可控、生命周期可追踪特性。来源：`C4`。
- `PERM` 权限约束：需识别系统权限状态并支持提示、降级与跳转引导。来源：`C5`。
- `RUNTIME` 运行边界约束：仅主进程可直接访问系统级桌面能力。来源：`C1 + C4 + C5`。
- `COMPAT` 兼容约束：需显式声明目标平台集合、最低支持范围与权限降级路径。来源：`C1 + C5`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 生命周期承接基：L1.M0[应用生命周期管理器]。来源：`C1 + APP + COMPAT`。
- `B2` 系统触发入口基：L1.M1[托盘入口管理器] + L1.M2[全局快捷键路由器]。来源：`C2 + C3 + TRAY + SHORTCUT + COMPAT`。
- `B3` 桌面执行承接基：L1.M3[窗口调度器] + L1.M4[权限协调器]。来源：`C4 + C5 + WINDOW + PERM + RUNTIME + COMPAT`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 生命周期承接组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立应用启动、单实例与退出清理规则，再接入其他入口能力。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`APP / COMPAT`。

- `R2` 系统触发组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在生命周期承接面上叠加托盘入口与全局快捷键路由，形成稳定的系统驻留与触发入口。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`TRAY / SHORTCUT / APP / COMPAT`。

- `R3` 桌面执行组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将系统触发入口与窗口调度、权限协同绑定，形成受控桌面能力访问路径。
  - `R3.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R3.4` 边界绑定：`WINDOW / PERM / RUNTIME / COMPAT`。

- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：绕过生命周期承接直接注册系统入口，或绕过权限协同直接创建窗口与访问桌面能力的方案均无效。
  - `R4.3` 输出能力：`C6`。
  - `R4.4` 边界绑定：`APP / PERM / RUNTIME / COMPAT`。

## 5. 验证（Verification）

- `V1` 生命周期承接有效性：应用启动、激活、退出、异常恢复链路必须完整可追踪。
- `V2` 系统触发有效性：托盘入口、菜单状态、快捷键注册与动作路由必须可执行。
- `V3` 桌面执行有效性：窗口调度、权限检测、引导与降级路径必须覆盖关键桌面能力调用。
- `V4` 边界符合性：所有有效组合必须满足 `RUNTIME / COMPAT` 约束。
- `V5` 最小必要性：移除任一 `B1~B3` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
