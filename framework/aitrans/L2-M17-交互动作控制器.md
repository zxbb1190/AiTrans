# 模块框架：L2-M17-交互动作控制器

@module

## 1. 能力声明（Capability Statement）

- `C1` 动作接入能力：提供复制、关闭、重试、展开与折叠等动作接入能力。
- `C2` 动作分发能力：提供用户动作到具体处理逻辑的分发能力。
- `C3` 反馈提示能力：提供动作成功、失败与处理中反馈提示能力。
- `C4` 节流保护能力：提供高频动作限制、重复触发保护与并发控制能力。
- `C5` 动作审计能力：提供动作触发记录、结果记录与异常追踪能力。
- `C6` 非能力项：不负责结果视图绘制与悬浮面板定位。

## 2. 边界定义（Boundary / 参数）

- `ACTION` 动作约束：交互动作必须具备唯一标识与明确处理目标。来源：`C1`。
- `DISPATCH` 分发约束：动作触发必须被路由到唯一处理器。来源：`C2`。
- `FEEDBACK` 反馈约束：每次动作执行必须具备明确反馈。来源：`C3`。
- `THROTTLE` 节流约束：高频动作必须受限且防止重复执行。来源：`C4`。
- `AUDIT` 审计约束：动作触发过程必须可记录、可追踪。来源：`C5`。
- `UX` 体验约束：动作执行不得阻塞主展示流程。来源：`C2 + C3 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 动作注册基：由动作注册器、动作标识器、处理器绑定器构成。来源：`C1 + ACTION`。
- `B2` 动作分发基：由分发器、路由表、执行器与回调器构成。来源：`C2 + DISPATCH`。
- `B3` 反馈提示基：由成功提示器、失败提示器、处理中提示器与反馈清理器构成。来源：`C3 + FEEDBACK`。
- `B4` 节流保护基：由节流器、去重器、并发锁与重复触发拦截器构成。来源：`C4 + THROTTLE + UX`。
- `B5` 审计记录基：由动作日志器、结果记录器、异常追踪器与查询器构成。来源：`C5 + AUDIT`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 动作先行组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先定义动作标识与处理器绑定，再接入分发与反馈链路。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`ACTION`。

- `R2` 分发反馈组合
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：将动作注册、分发与反馈提示绑定，形成完整动作执行路径。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`DISPATCH / FEEDBACK / UX`。

- `R3` 节流保护组合
  - `R3.1` 参与基：`B2 + B4`。
  - `R3.2` 组合方式：在动作分发链路上叠加节流、去重与并发保护能力。
  - `R3.3` 输出能力：`C2 + C4`。
  - `R3.4` 边界绑定：`THROTTLE / UX`。

- `R4` 反馈审计组合
  - `R4.1` 参与基：`B3 + B5`。
  - `R4.2` 组合方式：将动作执行反馈与审计记录绑定，形成可感知且可追踪的动作闭环。
  - `R4.3` 输出能力：`C3 + C5`。
  - `R4.4` 边界绑定：`FEEDBACK / AUDIT`。

- `R5` 完整交互动作模块组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R5.2` 组合方式：将动作接入、分发、反馈提示、节流保护与审计记录整合为完整交互动作模块。
  - `R5.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R5.4` 边界绑定：`ACTION / DISPATCH / FEEDBACK / THROTTLE / AUDIT / UX`。

- `R6` 禁止组合
  - `R6.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R6.2` 组合方式：无动作注册直接分发事件，或无节流保护直接处理高频触发的方案均无效。
  - `R6.3` 输出能力：`C6`。
  - `R6.4` 边界绑定：`ACTION / THROTTLE / AUDIT`。

## 5. 验证（Verification）

- `V1` 接入有效性：交互动作必须可注册并具备唯一标识。
- `V2` 分发有效性：动作必须可路由到唯一处理器并执行。
- `V3` 反馈有效性：动作执行中、成功与失败提示必须可触发。
- `V4` 节流有效性：高频动作必须受到节流与重复执行保护。
- `V5` 审计有效性：动作触发、结果与异常必须可追踪。
- `V6` 体验达标性：动作执行不得破坏 `UX` 约束。
- `V7` 最小必要性：移除任一 `B1~B5` 后，`V2~V6` 至少一项失败。
- `V8` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
