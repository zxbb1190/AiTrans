# 项目框架模块：L0-M0-桌面截图翻译框架

@project

## 1. 能力声明（Capability Statement）

* `C1` 桌面交互承载能力：提供系统级托盘驻留、全局快捷键、窗口调度与权限协同能力。
* `C2` 截图采集能力：提供桌面区域选择、屏幕图像捕获、图像裁剪与多屏适配能力。
* `C3` 文本识别能力：提供截图内容文字提取、语言初判与识别结果标准化能力。
* `C4` 翻译转换能力：提供源文本翻译、结果格式化与多翻译引擎适配能力。
* `C5` 结果展示能力：提供悬浮展示、原文译文对照、复制与交互反馈能力。
* `C6` 运行治理能力：提供安全控制、日志观测、异常恢复、配置管理与兼容治理能力。
* `C7` 非能力项：不负责具体业务知识库增强、复杂文档排版还原与人工专业术语审校。

## 2. 边界定义（Boundary / 参数）

* `DESKTOP` 桌面宿主约束：必须运行于桌面操作系统，具备托盘、快捷键与窗口管理能力。来源：`C1`。
* `CAPTURE` 截图约束：需支持区域截图、全屏截图与多显示器场景下的坐标映射。来源：`C2`。
* `OCR` 识别约束：识别引擎需支持中英文基础场景，并允许本地或云端切换。来源：`C3`。
* `TRANS` 翻译约束：翻译服务需具备可替换接口、失败回退与响应超时控制。来源：`C4`。
* `UIX` 展示交互约束：结果展示需低打扰、可关闭、可复制、可二次触发。来源：`C5`。
* `RUNTIME` 运行时约束：需明确主进程、渲染进程与预加载桥接边界。来源：`C1 + C6`。
* `BUILD` 构建发布约束：需明确开发态、打包态、自动更新与平台分发模式。来源：`C1 + C6`。
* `SEC` 安全约束：需限制渲染进程权限、控制 IPC 暴露面、治理第三方依赖与敏感配置。来源：`C1 + C6`。
* `OBS` 观测约束：需覆盖截图、OCR、翻译、展示关键链路日志与错误指标。来源：`C6`。
* `COMPAT` 兼容约束：需明确 Windows / macOS 最低支持范围及权限降级策略。来源：`C1 + C6`。
* `PERF` 性能约束：截图响应、OCR 处理与展示反馈需满足可接受交互时延。来源：`C2 + C3 + C4 + C5`。

## 3. 最小可行基（Minimum Viable Bases）

* `B1` 桌面宿主基：framework.L1.M0[桌面宿主层]。来源：`C1 + DESKTOP + RUNTIME + COMPAT`。
* `B2` 截图采集基：framework.L1.M1[截图交互层]。来源：`C2 + CAPTURE + PERF + COMPAT`。
* `B3` 识别翻译基：framework.L1.M2[识别翻译层]。来源：`C3 + C4 + OCR + TRANS + PERF`。
* `B4` 结果展示基：framework.L1.M3[结果展示层]。来源：`C5 + UIX + PERF`。
* `B5` 治理运行基：framework.L1.M4[运行治理层]。来源：`C6 + SEC + OBS + BUILD + RUNTIME + COMPAT`。

## 4. 基组合原则（Base Combination Principles）

* `R1` 宿主先行组合

  * `R1.1` 参与基：`B1`。
  * `R1.2` 组合方式：先建立桌面宿主生命周期、快捷键入口与窗口调度能力，再接入截图与后续处理链路。
  * `R1.3` 输出能力：`C1`。
  * `R1.4` 边界绑定：`DESKTOP / RUNTIME / COMPAT`。

* `R2` 采集闭环组合

  * `R2.1` 参与基：`B1 + B2`。
  * `R2.2` 组合方式：在宿主基上叠加截图交互与图像采集能力，形成从快捷键触发到图像输出的闭环。
  * `R2.3` 输出能力：`C1 + C2`。
  * `R2.4` 边界绑定：`DESKTOP / CAPTURE / PERF / COMPAT`。

* `R3` 识别转换组合

  * `R3.1` 参与基：`B1 + B2 + B3`。
  * `R3.2` 组合方式：在采集闭环基础上接入 OCR 与翻译任务流，形成从截图到文本再到译文的处理链路。
  * `R3.3` 输出能力：`C1 + C2 + C3 + C4`。
  * `R3.4` 边界绑定：`OCR / TRANS / PERF / RUNTIME`。

* `R4` 结果交付组合

  * `R4.1` 参与基：`B1 + B2 + B3 + B4`。
  * `R4.2` 组合方式：将识别翻译结果绑定到低打扰展示界面，形成可见、可复制、可关闭的用户交付链路。
  * `R4.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  * `R4.4` 边界绑定：`UIX / PERF / CAPTURE / OCR / TRANS`。

* `R5` 治理可发布组合

  * `R5.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  * `R5.2` 组合方式：将安全、配置、观测、异常恢复和构建发布策略绑定到完整执行链路，形成可运行、可维护、可发布框架。
  * `R5.3` 输出能力：`C1 + C2 + C3 + C4 + C5 + C6`。
  * `R5.4` 边界绑定：`SEC / OBS / BUILD / RUNTIME / COMPAT / PERF`。

* `R6` 禁止组合

  * `R6.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  * `R6.2` 组合方式：跳过主进程与预加载边界直接在渲染进程访问系统能力，或缺失截图链路、识别链路、治理链路的方案均无效。
  * `R6.3` 输出能力：`C7`。
  * `R6.4` 边界绑定：`RUNTIME / SEC / CAPTURE / OCR / TRANS / OBS / COMPAT`。

## 5. 验证（Verification）

* `V1` 宿主一致性：主进程、预加载、渲染进程职责必须满足 `RUNTIME` 约束，且系统能力不得直接暴露给不受控渲染上下文。
* `V2` 采集完整性：快捷键触发、截图选区、图像裁剪与多屏坐标换算必须形成全链路可执行路径。
* `V3` 识别转换有效性：OCR 与翻译链路必须在成功、失败、超时三类场景下具备确定性输出或回退结果。
* `V4` 交付可用性：结果展示必须满足 `UIX` 约束，支持关闭、复制与再次触发操作。
* `V5` 治理有效性：安全规则、日志事件、错误指标与配置策略必须覆盖截图、识别、翻译、展示关键路径。
* `V6` 兼容符合性：所有有效组合必须满足 `COMPAT` 中定义的平台支持范围与权限降级策略。
* `V7` 性能达标性：截图响应、识别处理与结果展示应满足 `PERF` 设定的用户可接受时延目标。
* `V8` 最小必要性：移除任一 `B1~B5` 后，`V2~V7` 至少一项失败。
* `V9` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
