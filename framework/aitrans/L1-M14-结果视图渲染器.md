# 结果视图渲染器:JieGuoShiTuXuanRanQi

@framework

## 1. 能力声明（Capability Statement）

- `C1` 视图装配能力：提供原文区、译文区、状态区与错误区的视图装配能力。
- `C2` 内容渲染能力：提供文本内容、语言标签与辅助信息渲染能力。
- `C3` 视图切换能力：提供成功态、失败态、空结果态与加载态视图切换能力。
- `C4` 局部刷新能力：提供原文、译文与状态区域的局部刷新能力。
- `C5` 渲染一致性能力：提供视图结构、字段绑定与显示规则一致性能力。
- `C6` 非能力项：不负责交互动作执行与悬浮布局控制。

## 2. 边界定义（Boundary / 参数）

- `VIEW` 视图约束：视图必须包含原文、译文、状态与错误表达区。来源：`C1`。
- `CONTENT` 内容约束：展示内容必须正确绑定到结果字段。来源：`C2`。
- `SWITCH` 切换约束：不同展示状态必须可切换且互斥。来源：`C3`。
- `REFRESH` 刷新约束：局部更新不得破坏整体视图一致性。来源：`C4`。
- `CONSISTENCY` 一致性约束：字段绑定与显示规则必须确定可复现。来源：`C5`。
- `UX` 体验约束：渲染结果必须清晰、稳定、低干扰。来源：`C1 + C2 + C3 + C4 + C5`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 视图布局骨架：L0.M23[面板布局约束器]。来源：`C1 + VIEW`。
- `B2` 内容渲染骨架：L0.M8[标准输出装配器] + L0.M26[展示驱动恢复器]。来源：`C2 + CONTENT`。
- `B3` 状态刷新骨架：L0.M25[展示状态流转器] + L0.M26[展示驱动恢复器]。来源：`C3 + C4 + C5 + SWITCH + REFRESH + CONSISTENCY + UX`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 视图布局组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立视图结构骨架，再接入内容绑定与状态切换。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`VIEW`。

- `R2` 内容渲染组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：将字段绑定与内容渲染叠加到视图骨架，形成基础结果展示能力。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`CONTENT / UX`。

- `R3` 状态刷新组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：在内容渲染基础上接入成功、失败、空结果与加载态切换、局部刷新和恢复收敛，形成一致的视图输出。
  - `R3.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R3.4` 边界绑定：`SWITCH / REFRESH / CONSISTENCY / UX`。

- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：无统一视图布局直接拼接文本区域，或局部刷新不经状态刷新骨架直接替换的方案均无效。
  - `R4.3` 输出能力：`C6`。
  - `R4.4` 边界绑定：`VIEW / REFRESH / CONSISTENCY`。

## 5. 验证（Verification）

- `V1` 视图布局有效性：原文、译文、状态与错误区域必须完整存在。
- `V2` 内容渲染有效性：文本、语言标签与辅助信息必须正确绑定并渲染。
- `V3` 状态刷新有效性：成功态、失败态、空态与加载态必须可切换，局部更新不得破坏整体视图一致性。
- `V4` 体验达标性：渲染结果必须满足 `UX` 约束。
- `V5` 最小必要性：移除任一 `B1~B3` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
