# 权限协调器:QuanXianXieTiaoQi

@framework

## 1. 能力声明（Capability Statement）

- `C1` 权限探测能力：提供系统权限状态识别与查询能力。
- `C2` 权限缓存能力：提供权限状态缓存、刷新与失效管理能力。
- `C3` 权限引导能力：提供权限缺失提示、引导说明与跳转协助能力。
- `C4` 降级决策能力：提供权限不足场景下的降级与替代路径能力。
- `C5` 权限事件通知能力：提供权限变化事件广播与依赖模块同步能力。
- `C6` 非能力项：不负责具体系统权限申请界面的实现。

## 2. 边界定义（Boundary / 参数）

- `CHECK` 探测约束：权限检查必须可重复执行且结果可判定。来源：`C1`。
- `CACHE` 缓存约束：缓存必须支持时效性、手动刷新与失效策略。来源：`C2`。
- `GUIDE` 引导约束：权限不足时必须向用户提供明确引导。来源：`C3`。
- `FALLBACK` 降级约束：关键权限缺失时必须具备可执行降级路径。来源：`C4`。
- `NOTIFY` 通知约束：权限状态变化必须可广播与可消费。来源：`C5`。
- `COMPAT` 兼容约束：不同平台权限语义与检查方式需可适配。来源：`C1 + C3 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 权限判定骨架：L0.M31[权限访问控制器]。来源：`C1 + C2 + C4 + CHECK + CACHE + FALLBACK`。
- `B2` 引导通知骨架：L0.M8[标准输出装配器] + L0.M3[事件总线] + L0.M9[审计追踪器]。来源：`C3 + C5 + GUIDE + NOTIFY + COMPAT`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 权限判定组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立权限探测、缓存刷新与降级决策的统一判定面，形成稳定权限状态源。
  - `R1.3` 输出能力：`C1 + C2 + C4`。
  - `R1.4` 边界绑定：`CHECK / CACHE / FALLBACK`。

- `R2` 引导通知组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：将权限判定结果与用户引导、事件广播、变化追踪绑定，形成权限不足处理闭环。
  - `R2.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R2.4` 边界绑定：`GUIDE / NOTIFY / FALLBACK / COMPAT`。

- `R3` 禁止组合
  - `R3.1` 参与基：`B1 + B2`。
  - `R3.2` 组合方式：无权限判定直接执行关键能力，或权限缺失时无引导通知与降级路径的方案均无效。
  - `R3.3` 输出能力：`C6`。
  - `R3.4` 边界绑定：`CHECK / GUIDE / FALLBACK / NOTIFY`。

## 5. 验证（Verification）

- `V1` 权限判定有效性：权限检查、状态缓存、刷新失效与降级决策必须可执行。
- `V2` 引导通知有效性：权限不足时必须提供明确提示、引导说明、状态广播与变化追踪。
- `V3` 兼容符合性：权限语义与处理路径必须满足 `COMPAT` 约束。
- `V4` 最小必要性：移除任一 `B1~B2` 后，`V1~V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
