# 模块框架：L2-M3-窗口调度器

@module

## 1. 能力声明（Capability Statement）

- `C1` 窗口创建能力：提供不同类型窗口的创建与初始化能力。
- `C2` 窗口注册能力：提供窗口标识注册、索引维护与实例查询能力。
- `C3` 窗口可见性控制能力：提供显示、隐藏、聚焦、置顶与恢复能力。
- `C4` 窗口销毁能力：提供窗口关闭、资源回收与安全销毁能力。
- `C5` 窗口协同能力：提供多窗口之间的切换、互斥与联动调度能力。
- `C6` 非能力项：不负责具体窗口内部 UI 渲染逻辑。

## 2. 边界定义（Boundary / 参数）

- `CREATE` 创建约束：窗口创建必须支持模板化配置与初始化钩子。来源：`C1`。
- `REGISTRY` 注册约束：窗口实例必须具备唯一标识与可查询性。来源：`C2`。
- `VISIBILITY` 可见性约束：窗口显示与隐藏行为必须受统一控制。来源：`C3`。
- `DESTROY` 销毁约束：窗口关闭后必须完成资源回收与注册解绑。来源：`C4`。
- `COORD` 协同约束：多窗口调度必须满足顺序、互斥与依赖关系。来源：`C5`。
- `RUNTIME` 运行边界约束：窗口调度权应统一收敛于主进程。来源：`C1 + C2 + C3 + C4 + C5`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 窗口工厂基：由窗口模板、创建器、初始化器与默认策略构成。来源：`C1 + CREATE`。
- `B2` 注册索引基：由窗口注册表、标识生成器、查询器与解绑器构成。来源：`C2 + REGISTRY`。
- `B3` 可见性控制基：由显示器、隐藏器、聚焦器、置顶器与恢复器构成。来源：`C3 + VISIBILITY`。
- `B4` 销毁回收基：由关闭控制器、销毁器、资源回收器与状态清理器构成。来源：`C4 + DESTROY`。
- `B5` 协同调度基：由窗口协调器、优先级控制器、互斥器与联动触发器构成。来源：`C5 + COORD + RUNTIME`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 创建先行组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先统一窗口模板与创建策略，再接入注册与调度能力。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`CREATE`。

- `R2` 注册控制组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在创建成功后立即注册窗口实例，形成可追踪窗口索引。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`CREATE / REGISTRY`。

- `R3` 可见销毁组合
  - `R3.1` 参与基：`B2 + B3 + B4`。
  - `R3.2` 组合方式：对已注册窗口统一实施显示隐藏与销毁管理，形成完整实例生命周期。
  - `R3.3` 输出能力：`C2 + C3 + C4`。
  - `R3.4` 边界绑定：`VISIBILITY / DESTROY`。

- `R4` 多窗协同组合
  - `R4.1` 参与基：`B2 + B3 + B5`。
  - `R4.2` 组合方式：将窗口可见性控制与多窗口协同策略绑定，形成受控窗口联动。
  - `R4.3` 输出能力：`C2 + C3 + C5`。
  - `R4.4` 边界绑定：`COORD / VISIBILITY / RUNTIME`。

- `R5` 完整窗口模块组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R5.2` 组合方式：将窗口创建、注册、可见性控制、销毁与协同调度整合为完整窗口模块。
  - `R5.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R5.4` 边界绑定：`CREATE / REGISTRY / VISIBILITY / DESTROY / COORD / RUNTIME`。

- `R6` 禁止组合
  - `R6.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R6.2` 组合方式：无注册表直接控制窗口，或销毁后不解绑索引的方案均无效。
  - `R6.3` 输出能力：`C6`。
  - `R6.4` 边界绑定：`REGISTRY / DESTROY / RUNTIME`。

## 5. 验证（Verification）

- `V1` 创建有效性：窗口模板化创建与初始化必须可执行。
- `V2` 注册有效性：窗口实例必须可注册、可索引、可解绑。
- `V3` 可见性有效性：显示、隐藏、聚焦、置顶与恢复必须可执行。
- `V4` 销毁有效性：关闭与销毁后资源必须被正确回收。
- `V5` 协同有效性：多窗口互斥、切换与联动必须满足 `COORD` 约束。
- `V6` 边界符合性：所有有效组合必须满足 `RUNTIME` 约束。
- `V7` 最小必要性：移除任一 `B1~B5` 后，`V2~V6` 至少一项失败。
- `V8` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
