# 模块框架：L1-M1-截图交互层

@module

## 1. 能力声明（Capability Statement）

- `C1` 截图入口承接能力：承接来自宿主模块的截图触发并建立当前会话上下文。
- `C2` 选区交互能力：提供遮罩绘制、鼠标拖拽、区域调整与取消操作能力。
- `C3` 屏幕采集能力：提供屏幕源选择、帧获取、像素裁剪与图像输出能力。
- `C4` 多屏映射能力：提供多显示器、缩放比例与坐标换算能力。
- `C5` 截图结果标准化能力：提供截图元数据、位图对象与后续处理输入格式统一能力。
- `C6` 非能力项：不负责 OCR 文本识别、翻译与结果展示。

## 2. 边界定义（Boundary / 参数）

- `SESSION` 截图会话约束：每次截图必须生成唯一会话并支持完成、取消、失败三类状态。来源：`C1`。
- `OVERLAY` 交互遮罩约束：遮罩层必须全屏、低延迟、可取消且不干扰选区反馈。来源：`C2`。
- `SOURCE` 屏幕源约束：必须支持屏幕源识别、目标屏选取与当前帧读取。来源：`C3`。
- `COORD` 坐标换算约束：必须正确处理逻辑坐标、设备像素比与多屏相对坐标。来源：`C4`。
- `IMAGE` 图像输出约束：截图结果必须统一为可传递、可序列化或可缓存的图像对象。来源：`C5`。
- `PERF` 性能约束：截图操作应满足低延迟与可感知实时反馈要求。来源：`C2 + C3 + C4 + C5`。
- `COMPAT` 兼容约束：需支持目标平台下的多屏与高分屏场景。来源：`C3 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 会话管理基：framework.L2.M5[截图会话管理器]。来源：`C1 + SESSION`。
- `B2` 遮罩交互基：framework.L2.M6[遮罩选区控制器]。来源：`C2 + OVERLAY + PERF`。
- `B3` 图像采集基：framework.L2.M7[屏幕帧获取器]。来源：`C3 + SOURCE + IMAGE`。
- `B4` 坐标映射基：framework.L2.M8[坐标映射引擎]。来源：`C4 + COORD + COMPAT`。
- `B5` 结果标准化基：framework.L2.M9[截图结果标准化器]。来源：`C5 + IMAGE + SESSION`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 会话先行组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立截图会话状态与上下文，再进入遮罩交互与图像采集阶段。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`SESSION`。

- `R2` 交互采集组合
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：将选区交互与帧采集绑定，形成从拖拽到图像输出的截图闭环。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`OVERLAY / SOURCE / PERF`。

- `R3` 多屏校正组合
  - `R3.1` 参与基：`B2 + B3 + B4`。
  - `R3.2` 组合方式：将选区逻辑坐标与实际设备像素映射绑定，保证多屏和缩放场景下的裁剪正确性。
  - `R3.3` 输出能力：`C2 + C3 + C4`。
  - `R3.4` 边界绑定：`COORD / COMPAT / PERF`。

- `R4` 输出标准化组合
  - `R4.1` 参与基：`B1 + B3 + B4 + B5`。
  - `R4.2` 组合方式：在采集和坐标校正后统一截图输出格式，形成下游可消费的标准结果。
  - `R4.3` 输出能力：`C1 + C3 + C4 + C5`。
  - `R4.4` 边界绑定：`IMAGE / SESSION / COORD`。

- `R5` 完整截图模块组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R5.2` 组合方式：将会话、遮罩交互、图像采集、坐标映射与结果标准化整合为完整截图模块。
  - `R5.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R5.4` 边界绑定：`SESSION / OVERLAY / SOURCE / COORD / IMAGE / PERF / COMPAT`。

- `R6` 禁止组合
  - `R6.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R6.2` 组合方式：跳过坐标映射直接裁剪图像，或无会话上下文直接输出截图结果的方案均无效。
  - `R6.3` 输出能力：`C6`。
  - `R6.4` 边界绑定：`SESSION / COORD / IMAGE / COMPAT`。

## 5. 验证（Verification）

- `V1` 会话一致性：截图会话必须具备创建、取消、完成、失败的明确状态流转。
- `V2` 交互有效性：遮罩显示、拖拽选区、取消与确认操作必须可执行。
- `V3` 采集完整性：屏幕源读取、帧获取与裁剪链路必须全链路可执行。
- `V4` 映射正确性：多屏与高分屏场景下的坐标换算结果必须正确。
- `V5` 输出标准化：截图结果必须满足 `IMAGE` 约束并可被下游模块直接消费。
- `V6` 性能达标性：交互反馈与截图输出必须满足 `PERF` 约束。
- `V7` 最小必要性：移除任一 `B1~B5` 后，`V2~V6` 至少一项失败。
- `V8` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
