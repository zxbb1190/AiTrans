# 可观测性中心:KeGuanCeXingZhongXin

@framework

## 1. 能力声明（Capability Statement）

- `C1` 日志记录能力：提供关键链路日志记录、分级与结构化输出能力。
- `C2` 事件埋点能力：提供业务事件埋点、阶段事件记录与统计聚合能力。
- `C3` 错误采集能力：提供异常捕获、错误聚合与失败追踪能力。
- `C4` 链路关联能力：提供任务、会话与调用链路关联标识能力。
- `C5` 诊断输出能力：提供调试定位、问题回放与诊断上下文输出能力。
- `C6` 非能力项：不负责业务逻辑恢复与具体告警系统实现。

## 2. 边界定义（Boundary / 参数）

- `LOG` 日志约束：日志必须具备级别、结构化字段与输出策略。来源：`C1`。
- `METRIC` 事件约束：关键事件必须可埋点、统计与聚合。来源：`C2`。
- `ERROR` 错误约束：错误必须具备捕获、归类与上下文绑定能力。来源：`C3`。
- `TRACE` 链路约束：关键调用链路必须具备统一关联标识。来源：`C4`。
- `DIAG` 诊断约束：问题定位必须可输出足够诊断上下文。来源：`C5`。
- `PRIVACY` 隐私约束：日志与诊断信息不得泄露不必要敏感数据。来源：`C1 + C3 + C5`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 事件记录骨架：L0.M29[观测记录器]。来源：`C1 + C2 + C3 + LOG + METRIC + ERROR`。
- `B2` 链路诊断骨架：L0.M30[链路诊断收敛器]。来源：`C4 + C5 + TRACE + DIAG + PRIVACY`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 事件记录组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立统一日志级别、事件埋点与错误采集机制，形成基础观测面。
  - `R1.3` 输出能力：`C1 + C2 + C3`。
  - `R1.4` 边界绑定：`LOG / METRIC / ERROR`。

- `R2` 链路诊断组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：将基础观测面与链路标识、诊断快照、隐私裁剪绑定，形成可定位、可回放的观测闭环。
  - `R2.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R2.4` 边界绑定：`TRACE / DIAG / PRIVACY`。

- `R3` 禁止组合
  - `R3.1` 参与基：`B1 + B2`。
  - `R3.2` 组合方式：无链路关联骨架直接拼接跨阶段日志，或诊断信息不做隐私裁剪直接输出的方案均无效。
  - `R3.3` 输出能力：`C6`。
  - `R3.4` 边界绑定：`TRACE / DIAG / PRIVACY`。

## 5. 验证（Verification）

- `V1` 事件记录有效性：日志级别、结构化字段、关键业务事件与错误采集必须正确执行。
- `V2` 链路诊断有效性：关键流程必须具备统一关联标识，且问题定位必须可输出足够诊断上下文。
- `V3` 隐私符合性：所有有效组合必须满足 `PRIVACY` 约束。
- `V4` 最小必要性：移除任一 `B1~B2` 后，`V1~V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
