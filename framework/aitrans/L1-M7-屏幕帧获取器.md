# 屏幕帧获取器:PingMuZhenHuoQuQi

@framework

## 1. 能力声明（Capability Statement）

- `C1` 屏幕源发现能力：提供可用屏幕源识别与目标源选择能力。
- `C2` 帧获取能力：提供当前屏幕帧快照获取能力。
- `C3` 图像裁剪能力：提供基于选区的图像裁剪能力。
- `C4` 图像编码能力：提供截图结果的位图、Buffer 或 DataURL 编码能力。
- `C5` 异常回退能力：提供源不可用、读取失败与裁剪失败时的回退能力。
- `C6` 非能力项：不负责选区坐标换算逻辑。

## 2. 边界定义（Boundary / 参数）

- `SOURCE` 屏幕源约束：必须支持源发现、目标选取与不可用判定。来源：`C1`。
- `FRAME` 帧读取约束：帧快照必须具备时效性与可读取性。来源：`C2`。
- `CROP` 裁剪约束：裁剪必须接受标准选区输入并输出有效图像。来源：`C3`。
- `ENCODE` 编码约束：图像输出必须支持统一编码格式。来源：`C4`。
- `FALLBACK` 回退约束：失败场景必须可返回明确错误或回退结果。来源：`C5`。
- `PERF` 性能约束：帧获取与裁剪操作必须满足可接受响应时间。来源：`C2 + C3 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 源区域解析骨架：L0.M15[显示器拓扑提供器] + L0.M17[多屏区域校正器]。来源：`C1 + C2 + C3 + SOURCE + FRAME + CROP`。
- `B2` 编码输出骨架：L0.M8[标准输出装配器]。来源：`C4 + ENCODE`。
- `B3` 故障回退骨架：L0.M7[错误分流器] + L0.M22[降级结果构造器]。来源：`C5 + FALLBACK + PERF`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 源区域解析组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立屏幕源发现、目标区域解析与边界修正，形成受控帧读取入口。
  - `R1.3` 输出能力：`C1 + C2 + C3`。
  - `R1.4` 边界绑定：`SOURCE / FRAME / CROP`。

- `R2` 编码输出组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：将帧快照与裁剪结果统一编码为标准图像输出。
  - `R2.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R2.4` 边界绑定：`FRAME / CROP / ENCODE`。

- `R3` 回退保护组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将源区域解析与编码输出过程接入故障回退机制，形成受控读取路径。
  - `R3.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R3.4` 边界绑定：`FALLBACK / PERF / SOURCE / CROP`。

- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：无源区域解析直接读取，或失败后无错误分流与回退结果的方案均无效。
  - `R4.3` 输出能力：`C6`。
  - `R4.4` 边界绑定：`SOURCE / CROP / FALLBACK`。

## 5. 验证（Verification）

- `V1` 源区域解析有效性：可用屏幕源识别、帧读取与标准选区裁剪必须可执行。
- `V2` 编码输出有效性：裁剪结果必须可编码为统一输出格式。
- `V3` 回退有效性：源不可用、读取失败与裁剪失败必须可分流处理并给出回退结果。
- `V4` 性能达标性：读取与裁剪必须满足 `PERF` 约束。
- `V5` 最小必要性：移除任一 `B1~B3` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
