# 模块框架：L2-M2-全局快捷键路由器

@module

## 1. 能力声明（Capability Statement）

- `C1` 快捷键注册能力：提供全局快捷键注册、注销与重注册能力。
- `C2` 冲突识别能力：提供快捷键冲突检测、冲突提示与冲突规避能力。
- `C3` 动作路由能力：提供快捷键触发到具体系统动作的分发能力。
- `C4` 配置更新能力：提供快捷键动态修改、持久化与热更新能力。
- `C5` 触发审计能力：提供快捷键触发日志记录与失败追踪能力。
- `C6` 非能力项：不负责动作执行目标的内部实现。

## 2. 边界定义（Boundary / 参数）

- `REGISTER` 注册约束：快捷键必须支持注册成功、失败与注销状态。来源：`C1`。
- `CONFLICT` 冲突约束：冲突必须可识别、可提示、可回退。来源：`C2`。
- `ROUTE` 路由约束：触发事件必须映射到唯一有效动作。来源：`C3`。
- `CONFIG` 配置约束：快捷键配置必须支持读取、变更与持久化。来源：`C4`。
- `AUDIT` 审计约束：每次注册与触发必须可追踪。来源：`C5`。
- `COMPAT` 兼容约束：快捷键语义需适配不同平台键位约定。来源：`C1 + C2 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 注册控制基：由注册器、注销器、状态缓存器与重注册器构成。来源：`C1 + REGISTER`。
- `B2` 冲突处理基：由冲突检测器、占用检查器、提示器与回退器构成。来源：`C2 + CONFLICT + COMPAT`。
- `B3` 动作路由基：由动作映射表、路由器、触发器与分发器构成。来源：`C3 + ROUTE`。
- `B4` 配置管理基：由配置读取器、更新器、持久化器与热加载器构成。来源：`C4 + CONFIG`。
- `B5` 审计记录基：由触发日志器、失败记录器、统计器与查询器构成。来源：`C5 + AUDIT`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 注册先行组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立快捷键注册生命周期，再接入冲突控制与动作路由。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`REGISTER`。

- `R2` 冲突控制组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在注册链路上叠加冲突检测与回退，形成受控注册路径。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`REGISTER / CONFLICT / COMPAT`。

- `R3` 动作分发组合
  - `R3.1` 参与基：`B1 + B3`。
  - `R3.2` 组合方式：将已注册快捷键与动作路由绑定，形成输入到动作的分发链路。
  - `R3.3` 输出能力：`C1 + C3`。
  - `R3.4` 边界绑定：`REGISTER / ROUTE`。

- `R4` 配置审计组合
  - `R4.1` 参与基：`B4 + B5`。
  - `R4.2` 组合方式：将快捷键配置变更与触发审计绑定，形成可更新、可追踪闭环。
  - `R4.3` 输出能力：`C4 + C5`。
  - `R4.4` 边界绑定：`CONFIG / AUDIT`。

- `R5` 完整快捷模块组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R5.2` 组合方式：将注册控制、冲突识别、动作路由、配置更新与审计记录整合为完整快捷键模块。
  - `R5.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R5.4` 边界绑定：`REGISTER / CONFLICT / ROUTE / CONFIG / AUDIT / COMPAT`。

- `R6` 禁止组合
  - `R6.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R6.2` 组合方式：无冲突检测直接覆盖注册，或无动作路由直接消费触发事件的方案均无效。
  - `R6.3` 输出能力：`C6`。
  - `R6.4` 边界绑定：`CONFLICT / ROUTE / CONFIG`。

## 5. 验证（Verification）

- `V1` 注册有效性：快捷键注册、注销与重注册必须可执行。
- `V2` 冲突有效性：冲突检测、提示与回退必须可执行。
- `V3` 路由有效性：触发事件必须可分发到唯一有效动作。
- `V4` 配置有效性：快捷键配置必须可修改并持久化。
- `V5` 审计有效性：注册与触发记录必须可追踪。
- `V6` 兼容符合性：不同平台快捷键语义必须满足 `COMPAT` 约束。
- `V7` 最小必要性：移除任一 `B1~B5` 后，`V2~V6` 至少一项失败。
- `V8` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
