# 模块框架：L2-M5-截图会话管理器

@module

## 1. 能力声明（Capability Statement）

- `C1` 会话创建能力：提供截图会话初始化与唯一标识生成能力。
- `C2` 状态流转能力：提供待开始、进行中、完成、取消、失败等状态流转能力。
- `C3` 上下文管理能力：提供当前截图选区、屏幕源与临时数据管理能力。
- `C4` 取消终止能力：提供主动取消、异常终止与资源回收能力。
- `C5` 会话事件通知能力：提供会话开始、更新、结束与失败事件广播能力。
- `C6` 非能力项：不负责图像采集与选区绘制实现。

## 2. 边界定义（Boundary / 参数）

- `SESSION` 会话约束：每次截图必须绑定唯一会话标识。来源：`C1`。
- `STATE` 状态约束：会话状态必须具备单向合法流转规则。来源：`C2`。
- `CONTEXT` 上下文约束：会话上下文必须支持读写、更新与销毁。来源：`C3`。
- `CANCEL` 取消约束：取消与终止必须可中断后续链路。来源：`C4`。
- `EVENT` 事件约束：会话事件必须可追踪与订阅。来源：`C5`。
- `RUNTIME` 运行约束：同一时刻仅允许存在受控主会话。来源：`C1 + C2 + C3 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 会话初始化基：framework.L3.M0[会话初始化器]。来源：`C1 + SESSION`。
- `B2` 状态机基：framework.L3.M2[状态机内核]。来源：`C2 + STATE`。
- `B3` 上下文存储基：framework.L3.M1[上下文存储器]。来源：`C3 + CONTEXT`。
- `B4` 终止控制基：framework.L3.M4[生命周期终止控制器]。来源：`C4 + CANCEL`。
- `B5` 事件广播基：framework.L3.M3[事件总线]。来源：`C5 + EVENT + RUNTIME`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 初始化先行组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先完成会话创建与标识分配，再接入状态与上下文能力。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`SESSION`。

- `R2` 状态上下文组合
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：将状态机与会话上下文绑定，形成完整会话管理结构。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`STATE / CONTEXT`。

- `R3` 中断回收组合
  - `R3.1` 参与基：`B2 + B3 + B4`。
  - `R3.2` 组合方式：在状态与上下文之上叠加取消和终止控制，形成可中断会话路径。
  - `R3.3` 输出能力：`C2 + C3 + C4`。
  - `R3.4` 边界绑定：`CANCEL / STATE / CONTEXT`。

- `R4` 事件同步组合
  - `R4.1` 参与基：`B1 + B2 + B5`。
  - `R4.2` 组合方式：将会话创建与状态变更绑定到事件广播链路，形成对外同步能力。
  - `R4.3` 输出能力：`C1 + C2 + C5`。
  - `R4.4` 边界绑定：`EVENT / RUNTIME`。

- `R5` 完整会话模块组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R5.2` 组合方式：将会话创建、状态机、上下文存储、终止控制与事件广播整合为完整会话模块。
  - `R5.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R5.4` 边界绑定：`SESSION / STATE / CONTEXT / CANCEL / EVENT / RUNTIME`。

- `R6` 禁止组合
  - `R6.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R6.2` 组合方式：无状态机直接变更会话状态，或结束后不清理上下文的方案均无效。
  - `R6.3` 输出能力：`C6`。
  - `R6.4` 边界绑定：`STATE / CONTEXT / CANCEL`。

## 5. 验证（Verification）

- `V1` 创建有效性：会话创建与唯一标识分配必须可执行。
- `V2` 状态有效性：会话状态流转必须合法且可追踪。
- `V3` 上下文有效性：上下文读写、更新与销毁必须可执行。
- `V4` 终止有效性：取消与失败终止必须中断后续链路。
- `V5` 事件有效性：会话事件必须可广播、可订阅、可审计。
- `V6` 边界符合性：同一时刻会话控制必须满足 `RUNTIME` 约束。
- `V7` 最小必要性：移除任一 `B1~B5` 后，`V2~V6` 至少一项失败。
- `V8` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
