# 回退策略控制器:HuiTuiCeLveKongZhiQi

@framework

## 1. 能力声明（Capability Statement）

- `C1` 失败分类能力：提供 OCR 与翻译链路失败场景分类能力。
- `C2` 超时控制能力：提供处理阶段超时检测、终止与分流能力。
- `C3` 重试决策能力：提供可重试场景判断、重试次数控制与重试调度能力。
- `C4` 引擎切换能力：提供候选 OCR/翻译引擎切换与替代路径选择能力。
- `C5` 回退结果输出能力：提供失败态、部分成功态与降级结果统一输出能力。
- `C6` 非能力项：不负责具体 OCR/翻译 Provider 的实现细节。

## 2. 边界定义（Boundary / 参数）

- `CLASSIFY` 分类约束：错误必须可被归类为网络、权限、配置、数据或服务失败等类型。来源：`C1`。
- `TIMEOUT` 超时约束：每个处理阶段必须具备独立超时规则。来源：`C2`。
- `RETRY` 重试约束：重试必须有上限、有条件并具备幂等控制。来源：`C3`。
- `SWITCH` 切换约束：候选引擎切换必须具备优先级与适用范围。来源：`C4`。
- `RESULT` 输出约束：回退结果必须统一建模并可被上游理解。来源：`C5`。
- `PERF` 性能约束：回退与重试决策不能无限放大总耗时。来源：`C2 + C3 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 失败判定骨架：L0.M20[失败超时判定器]。来源：`C1 + C2 + CLASSIFY + TIMEOUT`。
- `B2` 恢复切换骨架：L0.M21[重试切换执行器]。来源：`C3 + C4 + RETRY + SWITCH + PERF`。
- `B3` 回退结果骨架：L0.M22[降级结果构造器]。来源：`C5 + RESULT`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 失败判定组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先完成失败分类与阶段超时判定，形成统一回退入口。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`CLASSIFY / TIMEOUT`。

- `R2` 恢复切换组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在失败判定基础上叠加重试决策与候选引擎切换，形成受控恢复路径。
  - `R2.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R2.4` 边界绑定：`RETRY / SWITCH / PERF`。

- `R3` 回退结果组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将失败判定、恢复切换与降级结果构造绑定，形成统一回退输出。
  - `R3.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R3.4` 边界绑定：`RESULT / TIMEOUT / RETRY / SWITCH`。

- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：无失败判定直接重试，或无回退结果骨架直接输出失败碎片的方案均无效。
  - `R4.3` 输出能力：`C6`。
  - `R4.4` 边界绑定：`CLASSIFY / RETRY / RESULT / PERF`。

## 5. 验证（Verification）

- `V1` 失败判定有效性：失败类型识别与阶段超时检测必须可执行。
- `V2` 恢复切换有效性：重试条件、次数控制、幂等约束与候选切换必须正确。
- `V3` 回退结果有效性：失败态、部分成功态与降级结果必须统一封装。
- `V4` 性能达标性：回退策略必须满足 `PERF` 约束。
- `V5` 最小必要性：移除任一 `B1~B3` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
